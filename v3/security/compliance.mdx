Refold is built to enterprise security standards. This page summarizes the compliance frameworks Refold aligns to and the controls behind them. For **authoritative, always-current certification status** — and to request reports — see the [Refold Trust Portal](https://security.refold.ai/compliance).

## Compliance frameworks

Refold aligns to the following industry standards:

- **SOC 2** — AICPA Service Organization Control
- **ISO 27001:2022** — Information Security Management System
- **VAPT** — Vulnerability Assessment &amp; Penetration Testing
- **GDPR** — EU data protection and privacy
- **HIPAA** — protected health information

<Note>
Check the [trust portal](https://security.refold.ai/compliance)
  for the current state of each framework and to request reports.
</Note>

## Security posture

<CardGroup cols={2}>
  <Card title="Cloud infrastructure" icon="cloud">
    Hosted on AWS. Data is encrypted with **AES-256 at rest** and **TLS 1.2+ in transit**. Continuous vulnerability management with regular penetration tests and dynamic (DAST) and static (SAST) security scans.
  </Card>
  <Card title="Data protection" icon="lock">
    Encryption at rest and in transit, with an additional application-layer of encryption for PII.
  </Card>
  <Card title="Monitoring &amp; incident response" icon="shield-halved">
    24/7 security monitoring with automated threat detection, defined incident-response protocols, and real-time logging and alerts.
  </Card>
  <Card title="Business continuity &amp; DR" icon="arrows-rotate">
    Frequent encrypted backups, routine disaster-recovery testing, and defined recovery-time (RTO) and recovery-point (RPO) objectives.
  </Card>
</CardGroup>

For the full picture — including application &amp; product security, privacy &amp; user control, secure access management, responsible disclosure, and transparency — see the [Refold security overview](https://www.refold.ai/security).
