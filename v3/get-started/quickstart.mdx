Refold connects your product to third-party apps through [linked accounts](/v3/concepts/linked-account), connectors, actions, and workflows. This quickstart runs that loop end to end with **NetSuite**: get an API key, connect an account, fetch data from it, and run a workflow. Plan about 10 minutes.

## Prerequisites

- **A Refold account.** Log in at [app.refold.ai](https://app.refold.ai). Don't have one? [Contact us](https://www.refold.ai/contact-us) to get set up.
- **A NetSuite account.** Use a sandbox account for this walkthrough; you'll connect it with key-based authentication (covered below).

<Note>
  Every API call uses the base URL `https://app.refold.ai` and sends your API key in the `x-api-key` header. Account-scoped calls also send the customer's `linked_account_id` as a header.
</Note>

## Get your API key

<Steps>
  <Step title="Open Settings > Credentials">
    In the [Refold Console](https://app.refold.ai/settings/credentials), go to **Settings > Credentials**.
  </Step>
  <Step title="Copy your key">
    Copy your API key and export it as an environment variable so the commands below can read it.

    ```bash
    export REFOLD_API_KEY="your-api-key-here"
    ```
  </Step>
</Steps>

## Choose an authentication method

Refold supports multiple authentication methods per application, so you choose what to offer. For this walkthrough, use **key-based authentication** — it's the quickest because it needs **no app-level configuration**. The customer supplies their NetSuite credentials directly in the Connect portal when they connect (next step).

<Note>
NetSuite also supports **OAuth 2.0** and **M2M**, which *do* require app-level setup (client ID and secret, and so on). Offer whichever your customers prefer — see the [NetSuite connector docs](/applications/providers/netsuite).
</Note>

## Create a linked account

A linked account represents one of your customers. Create one through the API — you choose the `linked_account_id`, then reuse it in every account-scoped call.

```bash cURL
curl -X POST https://app.refold.ai/api/v2/public/linked-account \
  -H "x-api-key: $REFOLD_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "linked_account_id": "demo-user-1",
    "name": "Demo User"
  }'
```

```json Output
{ "success": true }
```

## Connect NetSuite

Generate a [Connect portal](/v3/embedded/connect-flow) URL for the linked account, then open it to authenticate the account.

```bash cURL
curl -X POST https://app.refold.ai/api/v2/public/connect-url \
  -H "x-api-key: $REFOLD_API_KEY" \
  -H "linked_account_id: demo-user-1" \
  -H "Content-Type: application/json" \
  -d '{ "name": "Demo User" }'
```

```json Output
{ "hosted_url": "https://app.refold.ai/connect/eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." }
```

<Steps>
  <Step title="Open the Connect portal">
    Open the `hosted_url` from the response in your browser.
  </Step>
  <Step title="Authenticate NetSuite">
    Find **NetSuite** and enter your NetSuite credentials — with key-based auth, they're supplied right here in the portal. Complete the connection and a **Connected** badge appears.
  </Step>
</Steps>

<Note>
  The Connect portal is Refold's hosted connection UI. You can build your own with the [frontend SDKs](/v3/embedded/connect-flow).
</Note>

## Fetch data from NetSuite

With the account connected, read data straight from the customer's NetSuite. Refold uses the stored credentials automatically, so no NetSuite tokens go in the request.

First, list the actions to find the one you need and its exact ID:

```bash cURL
curl https://app.refold.ai/api/v2/integration-schema/netsuite/actions \
  -H "x-api-key: $REFOLD_API_KEY" \
  -H "linked_account_id: demo-user-1"
```

Then execute a read action to fetch records. For example, **Get List Of Records Invoice** (`get_list_of_records_invoice`) returns the account's invoices:

```bash cURL
curl -X POST https://app.refold.ai/api/v2/integration-schema/netsuite/actions/get_list_of_records_invoice/execute \
  -H "x-api-key: $REFOLD_API_KEY" \
  -H "linked_account_id: demo-user-1" \
  -H "Content-Type: application/json" \
  -d '{}'
```

```json Output
{
  "success": true,
  "data": {
    "items": [
      { "id": "1024", "tranId": "INV-1024", "entity": "Acme Corp", "total": 4800.00, "status": "open" }
    ]
  }
}
```

## Run a workflow

A single action is one call; a [workflow](/v3/workflows/overview) chains steps — read, transform, then write, or fan out across several apps. Run a published workflow for the linked account through the workflow API. Pass the workflow's ID in the path and the connector slug, `netsuite`, in the `slug` header.

<Note>
  The workflow must be **published** before you can run it. Get its ID from [List Workflows](/v3/api-reference/workflows/list-workflows) or the Console.
</Note>

```bash cURL
curl -X POST https://app.refold.ai/api/v2/public/workflow/YOUR_WORKFLOW_ID/execute \
  -H "x-api-key: $REFOLD_API_KEY" \
  -H "linked_account_id: demo-user-1" \
  -H "slug: netsuite" \
  -H "Content-Type: application/json" \
  -d '{ "payload": { "invoice_id": "1024" } }'
```

```json Output
{
  "message": "Workflow execution started",
  "execution_id": "6a314c54d4616e99e7fa5a02"
}
```

Refold acknowledges with an `execution_id` and starts the run. Open **Logs** in the Console to follow it, or add the `sync_execution: true` header to get the output inline. For every option, see [Execute Workflow](/v3/api-reference/workflows/execute-workflow).
