---
title: List Actions
api: get /api/v2/integration-schema/:slug/actions
description: Retrieve all available actions for a specific integration application.
---

Retrieve a list of all available actions for a specific integration application. Each action represents an operation you can execute against the connected application, such as creating contacts, updating records, or fetching data.

## Authentication

This API supports two authentication methods. Use either Session Token OR API Key with Linked Account ID.

<Tabs>
  <Tab title="Session Token">
    <ParamField header="Authorization" type="string" required>
      Bearer token for authentication.
      
      **Format:** `Bearer YOUR_SESSION_TOKEN`
    </ParamField>
  </Tab>
  <Tab title="API Key + Linked Account">
    <ParamField header="x-api-key" type="string" required>
      Your Refold API key. Find it in **Settings → Credentials**.
    </ParamField>
    
    <ParamField header="linked_account_id" type="string" required>
      The unique identifier for the linked account.
      
      **Example:** `user_12345`
    </ParamField>
  </Tab>
</Tabs>

## Path Parameters

<ParamField path="slug" type="string" required>
  The application slug identifier.
  
  **Example:** `hubspot`, `salesforce`, `slack`
</ParamField>

## Response

<Tabs>
  <Tab title="200 OK">
    Returns an array of action objects.

    <ResponseField name="name" type="string">
      Display name of the action
    </ResponseField>

    <ResponseField name="action_id" type="string">
      Unique identifier for the action. Use this ID when executing the action or fetching parameters.
    </ResponseField>

    <ResponseField name="description" type="string">
      Description of what the action does

  </Tab>

## Example: List Available Actions
```javascript
async function listActions(slug, linkedAccountId, apiKey) {
  const response = await fetch(
    `https://api.gocobalt.io/api/v2/integration-schema/${slug}/actions`,
    {
      headers: {
        'x-api-key': apiKey,
        'linked_account_id': linkedAccountId
      }
    }
  );

  if (!response.ok) {
    const error = await response.json();
    throw new Error(error.message);
  }

  return response.json();
}

// Usage: Get all HubSpot actions
const actions = await listActions('hubspot', 'user_12345', 'your_api_key');

console.log('Available actions:');
actions.forEach(action => {
  console.log(`- ${action.name} (${action.action_id})`);
});
```

## Example: Build Action Selector UI
```javascript
async function getActionsByCategory(slug, linkedAccountId, apiKey) {
  const actions = await listActions(slug, linkedAccountId, apiKey);

  // Categorize actions by operation type
  const categories = {
    create: actions.filter(a => a.action_id.startsWith('create_')),
    update: actions.filter(a => a.action_id.startsWith('update_')),
    delete: actions.filter(a => a.action_id.startsWith('delete_')),
    get: actions.filter(a => a.action_id.startsWith('get_') || a.action_id.startsWith('search_')),
    other: actions.filter(a => 
      !a.action_id.startsWith('create_') && 
      !a.action_id.startsWith('update_') && 
      !a.action_id.startsWith('delete_') &&
      !a.action_id.startsWith('get_') &&
      !a.action_id.startsWith('search_')
    )
  };

  return categories;
}

// Usage: Build UI with categorized actions
const categories = await getActionsByCategory('salesforce', 'user_12345', 'your_api_key');

console.log(`Create actions: ${categories.create.length}`);
console.log(`Read actions: ${categories.get.length}`);
console.log(`Update actions: ${categories.update.length}`);
console.log(`Delete actions: ${categories.delete.length}`);
```
