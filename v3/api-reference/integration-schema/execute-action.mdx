---
title: Execute Action
api: post /api/v2/integration-schema/:slug/actions/:action_id/execute
description: Execute a specific action within an integration application.
---

Execute a specific action within an integration application. The request body should contain the parameters required for the action as defined by the action's parameter schema.

<Info>
Authentication is handled automatically. You do not need to include API keys, tokens, or authentication headers in the request body.
</Info>

## Authentication

This API supports two authentication methods. Use either Session Token OR API Key with Linked Account ID.

<Tabs>
  <Tab title="Session Token">
    <ParamField header="Authorization" type="string" required>
      Bearer token for authentication.
      
      **Format:** `Bearer YOUR_SESSION_TOKEN`
    </ParamField>
  </Tab>
  <Tab title="API Key + Linked Account">
    <ParamField header="x-api-key" type="string" required>
      Your Refold API key. Find it in **Settings → Credentials**.
    </ParamField>
    
    <ParamField header="linked_account_id" type="string" required>
      The unique identifier for the linked account.
      
      **Example:** `user_12345`
    </ParamField>
  </Tab>
</Tabs>

## Path Parameters

<ParamField path="slug" type="string" required>
  The application slug identifier.
  
  **Example:** `hubspot`, `salesforce`, `slack`
</ParamField>

<ParamField path="action_id" type="string" required>
  The unique action identifier to execute.
  
  **Example:** `create_contact`, `create_deal`, `update_account`
</ParamField>

## Body Parameters

<ParamField body="*" type="object" required>
  Dynamic parameters based on the action's schema. Use the [Get Action Schema](/v3/api-reference/integration-schema/get-action-parameters) endpoint to retrieve the required and optional parameters.
  
  <Tip>
  The body structure varies depending on the action being executed. Each action has its own set of required and optional parameters.
  </Tip>
</ParamField>

## Example: Execute Action
```javascript
async function executeAction(slug, actionId, params, linkedAccountId, apiKey) {
  const response = await fetch(
    `https://api.gocobalt.io/api/v2/integration-schema/${slug}/actions/${actionId}/execute`,
    {
      method: 'POST',
      headers: {
        'x-api-key': apiKey,
        'linked_account_id': linkedAccountId,
        'Content-Type': 'application/json'
      },
      body: JSON.stringify(params)
    }
  );

  const result = await response.json();

  if (!response.ok || result.node_status === 'Failed') {
    throw new Error(result.message || 'Action execution failed');
  }

  return result;
}

// Usage: Create a contact in HubSpot
const result = await executeAction(
  'hubspot',
  'create_contact',
  {
    email: 'john.smith@acme.com',
    firstname: 'John',
    lastname: 'Smith',
    company: 'Acme Inc.',
    jobtitle: 'Software Engineer'
  },
  'user_12345',
  'your_api_key'
);

console.log(`Contact created with ID: ${result.data.id}`);
```

## Example: Create Salesforce Account
```javascript
async function createSalesforceAccount(accountData, linkedAccountId, apiKey) {
  const result = await executeAction(
    'salesforce',
    'create_account',
    {
      Name: accountData.name,
      Industry: accountData.industry,
      Website: accountData.website,
      Phone: accountData.phone,
      BillingCity: accountData.city,
      BillingCountry: accountData.country
    },
    linkedAccountId,
    apiKey
  );

  return {
    id: result.data.id,
    success: result.node_status === 'Success'
  };
}

// Usage
const account = await createSalesforceAccount(
  {
    name: 'Acme Corporation',
    industry: 'Technology',
    website: 'https://acme.com',
    phone: '+1-555-123-4567',
    city: 'San Francisco',
    country: 'USA'
  },
  'user_12345',
  'your_api_key'
);

console.log(`Salesforce Account ID: ${account.id}`);
```

## Example: Execute with Error Handling
```javascript
async function safeExecuteAction(slug, actionId, params, linkedAccountId, apiKey) {
  try {
    const result = await executeAction(slug, actionId, params, linkedAccountId, apiKey);

    return {
      success: true,
      data: result.data,
      httpStatus: result.http_status
    };
  } catch (error) {
    return {
      success: false,
      error: error.message,
      params: params // Include params for debugging
    };
  }
}

// Usage: Batch create contacts with error handling
async function batchCreateContacts(contacts, linkedAccountId, apiKey) {
  const results = {
    successful: [],
    failed: []
  };

  for (const contact of contacts) {
    const result = await safeExecuteAction(
      'hubspot',
      'create_contact',
      contact,
      linkedAccountId,
      apiKey
    );

    if (result.success) {
      results.successful.push({
        email: contact.email,
        id: result.data.id
      });
    } else {
      results.failed.push({
        email: contact.email,
        error: result.error
      });
    }
  }

  return results;
}

// Usage
const batchResults = await batchCreateContacts(
  [
    { email: 'john@acme.com', firstname: 'John', lastname: 'Doe' },
    { email: 'jane@acme.com', firstname: 'Jane', lastname: 'Smith' },
    { firstname: 'Missing Email' } // Will fail - missing required email
  ],
  'user_12345',
  'your_api_key'
);

console.log(`Created: ${batchResults.successful.length}`);
console.log(`Failed: ${batchResults.failed.length}`);
```
