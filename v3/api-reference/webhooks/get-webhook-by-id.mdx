---
title: Get Webhook by ID
api: get /api/v2/public/webhook/:id
description: Retrieve details of a specific webhook by its ID.
---

Retrieve the configuration details of a specific webhook subscription by its unique identifier.

## Authentication

<ParamField header="x-api-key" type="string" required>
  Your Refold API key. Find it in **Settings → Developer → Setup**.
</ParamField>

## Path Parameters

<ParamField path="id" type="string" required>
  The unique webhook identifier.
  
  **Example:** `66e5701971ff81574d55b2ba`
</ParamField>

## Response

<Tabs>
  <Tab title="200 OK">
    Returns the webhook object.

    <ResponseField name="_id" type="string">

    </ResponseField>

    <ResponseField name="webhook_url" type="string">

## Example: Verify Webhook Configuration
```javascript
async function getWebhook(webhookId, apiKey) {
  const response = await fetch(
    `https://api.gocobalt.io/api/v2/public/webhook/${webhookId}`,
    {
      headers: {
        'x-api-key': apiKey
      }
    }
  );

  if (!response.ok) {
    const error = await response.json();
    throw new Error(error.message);
  }

  return response.json();
}

// Usage: Verify webhook is properly configured
const webhook = await getWebhook('66e5701971ff81574d55b2ba', 'your_api_key');

const requiredEvents = ['Workflow Errored', 'Connection Expired'];
const missingEvents = requiredEvents.filter(
  event => !webhook.webhook_events.includes(event)
);

if (missingEvents.length > 0) {
  console.log('Missing required events:', missingEvents);
}

if (!webhook.is_enabled) {
  console.log('Warning: Webhook is disabled');
}
```
