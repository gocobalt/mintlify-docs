---
title: List Configs
api: get /api/v2/public/slug/:slug/configs
description: Retrieve all configs for an application across a linked account.
---

Retrieve all configurations installed for a specific application. Use this to see all config instances a linked account has created for an integration.

## Authentication

<ParamField header="x-api-key" type="string" required>
  Your Refold API key. Find it in **Settings → Developer → Setup**.
</ParamField>

<ParamField header="linked_account_id" type="string" required>
  The unique identifier for the linked account.
  
  **Example:** `user_12345`
</ParamField>

## Headers

<ParamField header="disable_field_options" type="boolean">
  When `true`, field options are not returned in the response. Use this for faster responses when you only need config metadata.
  
  **Default:** `false`
</ParamField>

## Path Parameters

<ParamField path="slug" type="string" required>
  The application slug.
  
  **Examples:** `hubspot`, `salesforce`, `slack`
</ParamField>

## Example: List All Configs with Status
```javascript
async function listConfigsWithStatus(linkedAccountId, slug, apiKey) {
  const response = await fetch(
    `https://api.gocobalt.io/api/v2/public/slug/${slug}/configs`,
    {
      headers: {
        'x-api-key': apiKey,
        'linked_account_id': linkedAccountId
      }
    }
  );

  const configs = await response.json();
  
  return configs.map(config => ({
    configId: config.config_id,
    enabledWorkflows: config.workflows
      .filter(w => w.enabled)
      .map(w => w.name),
    totalWorkflows: config.workflows.length,
    hasRequiredFieldsMissing: config.fields
      .some(f => f.required && !f.value)
  }));
}
```
