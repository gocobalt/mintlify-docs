---
title: Upsert Config
api: post /api/v2/public/config
description: Find an existing config or create a new one for a linked account and application.
---

Retrieve an existing configuration or create a new one for a linked account. A config stores the integration settings for each of your end-users, including which workflows are enabled and field values they have configured. If a config with the specified `config_id` exists, it returns the existing setup. Otherwise, it creates a new config.

## Authentication

<ParamField header="x-api-key" type="string" required>
  Your Refold API key. Find it in **Settings → Developer → Setup**.
</ParamField>

<ParamField header="linked_account_id" type="string" required>
  The unique identifier for the linked account.
  
  **Example:** `user_12345`
</ParamField>

## Body Parameters

<ParamField body="slug" type="string" required>
  The application slug for which to create or retrieve the config.
  
  **Examples:** `hubspot`, `salesforce`, `slack`
</ParamField>

<ParamField body="config_id" type="string">
  A unique identifier for this config. If not provided, defaults to the `linked_account_id`.
  
  **Example:** `config_hubspot_001`
</ParamField>

<ParamField body="labels" type="object">
  Dynamic labels for field mapping. Pass custom field names that your users can map to integration fields.
  
  <Expandable title="Example Structure">
```json
    {
      "customer_mapping": [
        { "name": "Customer Name", "value": "customer_name" },
        { "name": "Customer Email", "value": "customer_email" }
      ]
    }
```
  </Expandable>
</ParamField>

## Example: Initialize Config After Connection
```javascript
async function initializeConfig(linkedAccountId, slug, apiKey) {
  const response = await fetch('https://api.gocobalt.io/api/v2/public/config', {
    method: 'POST',
    headers: {
      'x-api-key': apiKey,
      'linked_account_id': linkedAccountId,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      slug,
      labels: {
        // Pass your application's field names for mapping
        contact_fields: [
          { name: 'Full Name', value: 'full_name' },
          { name: 'Email Address', value: 'email' },
          { name: 'Phone', value: 'phone_number' }
        ]
      }
    })
  });

  const config = await response.json();
  
  return {
    configId: config.config_id,
    availableWorkflows: config.workflows.map(w => ({
      id: w.id,
      name: w.name,
      enabled: w.enabled,
      hasRequiredFields: w.fields.some(f => f.required)
    })),
    globalFields: config.fields
  };
}
```

## Field Types

| Type | Description | Example Usage |
|------|-------------|---------------|
| `text` | Free-form text input | API keys, custom identifiers |
| `number` | Numeric input | Thresholds, limits |
| `select` | Dropdown selection | Channels, pipelines, stages |
| `map` | Field mapping | Map your fields to integration fields |
