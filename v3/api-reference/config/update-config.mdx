---
title: Update Config
api: put /api/v2/public/config
description: Update field values and enable or disable workflows for a linked account's config.
---

Update an existing configuration for a linked account. Use this endpoint to enable or disable workflows, set field values, and configure field mappings. This is the primary way to activate integrations for your end-users.

## Authentication

<ParamField header="x-api-key" type="string" required>
  Your Refold API key. Find it in **Settings → Developer → Setup**.
</ParamField>

<ParamField header="linked_account_id" type="string" required>
  The unique identifier for the linked account.
  
  **Example:** `user_12345`
</ParamField>

## Body Parameters

<ParamField body="slug" type="string" required>
  The application slug.
  
  **Examples:** `hubspot`, `salesforce`, `slack`
</ParamField>

<ParamField body="config_id" type="string">
  The config identifier. Defaults to `linked_account_id` if not provided.
</ParamField>

<ParamField body="fields" type="object">
  Global field values for the application. Keys are field IDs, values are the selected values.
  
  <Expandable title="Field Value Examples">
    **Static value:**
```json
    {
      "64abea6d941ba774d124b19c": "default_value"
    }
```
    
    **Map field (field mapping):**
```json
    {
      "649ade09c7b80fc6f0a3d1bc": {
        "customer_name": "firstname",
        "customer_email": "email"
      }
    }
```
  </Expandable>
</ParamField>

<ParamField body="workflows" type="array" required>
  Array of workflow configurations.
  
  <Expandable title="Workflow Object">
    <ParamField body="id" type="string" required>
      Workflow ID from the config response
    </ParamField>
    
    <ParamField body="enabled" type="boolean" required>
      Enable or disable the workflow
    </ParamField>
    
    <ParamField body="fields" type="object">
      Workflow-specific field values. Keys are field IDs.
    </ParamField>
  </Expandable>
</ParamField>

## Example: Enable Workflow and Set Fields
```javascript
async function enableWorkflow(linkedAccountId, slug, workflowId, fieldValues, apiKey) {
  const response = await fetch('https://api.gocobalt.io/api/v2/public/config', {
    method: 'PUT',
    headers: {
      'x-api-key': apiKey,
      'linked_account_id': linkedAccountId,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      slug,
      workflows: [
        {
          id: workflowId,
          enabled: true,
          fields: fieldValues
        }
      ]
    })
  });

  const config = await response.json();
  
  const workflow = config.workflows.find(w => w.id === workflowId);
  return {
    success: workflow?.enabled === true,
    config
  };
}

// Usage: Enable Slack workflow with channel selection
await enableWorkflow(
  'user_12345',
  'slack',
  '64d1fac58716dc5065127ffe',
  { '64da0b57c9ae95561bb0a24f': 'C044U7Q074J' },
  'your_api_key'
);
```

## Example: Configure Field Mapping
```javascript
async function configureFieldMapping(linkedAccountId, slug, mappings, apiKey) {
  // mappings: { yourFieldName: integrationFieldValue }
  const response = await fetch('https://api.gocobalt.io/api/v2/public/config', {
    method: 'PUT',
    headers: {
      'x-api-key': apiKey,
      'linked_account_id': linkedAccountId,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      slug,
      fields: {
        // Map field ID to mapping object
        '670fcb9e023ebc1f370c5139': mappings
      },
      workflows: [] // Keep existing workflow states
    })
  });

  return response.json();
}

// Usage: Map your fields to HubSpot fields
await configureFieldMapping(
  'user_12345',
  'hubspot',
  {
    'customer_name': 'firstname',
    'customer_email': 'email',
    'customer_company': 'company'
  },
  'your_api_key'
);
```
