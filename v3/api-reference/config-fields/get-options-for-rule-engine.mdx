---
title: Get Rule Engine Options
api: post /api/v2/public/config/rule-engine/:fieldId
description: Retrieve dynamic options for a Rule Engine field based on current selections.
---

Retrieve the dynamic options for a Rule Engine field. Rule Engine fields allow users to define conditional rules with three columns: LHS (left-hand side), Operator, and RHS (right-hand side). This endpoint returns the available options for the Operator and RHS columns based on the current LHS selection.

<Info>
Rule Engine fields are used for complex conditional logic, such as routing workflows based on user type, region, or other attributes.
</Info>

## Authentication

<ParamField header="x-api-key" type="string" required>
  Your Refold API key. Find it in **Settings → Developer → Setup**.
</ParamField>

<ParamField header="linked_account_id" type="string" required>
  The unique identifier for the linked account.
  
  **Example:** `user_12345`
</ParamField>

<ParamField header="slug" type="string" required>
  The application slug.
  
  **Examples:** `hubspot`, `salesforce`, `workday`
</ParamField>

## Path Parameters

<ParamField path="fieldId" type="string" required>
  The unique identifier of the Rule Engine field.
  
  **Example:** `67231c960f8ac1513443c75`
</ParamField>

## Query Parameters

<ParamField query="workflow_id" type="string">
  Filter to a specific workflow's Rule Engine field.
  
  **Example:** `67fbc1ec954bde0dcae218c8`
</ParamField>

<ParamField query="config_id" type="string">
  The config identifier. Defaults to `linked_account_id` if not provided.
  
  **Example:** `config_workday_001`
</ParamField>

## Example: Build Rule Engine UI
```javascript
async function getRuleEngineOptions(linkedAccountId, slug, fieldId, configId, apiKey) {
  const url = new URL(`https://api.gocobalt.io/api/v2/public/config/rule-engine/${fieldId}`);
  if (configId) {
    url.searchParams.set('config_id', configId);
  }

  const response = await fetch(url, {
    method: 'POST',
    headers: {
      'x-api-key': apiKey,
      'linked_account_id': linkedAccountId,
      'slug': slug
    }
  });

  const result = await response.json();
  
  return {
    rhsColumn: {
      name: result.rule_column.rhs.name,
      type: result.rule_column.rhs.type,
      options: result.rule_column.rhs.options
    },
    operatorColumn: {
      name: result.rule_column.operator.name,
      type: result.rule_column.operator.type,
      options: result.rule_column.operator.options
    },
    debugOutput: result.conditional_code_stdout
  };
}

// Usage: Load rule engine options for a provisioning rule
const ruleOptions = await getRuleEngineOptions(
  'user_12345',
  'workday',
  '67231c960f8ac1513443c75',
  null,
  'your_api_key'
);

// Build dynamic UI based on options
if (ruleOptions.operatorColumn.options.length > 0) {
  renderOperatorDropdown(ruleOptions.operatorColumn.options);
}

if (ruleOptions.rhsColumn.type === 'select') {
  renderRhsDropdown(ruleOptions.rhsColumn.options);
} else {
  renderRhsTextInput();
}
```

## Rule Engine Structure

| Column | Description | Example Values |
|--------|-------------|----------------|
| LHS | The attribute to evaluate | User Type, Region, Role |
| Operator | The comparison operator | Equals, Not Equals, Contains |
| RHS | The value to compare against | Agent, North America, Admin |
