---
title: Trigger Event by Slug
openapi: "POST /api/v2/public/event/{slug}"
description: Fire a custom event, specifying the app slug in the URL path.
---

Fire a custom event for a linked account, with the app slug supplied in the URL path instead of a header. This is an alias for [Trigger Event](/v3/api-reference/events/trigger-event) — same body and behavior — and additionally supports multipart uploads.

<ResponseExample>
```json 200
{
  "message": "Webhook Fired Success"
}
```
```json 404
{
  "status_code": 404,
  "http_error_type": "NOT_FOUND",
  "error_code": "INVALID_EVENT",
  "message": "Trigger not found"
}
```
</ResponseExample>

<RequestExample>
```bash cURL (API key)
curl -X POST "https://app.refold.ai/api/v2/public/event/{slug}" \
  -H "x-api-key: YOUR_API_KEY" \
  -H "linked_account_id: YOUR_LINKED_ACCOUNT_ID" \
  -H "Content-Type: application/json" \
  -d '{
    "event": "candidate_created",
    "payload": { "jobid": "123" }
  }'
```
```bash cURL (session token)
curl -X POST "https://app.refold.ai/api/v2/public/event/{slug}" \
  -H "Authorization: Bearer YOUR_SESSION_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{ "event": "candidate_created", "payload": {} }'
```
</RequestExample>
