---
title: List Datastores
api: get /api/v2/public/datastore/config/:slug
description: Retrieve all config datastores for an application.
---

Retrieve a list of all config datastores available for an application. Config datastores persist data across multiple workflow executions and can be accessed by any workflow within the same application.

<Info>
Config datastores are different from workflow datastores. Config datastores persist across all workflows for an application, while workflow datastores only exist within a single workflow execution.
</Info>

## Authentication

<ParamField header="x-api-key" type="string" required>
  Your Refold API key. Find it in **Settings → Developer → Setup**.
</ParamField>

<ParamField header="linked_account_id" type="string" required>
  The unique identifier for the linked account.
  
  **Example:** `user_12345`
</ParamField>

## Path Parameters

<ParamField path="slug" type="string" required>
  The application slug.
  
  **Examples:** `hubspot`, `salesforce`, `netsuite`
</ParamField>

## Example: List and Display Datastores
```javascript
async function listDatastores(linkedAccountId, slug, apiKey) {
  const response = await fetch(
    `https://api.gocobalt.io/api/v2/public/datastore/config/${slug}`,
    {
      headers: {
        'x-api-key': apiKey,
        'linked_account_id': linkedAccountId
      }
    }
  );

  if (!response.ok) {
    const error = await response.json();
    throw new Error(error.message);
  }

  const datastoreNames = await response.json();
  
  return {
    application: slug,
    datastoreCount: datastoreNames.length,
    datastores: datastoreNames
  };
}

// Usage
const result = await listDatastores('user_12345', 'netsuite', 'your_api_key');
console.log(`Found ${result.datastoreCount} datastores for ${result.application}`);
// Output: Found 4 datastores for netsuite
```

## Datastore Types

| Type | Scope | Persistence |
|------|-------|-------------|
| Config | All workflows in application | Persists until explicitly deleted |
| Workflow | Single workflow execution | Deleted after workflow completes |

<Note>
This API only returns config-type datastores. Workflow-type datastores are temporary and only accessible within their workflow execution context.
</Note>
