---
title: Migrate Token
api: put /api/v2/public/migrate-token
description: Migrate authentication credentials (API keys, access tokens, refresh tokens) from external systems to Refold for a linked account.
---

Migrate existing authentication credentials to Refold when you have tokens or API keys from an external system that need to be used with Refold integrations. This is useful when transitioning from a custom integration to Refold or when you manage OAuth flows externally.

## Authentication

<ParamField header="x-api-key" type="string" required>
  Your Refold API key. Find it in **Settings → Credentials**.
</ParamField>

## Header Parameters

<ParamField header="linked_account_id" type="string" required>
  Unique identifier for the linked account.
  
  **Example:** `user_12345`, `markwood@example.com`
</ParamField>

## Body Parameters

<ParamField body="slug" type="string" required>
  Application slug for the integration.
  
  **Examples:** `slack`, `salesforce`, `hubspot`, `shopify`
</ParamField>

<ParamField body="auth_object" type="object" required>
  Object containing the authentication credentials to migrate. The structure varies based on the application's authentication type.
  
  <Expandable title="OAuth2 Applications">
    <ParamField body="access_token" type="string">
      OAuth2 access token
    </ParamField>
    <ParamField body="refresh_token" type="string">
      OAuth2 refresh token (recommended for long-term access)
    </ParamField>
  </Expandable>
  
  <Expandable title="Key-Based Applications">
    <ParamField body="api_key" type="string">
      API key for the application
    </ParamField>
    <ParamField body="server_url" type="string">
      Server URL or instance URL (if required)
    </ParamField>
  </Expandable>
  
  <Expandable title="Slack">
    <ParamField body="bot_token" type="string">
      Slack bot token (xoxb-...)
    </ParamField>
    <ParamField body="user_token" type="string">
      Slack user token (xoxp-...)
    </ParamField>
  </Expandable>
  
  <Expandable title="Shopify">
    <ParamField body="access_token" type="string">
      Shopify access token
    </ParamField>
    <ParamField body="store_name" type="string">
      Shopify store name
    </ParamField>
  </Expandable>
</ParamField>

## Code Examples

<AccordionGroup>
  <Accordion title="Migrate OAuth2 Token">
```javascript
    async function migrateOAuth2Token(apiKey, linkedAccountId, slug, tokens) {
      const response = await fetch(
        'https://api.gocobalt.io/api/v2/public/migrate-token',
        {
          method: 'PUT',
          headers: {
            'Content-Type': 'application/json',
            'x-api-key': apiKey,
            'linked_account_id': linkedAccountId
          },
          body: JSON.stringify({
            slug,
            auth_object: {
              access_token: tokens.accessToken,
              refresh_token: tokens.refreshToken
            }
          })
        }
      );

      if (!response.ok) {
        const error = await response.json();
        throw new Error(error.message);
      }

      return response.json();
    }

    // Usage
    const result = await migrateOAuth2Token(
      'your_api_key',
      'user_12345',
      'salesforce',
      { accessToken: '00D5g00000XXXXX!...', refreshToken: '5Aep861xxx...' }
    );
    console.log(result.message); // "Integration - salesforce created"
```
  </Accordion>

## Auth Object Reference

| Auth Type | Required Fields | Example Applications |
|-----------|-----------------|---------------------|
| **OAuth2** | `access_token`, `refresh_token` | Salesforce, HubSpot, Google |
| **Key-Based** | `api_key`, `server_url` | Twilio, SendGrid, Stripe |
| **Slack** | `bot_token`, `user_token` | Slack |
| **Shopify** | `access_token`, `store_name` | Shopify |

<Warning>
For OAuth2 applications, only `access_token` and `refresh_token` are permitted. For key-based applications, only the fields specific to that application's auth type are allowed.
</Warning>
