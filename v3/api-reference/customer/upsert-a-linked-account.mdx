---
title: Upsert Linked Account
api: put /api/v2/public/linked-account
description: Create a new linked account or update an existing one.
---

Create a new linked account or update an existing one. If an account with the provided `linked_account_id` exists, it will be updated. Otherwise, a new account will be created.

This is the recommended method for managing linked accounts as it handles both creation and updates in a single call.

## Authentication

<ParamField header="x-api-key" type="string" required>
  Your Refold API key. Find it in **Settings → Credentials**.
</ParamField>

## Body Parameters

<ParamField body="linked_account_id" type="string" required>
  Unique identifier for the linked account. Typically matches an ID from your internal system.
  
  **Example:** `user_12345`, `account:98765`, `markwood@example.com`
</ParamField>

<ParamField body="name" type="string">
  Display name for the linked account.
  
  **Example:** `Acme Corporation`
</ParamField>

<ParamField body="your_app" type="object">
  Credentials for your application to receive data from Refold workflows.
  
  <Warning>
  If you include `your_app`, you must also include `auth_credentials`. An empty `your_app` object causes an error.
  </Warning>
  
  <Expandable title="properties">
    <ResponseField name="auth_credentials" type="object">
      Key-value pairs of credentials for authenticating with your APIs.
      <Expandable title="example">
        <ResponseField name="x-api-key" type="string">
          Your API key
        </ResponseField>
        <ResponseField name="base-url" type="string">
          Your API base URL
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ParamField>

<ParamField body="udf" type="object">
  User-defined fields to store custom data with the linked account. Useful for access control and workflow filtering.
  
  **Example:** `{ "plan": "enterprise", "region": "us-west", "role": "admin" }`
</ParamField>

## Example: Upsert Linked Account
```javascript
async function upsertLinkedAccount(accountId, data, apiKey) {
  const response = await fetch(
    'https://api.gocobalt.io/api/v2/public/linked-account',
    {
      method: 'PUT',
      headers: {
        'x-api-key': apiKey,
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({
        linked_account_id: accountId,
        ...data
      })
    }
  );

  if (!response.ok) {
    const error = await response.json();
    throw new Error(error.message);
  }

  return response.json();
}

// Usage: Create or update account
const account = await upsertLinkedAccount(
  'user_12345',
  {
    name: 'Acme Corporation',
    udf: {
      plan: 'enterprise',
      region: 'us-west',
      role: 'admin'
    }
  },
  'your_api_key'
);

console.log(`Account ${account.account_id} saved`);
```

## Example: Set Up Linked Account with Your App Credentials
```javascript
async function setupLinkedAccountWithCredentials(accountId, name, yourAppCredentials, apiKey) {
  const response = await fetch(
    'https://api.gocobalt.io/api/v2/public/linked-account',
    {
      method: 'PUT',
      headers: {
        'x-api-key': apiKey,
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({
        linked_account_id: accountId,
        name: name,
        your_app: {
          auth_credentials: yourAppCredentials
        }
      })
    }
  );

  return response.json();
}

// Usage: Set up account with your app credentials
const account = await setupLinkedAccountWithCredentials(
  'user_12345',
  'Acme Corporation',
  {
    'Authorization': 'Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...',
    'x-api-key': 'acme_api_key_here'
  },
  'your_api_key'
);
```
