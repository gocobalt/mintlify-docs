---
title: Get Auth Token
api: get /api/v2/public/token/:linked_account_id/:integration_type
description: Get refreshed, decrypted auth credentials for an integration.
---

Return the decrypted auth-token keys for an integration on a linked account. For OAuth integrations, the token is refreshed first via the app's update-auth routine, so you always get a usable access token. Only the keys originally stored for the integration are returned.

This endpoint authenticates with a **session token** (Bearer), making it suitable for hosted frontends.

<Warning>
  The response contains live secrets. Never log it or expose it to end users — use it server-side only.
</Warning>

## Authentication

<ParamField header="Authorization" type="string" required>
  Session token for the linked account.

  **Format:** `Bearer YOUR_SESSION_TOKEN`
</ParamField>

## Path Parameters

<ParamField path="linked_account_id" type="string" required>
  Your unique identifier for the account.

  **Example:** `user_12345`
</ParamField>

<ParamField path="integration_type" type="string" required>
  The integration type / app slug.

  **Example:** `hubspot`
</ParamField>

## Response

<Tabs>
  <Tab title="200 OK">
    <ResponseField name="auth_credentials" type="object">
      The decrypted token keys for the integration. Keys vary by app and auth type.
      <Expandable title="properties (OAuth example)">
        <ResponseField name="access_token" type="string">Current (refreshed) access token.</ResponseField>
        <ResponseField name="refresh_token" type="string">Refresh token, when the app issues one.</ResponseField>
      </Expandable>
    </ResponseField>
  </Tab>

  <Tab title="401 Unauthorized">
    <ResponseField name="status_code" type="integer">HTTP status code.</ResponseField>
    <ResponseField name="http_error_type" type="string">Error category.</ResponseField>
    <ResponseField name="error" type="string">Internal error code, e.g. `INVALID_TOKEN`.</ResponseField>
    <ResponseField name="message" type="string">Human-readable error description.</ResponseField>
  </Tab>
</Tabs>

<ResponseExample>
```json 200
{
  "auth_credentials": {
    "access_token": "CN...REDACTED_ACCESS_TOKEN...A8",
    "refresh_token": "REDACTED-REFRESH-TOKEN"
  }
}
```
```json 401
{
  "status_code": 401,
  "http_error_type": "UNAUTHORIZED",
  "error": "INVALID_TOKEN",
  "message": "Invalid token"
}
```
</ResponseExample>

<RequestExample>
```bash cURL
curl -X GET "https://app.refold.ai/api/v2/public/token/{linked_account_id}/{integration_type}" \
  -H "Authorization: Bearer YOUR_SESSION_TOKEN"
```
</RequestExample>
