---
title: Register Schedule
api: post /api/v1/workflow/public/scheduled
description: Create a scheduled trigger for a workflow using cron expressions or specific date/time intervals.
---

Register a scheduled trigger to execute a workflow automatically at specified times. Supports both cron expressions for recurring schedules and specific date/time intervals for precise execution.

## Authentication

<ParamField header="x-api-key" type="string" required>
  Your Refold API key. Find it in **Settings → Credentials**.
</ParamField>

<ParamField header="linked_account_id" type="string" required>
  The unique identifier for the linked account.
</ParamField>

<ParamField header="slug" type="string" required>
  Application slug for the integration.
  
  **Example:** `hubspot`, `salesforce`, `slack`
</ParamField>

<ParamField header="workflow_id_alias" type="string" required>
  Workflow ID or the workflow alias to schedule.
</ParamField>

## Body Parameters

<ParamField body="trigger_type" type="string" required>
  The type of schedule trigger.
  
  - `cron` - Use a cron expression for recurring schedules
  - `interval` - Use specific date/time values for precise execution
</ParamField>

<ParamField body="payload" type="object" required>
  Event payload to send during workflow execution. Pass an empty object `{}` if no data is needed.
</ParamField>

### Cron Trigger Parameters

<Info>
Required when `trigger_type` is `cron`.
</Info>

<ParamField body="regex" type="string">
  Cron expression defining the schedule pattern.
  
  **Examples:**
  - `*/30 * * * *` - Every 30 minutes
  - `0 9 * * *` - Daily at 9:00 AM
  - `0 0 * * 1` - Every Monday at midnight
  - `*/6 * * * *` - Every 6 minutes
</ParamField>

<ParamField body="currentDate" type="string">
  Current date in `YYYY-MM-DD` format.
  
  **Example:** `2025-01-15`
</ParamField>

<ParamField body="startDate" type="string">
  Start date for the cron job in `YYYY-MM-DD` format.
  
  **Example:** `2025-01-16`
</ParamField>

<ParamField body="endDate" type="string">
  End date for the cron job in `YYYY-MM-DD` format.
  
  **Example:** `2025-01-31`
</ParamField>

<ParamField body="tz" type="string">
  Timezone for the cron job.
  
  **Example:** `Asia/Kolkata`, `America/New_York`, `Europe/London`, `UTC`
</ParamField>

### Interval Trigger Parameters

<Info>
Required when `trigger_type` is `interval`.
</Info>

<ParamField body="interval_dates" type="array">
  Array of specific date/time values for execution.
  <Expandable title="Interval Date Object">
    <ParamField body="date" type="object" required>
      Date and time specification
      <Expandable title="Date Fields">
        <ParamField body="day" type="integer" required>
          Day of month (1-31). **Example:** `25`
        </ParamField>
        <ParamField body="month" type="integer" required>
          Month (1-12). **Example:** `5`
        </ParamField>
        <ParamField body="year" type="integer" required>
          Year. **Example:** `2025`
        </ParamField>
        <ParamField body="hour" type="integer" required>
          Hour in 24-hour format (0-23). **Example:** `21`
        </ParamField>
        <ParamField body="minute" type="integer" required>
          Minute (0-59). **Example:** `45`
        </ParamField>
        <ParamField body="second" type="integer" required>
          Second (0-59). **Example:** `30`
        </ParamField>
      </Expandable>
    </ParamField>
    <ParamField body="tz" type="string" required>
      Timezone for this specific date. **Example:** `Asia/Kolkata`
    </ParamField>
  </Expandable>
</ParamField>

## Example: Register Cron Schedule
```javascript
async function registerCronSchedule(apiKey, options) {
  const {
    linkedAccountId,
    slug,
    workflowIdOrAlias,
    cronExpression,
    startDate,
    endDate,
    timezone,
    payload = {}
  } = options;

  const today = new Date().toISOString().split('T')[0];

  const response = await fetch(
    'https://api.gocobalt.io/api/v1/workflow/public/scheduled',
    {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'x-api-key': apiKey,
        'linked_account_id': linkedAccountId,
        'slug': slug,
        'workflow_id_alias': workflowIdOrAlias
      },
      body: JSON.stringify({
        trigger_type: 'cron',
        payload,
        regex: cronExpression,
        currentDate: today,
        startDate,
        endDate,
        tz: timezone
      })
    }
  );

  if (!response.ok) {
    const error = await response.json();
    throw new Error(error.message);
  }

  return response.json();
}

// Usage: Schedule workflow to run every 30 minutes
const schedule = await registerCronSchedule('your_api_key', {
  linkedAccountId: 'user_12345',
  slug: 'hubspot',
  workflowIdOrAlias: 'sync-contacts',
  cronExpression: '*/30 * * * *',
  startDate: '2025-01-16',
  endDate: '2025-01-31',
  timezone: 'America/New_York',
  payload: { source: 'scheduled_sync' }
});

console.log(`Schedule created: ${schedule._id}`);
```

## Example: Register Interval Schedule
```javascript
async function registerIntervalSchedule(apiKey, options) {
  const {
    linkedAccountId,
    slug,
    workflowIdOrAlias,
    dates,
    timezone,
    payload = {}
  } = options;

  const intervalDates = dates.map(d => ({
    date: {
      day: d.getDate(),
      month: d.getMonth() + 1,
      year: d.getFullYear(),
      hour: d.getHours(),
      minute: d.getMinutes(),
      second: d.getSeconds()
    },
    tz: timezone
  }));

  const response = await fetch(
    'https://api.gocobalt.io/api/v1/workflow/public/scheduled',
    {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'x-api-key': apiKey,
        'linked_account_id': linkedAccountId,
        'slug': slug,
        'workflow_id_alias': workflowIdOrAlias
      },
      body: JSON.stringify({
        trigger_type: 'interval',
        payload,
        interval_dates: intervalDates
      })
    }
  );

  if (!response.ok) {
    const error = await response.json();
    throw new Error(error.message);
  }

  return response.json();
}

// Usage: Schedule workflow for specific dates
const schedule = await registerIntervalSchedule('your_api_key', {
  linkedAccountId: 'user_12345',
  slug: 'salesforce',
  workflowIdOrAlias: 'quarterly-report',
  dates: [
    new Date('2025-03-31T17:00:00'),
    new Date('2025-06-30T17:00:00'),
    new Date('2025-09-30T17:00:00'),
    new Date('2025-12-31T17:00:00')
  ],
  timezone: 'America/New_York',
  payload: { report_type: 'quarterly' }
});

console.log(`Schedule created with ${schedule.trigger_interval.length} execution dates`);
```

## Cron Expression Reference

| Expression | Description |
|------------|-------------|
| `* * * * *` | Every minute |
| `*/5 * * * *` | Every 5 minutes |
| `*/30 * * * *` | Every 30 minutes |
| `0 * * * *` | Every hour (at minute 0) |
| `0 */2 * * *` | Every 2 hours |
| `0 9 * * *` | Daily at 9:00 AM |
| `0 9,17 * * *` | Daily at 9:00 AM and 5:00 PM |
| `0 0 * * *` | Daily at midnight |
| `0 0 * * 1` | Every Monday at midnight |
| `0 0 1 * *` | First day of every month |
| `0 0 1 1 *` | January 1st at midnight |
