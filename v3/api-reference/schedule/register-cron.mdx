---
title: Register Schedule
api: post /api/v1/workflow/public/scheduled
description: Create a scheduled trigger to execute workflows at specified times using cron expressions or specific date intervals.
---

Register a new scheduled trigger to execute workflows automatically. You can create recurring schedules using cron expressions or specify exact dates and times for execution.

## Authentication

<ParamField header="x-api-key" type="string" required>
  Your Refold API key. Find it in **Settings → Credentials**.
</ParamField>

## Header Parameters

<ParamField header="linked_account_id" type="string" required>
  The unique identifier for the linked account that will execute the scheduled workflow.
  
  **Example:** `user_12345`
</ParamField>

<ParamField header="slug" type="string" required>
  The application slug for the integration.
  
  **Example:** `salesforce`, `hubspot`, `netsuite`
</ParamField>

<ParamField header="workflow_id_alias" type="string" required>
  The workflow ID alias or identifier to execute on schedule.
  
  **Example:** `sync_contacts_workflow`
</ParamField>

<ParamField header="config_id" type="string" required>
  The configuration ID associated with the linked account for this integration.
  
  **Example:** `config_abc123`
</ParamField>

## Body Parameters

<ParamField body="trigger_type" type="string" required>
  The type of schedule trigger.
  
  **Allowed values:** `cron`, `interval`
  
  - `cron`: Use cron expressions for recurring schedules
  - `interval`: Use specific date/time values for precise execution
</ParamField>

### Cron Trigger Parameters

When `trigger_type` is `cron`:

<ParamField body="regex" type="string" required>
  Cron expression defining the schedule pattern.
  
  **Format:** `minute hour day month weekday`
  
  **Examples:**
  - `*/30 * * * *` - Every 30 minutes
  - `0 9 * * *` - Daily at 9:00 AM
  - `0 0 * * 1` - Every Monday at midnight
</ParamField>

<ParamField body="currentDate" type="string">
  Current date for schedule calculation in `YYYY-MM-DD` format.
  
  **Example:** `2025-01-30`
</ParamField>

<ParamField body="startDate" type="string">
  Start date for the schedule in `YYYY-MM-DD` format. Schedule will not execute before this date.
  
  **Example:** `2025-02-01`
</ParamField>

<ParamField body="endDate" type="string">
  End date for the schedule in `YYYY-MM-DD` format. Schedule will not execute after this date.
  
  **Example:** `2025-12-31`
</ParamField>

<ParamField body="tz" type="string">
  Timezone for the schedule. Uses IANA timezone format.
  
  **Examples:** `Asia/Kolkata`, `America/New_York`, `Europe/London`, `UTC`
  
  **Default:** `UTC`
</ParamField>

<ParamField body="payload" type="object">
  Event payload to pass to the workflow when it executes.
  
  **Example:** `{ "sync_type": "full", "batch_size": 100 }`
</ParamField>

### Interval Trigger Parameters

When `trigger_type` is `interval`:

<ParamField body="interval_dates" type="array" required>
  Array of specific date/time objects for execution.
  <Expandable title="Interval Date Object">
    <ParamField body="day" type="integer" required>
      Day of the month (1-31)
    </ParamField>
    <ParamField body="month" type="integer" required>
      Month (1-12)
    </ParamField>
    <ParamField body="year" type="integer" required>
      Year (e.g., 2025)
    </ParamField>
    <ParamField body="hour" type="integer" required>
      Hour (0-23)
    </ParamField>
    <ParamField body="minute" type="integer" required>
      Minute (0-59)
    </ParamField>
    <ParamField body="second" type="integer">
      Second (0-59). Default: 0
    </ParamField>
    <ParamField body="tz" type="string">
      Timezone for this specific date. Default: UTC
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="payload" type="object">
  Event payload to pass to the workflow when it executes.
</ParamField>

## Example: Register Cron Schedule
```javascript
async function registerCronSchedule(apiKey, options) {
  const response = await fetch(
    'https://api.gocobalt.io/api/v1/workflow/public/scheduled',
    {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'x-api-key': apiKey,
        'linked_account_id': options.linkedAccountId,
        'slug': options.slug,
        'workflow_id_alias': options.workflowIdAlias,
        'config_id': options.configId
      },
      body: JSON.stringify({
        trigger_type: 'cron',
        regex: options.cronExpression,
        tz: options.timezone || 'UTC',
        startDate: options.startDate,
        endDate: options.endDate,
        payload: options.payload || {}
      })
    }
  );

  if (!response.ok) {
    const error = await response.json();
    throw new Error(error.message);
  }

  return response.json();
}

// Usage: Schedule workflow to run every 30 minutes
const schedule = await registerCronSchedule('your_api_key', {
  linkedAccountId: 'user_12345',
  slug: 'salesforce',
  workflowIdAlias: 'sync_contacts_workflow',
  configId: 'config_abc123',
  cronExpression: '*/30 * * * *',
  timezone: 'America/New_York',
  startDate: '2025-02-01',
  endDate: '2025-12-31',
  payload: {
    sync_type: 'incremental',
    batch_size: 100
  }
});

console.log(`Schedule created: ${schedule._id}`);
console.log(`Cron pattern: ${schedule.cron_patten}`);
```

## Example: Register Interval Schedule
```javascript
async function registerIntervalSchedule(apiKey, options) {
  const response = await fetch(
    'https://api.gocobalt.io/api/v1/workflow/public/scheduled',
    {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'x-api-key': apiKey,
        'linked_account_id': options.linkedAccountId,
        'slug': options.slug,
        'workflow_id_alias': options.workflowIdAlias,
        'config_id': options.configId
      },
      body: JSON.stringify({
        trigger_type: 'interval',
        interval_dates: options.dates,
        payload: options.payload || {}
      })
    }
  );

  if (!response.ok) {
    const error = await response.json();
    throw new Error(error.message);
  }

  return response.json();
}

// Usage: Schedule workflow for specific quarterly dates
const schedule = await registerIntervalSchedule('your_api_key', {
  linkedAccountId: 'user_12345',
  slug: 'netsuite',
  workflowIdAlias: 'quarterly_report',
  configId: 'config_xyz789',
  dates: [
    { day: 1, month: 4, year: 2025, hour: 9, minute: 0, second: 0, tz: 'America/New_York' },
    { day: 1, month: 7, year: 2025, hour: 9, minute: 0, second: 0, tz: 'America/New_York' },
    { day: 1, month: 10, year: 2025, hour: 9, minute: 0, second: 0, tz: 'America/New_York' },
    { day: 1, month: 1, year: 2026, hour: 9, minute: 0, second: 0, tz: 'America/New_York' }
  ],
  payload: {
    report_type: 'quarterly_summary'
  }
});

console.log(`Schedule created: ${schedule._id}`);
console.log(`Execution dates: ${schedule.trigger_interval.length}`);
```

## Example: Convert JavaScript Date to Interval Format
```javascript
function dateToIntervalFormat(date, timezone = 'UTC') {
  return {
    day: date.getDate(),
    month: date.getMonth() + 1, // JavaScript months are 0-indexed
    year: date.getFullYear(),
    hour: date.getHours(),
    minute: date.getMinutes(),
    second: date.getSeconds(),
    tz: timezone
  };
}

// Usage
const executionDate = new Date('2025-03-15T14:30:00');
const intervalDate = dateToIntervalFormat(executionDate, 'America/Los_Angeles');

console.log(intervalDate);
// { day: 15, month: 3, year: 2025, hour: 14, minute: 30, second: 0, tz: 'America/Los_Angeles' }
```

## Cron Expression Reference

| Expression | Description |
|------------|-------------|
| `* * * * *` | Every minute |
| `*/5 * * * *` | Every 5 minutes |
| `*/15 * * * *` | Every 15 minutes |
| `*/30 * * * *` | Every 30 minutes |
| `0 * * * *` | Every hour (at minute 0) |
| `0 */2 * * *` | Every 2 hours |
| `0 9 * * *` | Daily at 9:00 AM |
| `0 9,18 * * *` | Daily at 9:00 AM and 6:00 PM |
| `0 0 * * *` | Daily at midnight |
| `0 0 * * 1` | Every Monday at midnight |
| `0 0 * * 1-5` | Every weekday at midnight |
| `0 0 1 * *` | First day of every month at midnight |
| `0 0 1 1 *` | January 1st at midnight (yearly) |

<Info>
Cron format: `minute (0-59) hour (0-23) day (1-31) month (1-12) weekday (0-7, where 0 and 7 are Sunday)`
</Info>
