The Refold API is a REST API for integrating and automating the Refold platform from your own backend. Use it to create and manage linked accounts, trigger and inspect workflow executions, subscribe to events and webhooks, and work with every resource you configure in the dashboard.

This page covers the three things you need before your first call — the base URL, authentication, and rate limits — followed by the resources the API exposes.

## Base URL

Every request goes to a base URL that depends on where your Refold instance runs. Prepend it to the endpoint paths shown throughout this reference.

| Deployment | Base URL |
|------------|----------|
| **Refold Cloud** (current) | `https://app.refold.ai` |
| **Refold Cloud** (legacy) | `https://api.gocobalt.io` |
| **Self-hosted** | `https://{identifier}.refold.ai` |

For example, to create a linked account on Refold Cloud:

```bash cURL
curl --request POST \
  --url https://app.refold.ai/api/v2/public/linked-account \
  --header 'x-api-key: YOUR_API_KEY' \
  --header 'Content-Type: application/json'
```

<Note>
The legacy base URL (`https://api.gocobalt.io`) is still supported for existing customers who haven't migrated. New integrations should use `https://app.refold.ai`.
</Note>

<Note>
On a **self-hosted** deployment, replace `{identifier}` with your deployment identifier — for example, `https://acme.refold.ai`. Your platform administrator provides this value.
</Note>

## Authentication

Authenticate every request with an API key, or use a session token when you need to call the API from frontend code without exposing the key.

### API key

Pass your API key in the `x-api-key` header. Find it in the dashboard:

<Steps>
  <Step title="Open Developer settings">
    Log in to your [Refold account](https://app.refold.ai), then go to
    **Settings > Developer** in the side menu.
  </Step>
  <Step title="Copy the key">
    Under the **Setup** tab, find the **API Key** in the **Overview** section.
    Copy the key for either the **Test** or **Production** environment.
  </Step>
</Steps>

<Warning>
Keep your API key on the server. Anyone with it can act on your account, so never embed it in frontend or client-side code.
</Warning>

### Session token

To call the API from the frontend without exposing your API key, generate a session token and send it in the header in place of `x-api-key` and `Linked-account-id`. Session tokens authenticate these calls:

- Config APIs
- Config Field APIs

See [Generate a session token for a linked account](/v3/api-reference/session-token/generate-token-for-linked-account) for how to create one.

## Rate limits

Refold applies rate limits to keep usage efficient and fair across customers.

| Limit | Value |
|-------|-------|
| **Standard** | 20 events per second |
| **Burst** | 5 events in an instant (within a second) |

## API resources

Each resource maps to a group of endpoints in this reference.

| Resource | Description |
|----------|-------------|
| **Linked Account** | Your users' end customers, who use the integrations built on Refold |
| **Applications** | The integrations a user has enabled in Refold |
| **Config** | The configurations created for each integration |
| **Config Field** | The individual fields within a configuration |
| **Event** | Events created in Refold that trigger workflows |
| **Webhooks** | Real-time notifications you subscribe to when events occur |
| **Executions** | Details of a workflow execution |
| **Session Token** | Tokens that authenticate a single linked account |
| **Refold Hosted** | Hosted portal URLs and properties saved against a session token |
| **Public Workflows** | Public workflows created in Refold |
| **Datastores** | Datastores created in Refold |
