Error-prone steps are wrapped in a try/catch and routed to a **dead letter queue (DLQ)** node that collects every failed record. A reprocessing workflow can then re-run those failures, so even data that failed the first time still reaches the target system. For how failures are captured inside a workflow, see the [Dead Letter Queue node](/v3/workflows/nodes/dead-letter-queue).
