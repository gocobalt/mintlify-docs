---
title: "Build a Salesforce integration"
description: "Connect a customer's Salesforce org, read records with SOQL, and write records back, end to end."
---

By the end of this recipe you will have a customer's Salesforce org connected through Refold, opportunities read out of it, and a contact written back into it, all from your backend.

This recipe assumes you have run the [Quickstart](/v3/embedded/quickstart). It reuses that flow (create a linked account, connect the app, execute an action) and shows only what is specific to Salesforce. Salesforce is the connector slug `salesforce` and authenticates with OAuth 2.0.

## Prerequisites

- **A Refold account.** Log in at [app.refold.ai](https://app.refold.ai).
- **The Quickstart loop, once**, so the connect-and-execute pattern is familiar.
- **A Salesforce OAuth app**, if you want customers to use your own client credentials. You can also test with Refold's shared credentials.

## Step 1: Configure the Salesforce app

Tell Refold how to authenticate with Salesforce before a customer connects.

<Steps>
  <Step title="Open the app settings">
    In the [Refold Console](https://app.refold.ai), go to **Apps > Salesforce > Settings**.
  </Step>
  <Step title="Set the auth method">
    Choose **OAuth 2.0** and add your Salesforce **Client ID** and **Client Secret**, or select **Use our credentials** to test with Refold's. Click **Save**.
  </Step>
</Steps>

<Frame>{/* SCREENSHOT: Salesforce app Settings tab showing OAuth client ID/secret fields -> /images/embedded/recipes/salesforce-settings.png */}</Frame>

## Step 2: Connect the customer's Salesforce

Create a [linked account](/v3/embedded/backend/linked-accounts) for the customer and generate a [Connect portal](/v3/embedded/frontend/overview) URL.

```bash cURL
curl -X POST https://app.refold.ai/api/v2/public/connect-url \
  -H "x-api-key: $REFOLD_API_KEY" \
  -H "linked_account_id: demo-user-1" \
  -H "Content-Type: application/json" \
  -d '{ "name": "Demo User" }'
```

```json Output
{ "hosted_url": "https://connect.refold.ai/eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." }
```

Open the `hosted_url`, find **Salesforce**, click **Connect**, and complete the OAuth consent. A **Connected** tag appears when it succeeds.

<Frame>{/* SCREENSHOT: Connect portal showing Salesforce Connect button and the Connected state -> /images/embedded/recipes/salesforce-connect.png */}</Frame>

<Tip>
To embed this flow in your own UI instead of redirecting, see [Embed the connect flow](/v3/embedded/frontend/overview).
</Tip>

## Step 3: List the actions Salesforce supports

Confirm the actions you need before calling them.

```bash cURL
curl https://app.refold.ai/api/v2/integration-schema/salesforce/actions \
  -H "x-api-key: $REFOLD_API_KEY" \
  -H "linked_account_id: demo-user-1"
```

The response lists every action and its input fields. Use the exact action and field names it returns below.

## Step 4: Read opportunities with SOQL

Salesforce supports a SOQL query action, so you can read any object with a single call. Execute it to pull opportunities out of the customer's org.

```bash cURL
curl -X POST https://app.refold.ai/api/v2/integration-schema/salesforce/actions/query/execute \
  -H "x-api-key: $REFOLD_API_KEY" \
  -H "linked_account_id: demo-user-1" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "SELECT Id, Name, Amount, StageName, CloseDate FROM Opportunity"
  }'
```

```json Output
{
  "success": true,
  "data": {
    "records": [
      { "Id": "0065g00000XyZ", "Name": "Acme renewal", "Amount": 12000, "StageName": "Negotiation", "CloseDate": "2026-07-01" }
    ]
  }
}
```

<Warning>
The action name `query` and the SOQL field name are placeholders. Confirm the exact SOQL action and its field name from the Step 3 response before you ship.
</Warning>

<Tip>
To reshape these records into your own schema before storing them, see [Data mapping](/v3/embedded/move-data/data-mapping).
</Tip>

## Step 5: Write a contact back to Salesforce

Now write a record into the customer's org. Refold uses the stored OAuth token automatically.

```bash cURL
curl -X POST https://app.refold.ai/api/v2/integration-schema/salesforce/actions/create_contact/execute \
  -H "x-api-key: $REFOLD_API_KEY" \
  -H "linked_account_id: demo-user-1" \
  -H "Content-Type: application/json" \
  -d '{
    "LastName": "Doe",
    "FirstName": "Jane",
    "Email": "jane@example.com"
  }'
```

```json Output
{
  "success": true,
  "data": { "id": "0035g00000AbCdE" }
}
```

<Check>
Open the customer's Salesforce. The contact **Jane Doe** is there, and your read returned their opportunities. You just ran data both ways through Refold.
</Check>

<Warning>
`create_contact` and its fields are placeholders. Use the real action name and required fields from Step 3, and test against a Salesforce sandbox first.
</Warning>
