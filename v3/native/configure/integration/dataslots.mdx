A **Dataslot** is a field you add to a page to collect a value from your customer. In a connector's **Connect Portal → Fields**, the **Dataslots** tab (next to **Preview**) lists everything you can add: **Static Dataslots** that are always available, and **Dynamic Dataslots** that pull live data from the customer's connected account.

Adding a Dataslot to a page creates a [config field](/v3/native/configure/integration/config-fields) the customer fills in. This page covers how to add a field and the settings every field shares; for each field type in depth, see [Field Types](/v3/native/configure/integration/field-types/overview).

## Add a Dataslot to a page

<Steps>
  <Step title="Open the Fields section">
    Go to the connector's **Connect Portal → Fields**. The left pane lists your [Pages](/v3/native/configure/integration/config-fields#pages) and the fields on each.
  </Step>
  <Step title="Pick the page, then open Dataslots">
    Select the page where the field should live, then open the **Dataslots** tab on the right. The **Preview** tab beside it shows the page exactly as the customer will see it.
  </Step>
  <Step title="Select a Dataslot">
    Click a Dataslot to add it as a field on the chosen page.
  </Step>
  <Step title="Configure it">
    Edit the new field, set its [field settings](#field-settings) and any type-specific options, then save it for customer use.
  </Step>
</Steps>

## Field settings

Every Dataslot shares the same set of settings in the editor. Only the **Options** area changes by type (covered on each [field type](/v3/native/configure/integration/field-types/overview) page).

| Setting | What it controls |
| --- | --- |
| **Name** | The field's label, shown to your customer. |
| **Help Text** | Optional guidance shown beneath the field. |
| **Required** | The customer must supply a value before they can save. |
| **Hidden** | Hides the field from the customer, for values you set programmatically. |
| **Allow Multiple** | Lets the customer pick more than one value. Available on **Select**, **Field Mapping** columns, and **Dynamic Dataslots**. |
| **Use Data Source** | Loads a **Select** or **Field Mapping** column's options live from a published workflow instead of a fixed list. |
| **Default Value** | The value applied when the customer leaves the field empty. Its input matches the type (a date picker for Date, a JSON editor for JSON, and so on). |
| **Cache** | Caches the resolved value so Refold doesn't refetch it on every run, useful for live options that rarely change. |
| **Dependency** | Makes this field depend on another field, its options or visibility update with the parent's value. |
| **Page** | The [Page](/v3/native/configure/integration/config-fields#pages) this field appears on. |

<Note>
**Dependency** powers cascading fields, a child field's options load only after its parent has a value. See [Dependent Fields](/v3/native/configure/integration/field-types/dependent).
</Note>

## Field types

Pick the type that matches the value you're collecting. Each links to its full reference under [Field Types](/v3/native/configure/integration/field-types/overview).

| Field type | Collects | Reference |
| --- | --- | --- |
| **Text · Number · Email · Date · JSON** | Simple typed inputs | [Choose a Field Type](/v3/native/configure/integration/field-types/overview) |
| **Select** | One or many choices from a list (static or live) | [Choose a Field Type](/v3/native/configure/integration/field-types/overview) · live options via [Datasource Fields](/v3/native/configure/integration/field-types/datasource) |
| **Field Mapping** | A mapping between two lists | [Mapping Fields](/v3/native/configure/integration/field-types/mapping) |
| **Rule Engine** | Customer-defined conditional rules | [Rule Engine Fields](/v3/native/configure/integration/field-types/rule-engine) |

### Dynamic Dataslots

**Dynamic Dataslots** are prebuilt slots the Refold team ships per connector, they query the customer's connected account live (for HubSpot, things like **List Deal Pipelines**, **List Pipeline Stages**, or **Map Contact Properties**). Because they're built per connector they aren't always present; when the slot you need is listed, add it directly and it returns options specific to each customer's account.

When no Dynamic Dataslot covers your case, supply the data yourself with a [Datasource](/v3/native/configure/integration/field-types/datasource); to chain fields so each choice narrows the next, use [Dependent Fields](/v3/native/configure/integration/field-types/dependent).
