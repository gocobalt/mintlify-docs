---
title: "Authenticating systems"
description: "Connect the source and destination systems in a delivered integration, with standard flows for most apps and special handling for systems behind a firewall."
---

In an Integration Delivery project, authentication connects the systems your integration moves data between. Unlike a native integration, there is no single embedded UI. A connection can be made wherever it is most convenient: through the [Connect portal](/v3/platform/platform-settings/workspace-settings/connect-portal), a hosted link, or by your delivery team during setup.

For most applications, that connection uses the same flows as every other Refold product. The work specific to delivery projects is handling enterprise systems where the auth path is constrained, most often a system that sits behind a customer's firewall.

## Standard authentication

Most systems in a project authenticate with the shared Refold flows. Use whichever fits how the connection will be made:

## When to use enterprise connectivity

Reach for the [enterprise connectivity](/v3/integration-delivery/authentication/enterprise-connectivity) guidance when a system cannot be reached over the public internet. Examples include an on-premise SAP instance, a self-hosted database, or any app restricted to an allowlisted network. These cases need a connection path established before the standard auth flow can run.
