Go to your MCP server's **Skills** tab and click **+ Add Skill**.

<img src="/images/mcp-skill-create.png" alt="Creating a skill with name, description, app, and content" />

## Skill fields

| Field | Description |
|-------|-------------|
| **Name** | What the agent sees during discovery. Be descriptive (e.g., "Salesforce Lead Qualification Process"). |
| **Summary** | Short description of what the skill does and when to use it. |
| **App** | Which connected application the skill relates to. |
| **Content** | The procedure body: steps, API patterns, conditional logic, input/output specs. |
| **Status** | Enable or disable the skill without deleting it. |

## Writing skill content

Write the content as structured instructions an agent can follow. Skill content typically mixes prose with the concrete API or query patterns the agent should use — SOQL queries, REST endpoints, parameter names — so the agent doesn't have to guess them at runtime.

**Example: Lead qualification**

```markdown
## Goal
Qualify an inbound lead and assign to the right sales rep.

## Pre-checks
- Email address is provided
- Salesforce connection is active

## Steps

1. Check if contact exists in Salesforce.
   Use the SOQL query:
   `SELECT Id, AccountId, OwnerId FROM Contact WHERE Email = :email LIMIT 1`

2. If a contact exists, check for open opportunities on its account:
   `SELECT Id, StageName FROM Opportunity WHERE AccountId = :accountId AND IsClosed = false`

3. If no contact exists, create one via the `create_contact` action with:
   `{ Email, FirstName, LastName, LeadSource: "Webinar" }`

4. Score the lead: company size > 500 employees → "high",
   200–500 → "medium", < 200 → "low".

5. Assign to a sales rep using the `assign_owner` action,
   selecting the rep whose territory matches the contact's country.

## On success
Return the contact ID, qualification score, and assigned rep.

## On failure
If Salesforce is unreachable, return an error asking the user to retry.
```

<Tip>
Write skills the way you'd write a runbook for a junior team member. Clear steps, explicit conditions, no ambiguity.
</Tip>
