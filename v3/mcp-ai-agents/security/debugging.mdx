When an agent doesn't see the tools you expect, or a tool call fails in a way that isn't obvious from the agent's reply, two tools cover most debugging:

1. **MCP Inspector** — Anthropic's official debugger. Points at any MCP server, lists tools, lets you call them with arbitrary input.
2. **MCP Logs** — the audit panel in the Refold dashboard. Shows every tool call your agent actually made and what came back.

## MCP Inspector

[MCP Inspector](https://github.com/modelcontextprotocol/inspector) is a local UI and CLI for talking to any MCP server. Use it to verify Refold is exposing the tools you expect before you wire the URL into your agent.

### Launch the UI

```bash
npx @modelcontextprotocol/inspector
```

The Inspector opens a local web UI. In the server connection pane:

1. Select **Streamable HTTP** from the transport dropdown
2. Paste your Refold Server URL: `https://<domain>/mcp/v1/<token>/<server_id>`
3. Click **Connect**

The Inspector connects, lists the tools your server exposes, and shows the schema for each one.

### Key panels

| Panel | What it shows |
|---|---|
| **Tools** | Every MCP tool the server exposes, with its JSON schema. Click any tool to fill in arguments and run it. |
| **Resources** | MCP resources (Refold doesn't currently expose resources — this panel will be empty) |
| **Prompts** | MCP prompts (same — empty for Refold) |
| **Notifications** | Server logs and the raw JSON-RPC traffic |

### Call a tool

In the **Tools** panel, click the tool you want to test. The Inspector renders an input form from the tool's JSON schema. Fill in the fields, click **Run Tool**, and the response appears inline.

For agent mode servers, the typical test sequence:

1. Run `RESOLVE_ACTIONS` with `integration_query: ["List Contacts Salesforce"]`
2. Copy the returned `slug`, `identifier`, and `json_schema`
3. Run `EXECUTE_ACTION` with `application_slug`, `action_id`, `type`, and an `input_payload` that matches the schema

### CLI mode

For scripting or remote-server smoke tests without the UI:

```bash
# List tools
npx @modelcontextprotocol/inspector --cli \
  https://<domain>/mcp/v1/<token>/<server_id> \
  --transport http \
  --method tools/list

# Call a tool
npx @modelcontextprotocol/inspector --cli \
  https://<domain>/mcp/v1/<token>/<server_id> \
  --transport http \
  --method tools/call \
  --tool-name salesforce_list_contacts_action \
  --tool-arg input_payload='{"limit":10}'
```

## Reading MCP Logs

Once your agent is running against the Refold server, every tool call is captured in **Maintain > MCP Logs**. See [MCP Logs](/v3/mcp-ai-agents/security/mcp-logs) for the full field reference.

### Workflow: an agent call returned nothing useful

1. Open **MCP Logs**, filter by your agent's linked account
2. Find the call that should have produced the result (sort by timestamp)
3. Open the audit detail panel — check the **Status** field:
   - `success` — Refold completed the call. The issue is on the agent's interpretation side. Look at the response payload and compare it to what the agent told the user.
   - `validation_error` — The agent sent an input that didn't match the tool's schema, referenced an app or action that isn't on the server, or hit a known business-rule rejection. The `user_message` field shows what the agent saw.
   - `execution_error` — Refold couldn't reach the workflow service or another internal dependency. The agent sees a generic message; the detailed error is in `internal_error`.
   - `pending` — The call started but never completed (process crashed between log_start and log_complete).

### Workflow: trace a full agent conversation

Filter by **Session ID** to see every tool call from a single agent session in order. This is the fastest way to understand what an agent actually did, especially when it called several tools in sequence (e.g., `RESOLVE_ACTIONS` → `LOAD_SKILL` → `EXECUTE_ACTION` × 3).
