To make a workflow callable by an agent, the workflow itself is built in the Refold workflow builder, and then attached to an MCP server through the **Applications** tab.

## Steps

<Steps>
  <Step title="Build the workflow">
    Build your workflow in the Refold workflow builder with the steps, error handling, retry logic, and validation it needs. Test it independently of MCP first.
  </Step>

  <Step title="Add it to your MCP server">
    Open the MCP server, go to the **Applications** tab, and add the app the workflow belongs to. Select the workflow alongside any actions you want exposed.

    See [Server Configuration](/v3/mcp-ai-agents/overview/server-configuration#applications-tab) for the full Applications tab walkthrough.
  </Step>

  <Step title="The workflow appears as a tool">
    In **direct mode**, the workflow is exposed as `<app_slug>_<workflow_name>_workflow`.

    In **agent mode**, the workflow is reachable through `EXECUTE_ACTION` with `type: "workflow"`. See [Tools Reference](/v3/mcp-ai-agents/tools-reference/tools) for the exact parameter shape.
  </Step>
</Steps>

## Input and output

The agent sees the workflow's input schema as the tool's schema. The agent fills in the inputs; the workflow runs to completion and returns the final result.

If the workflow defines structured outputs, the agent receives them as a structured JSON result. If the workflow defines error paths, those become structured error responses the agent can act on.

## Mixing workflows and actions

You can attach both actions and workflows from the same app on the same server. The agent sees them all as tools. Use actions for one-shot operations and workflows for anything that needs sequencing.
