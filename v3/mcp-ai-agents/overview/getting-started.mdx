## Prerequisites

- A Refold account with at least one [linked account](/v3/developer/guide/build/basics/linked_account) created
- At least one application configured with valid credentials ([connector setup guide](/v3/developer/guide/build/connectors/overview))
- An MCP client installed (Claude Desktop, Cursor, or Windsurf)

## Create your MCP server

<Steps>
  <Step title="Open MCP Servers">
    In the Refold dashboard, go to **Embedded Agents > MCP Servers** and click **+ Create Server**.

    <img src="/images/mcp-create-server.png" alt="MCP Servers list with Create Server dialog" />
  </Step>

  <Step title="Name and describe the server">
    Set a **Name** and **Description**. These are what the agent sees when it connects, so write them for the AI, not for humans. Be specific about what apps and actions are available.

    **Good example:**
    - Name: `Acme CRM Integration`
    - Description: `Access Salesforce and HubSpot data. Create, update, and query contacts, deals, and accounts.`

    **Bad example:**
    - Name: `My Server`
    - Description: `Integration server`
  </Step>

  <Step title="Add applications">
    Go to the **Applications** tab and add the apps you want the agent to access. Each app's actions and workflows become available as tools.

    <img src="/images/mcp-add-application.png" alt="Adding a Workday application with available actions" />
  </Step>

  <Step title="Copy the Server URL">
    Go back to the **Overview** tab and copy the **Server URL**.

    ```
    https://<domain>/mcp/v1/<token>/<server_id>
    ```

    The `token` is tied to a specific linked account. Each of your end users gets their own URL.

    <img src="/images/mcp-server-overview.png" alt="Server Overview tab showing Server URL and toggles" />
  </Step>
</Steps>

## Connect to your MCP client

<Tabs>
  <Tab title="Claude Desktop">
    Open your Claude Desktop config file:

    | OS | Path |
    |----|------|
    | macOS | `~/Library/Application Support/Claude/claude_desktop_config.json` |
    | Windows | `%APPDATA%\Claude\claude_desktop_config.json` |

    Add your server:

    ```json
    {
      "mcpServers": {
        "refold": {
          "url": "https://<domain>/mcp/v1/<your-token>/<server-id>"
        }
      }
    }
    ```

    Or use **Settings > Connectors > Add custom connector** and paste the Server URL directly:

    <img src="/images/mcp-claude-desktop-setup.png" alt="Adding Refold MCP server in Claude Desktop" />

    Restart Claude Desktop. You should see the tools icon in the chat input showing the tools from your server.
  </Tab>

  <Tab title="Cursor">
    Go to **Cursor Settings > MCP** and add a new server, or add it to your project's `.cursor/mcp.json`:

    ```json
    {
      "mcpServers": {
        "refold": {
          "url": "https://<domain>/mcp/v1/<your-token>/<server-id>"
        }
      }
    }
    ```
  </Tab>

  <Tab title="Windsurf">
    Go to **Windsurf Settings > MCP** and add the Server URL as a new remote MCP server.
  </Tab>

  <Tab title="Embed Chat">
    Refold provides an embeddable chat widget. Go to the **Embed Chat** section in your server's Overview tab to get the iframe snippet or Chat URL.

    ```html
    <iframe
      src="<your-embed-chat-url>"
      width="400"
      height="600"
      style="border: none;"
    />
    ```
  </Tab>
</Tabs>

## Test the connection

Try a simple prompt in your MCP client:

```
What tools do you have available?
```

The agent should list the integration actions from your connected apps. Then try an actual operation:

```
List all contacts from Salesforce
```

<Info>
If a connected app returns an auth error, the linked account may not have completed the auth flow for that app. Check your linked account's connection status in the dashboard.
</Info>

## Troubleshooting

| Problem | Likely cause | Fix |
|---------|-------------|-----|
| No tools visible | Server URL is wrong or token is invalid | Re-copy the URL from the dashboard. Make sure you're using the full URL including token and server ID. |
| "Unauthorized" error | Token expired or linked account deleted | Generate a fresh token for the linked account. |
| Action fails with auth error | App credentials missing or expired | Have the end user re-authenticate the app through your auth flow. |
| Tools visible but calls fail | App not properly configured | Check the app's configuration in **Connectors** and verify scopes. |
