title: "Quickstart"

---

An MCP server exposes your customers' connected apps as tools an AI agent can call. This guide walks you through creating a server in the Refold dashboard, connecting an MCP client, and running a first tool call.

## Prerequisites

- A Refold account with at least one [linked account](/v3/platform/concepts/linked-account) created
- At least one connector configured with valid credentials — see [Connector setup](/v3/platform/concepts/connector/supported-apps-actions)
- An MCP client installed: Claude Desktop, Cursor, or Windsurf

<Note>
Each linked account gets its own Server URL. Create one linked account per end user before proceeding.
</Note>

## Create your MCP server

<Steps>
  <Step title="Open MCP Servers">
    In the Refold dashboard, go to **Embedded Agents > MCP Servers** and click **+ Create Server**.

    <Frame>
      <img src="/images/mcp-create-server.png" alt="MCP Servers list with Create Server dialog" />
    </Frame>
  </Step>

  <Step title="Name and describe the server">
    Set a **Name** and **Description**. The AI agent reads these when it connects, so write for the model — be specific about which apps and actions are available.

    | Field | Good | Avoid |
    |-------|------|-------|
    | **Name** | `Acme CRM Integration` | `My Server` |
    | **Description** | `Access Salesforce and HubSpot. Create, update, and query contacts, deals, and accounts.` | `Integration server` |
  </Step>

  <Step title="Add applications">
    Go to the **Applications** tab and add the apps you want the agent to access. Each app's actions and workflows become available as MCP tools.

    <Frame>
      <img src="/images/mcp-add-application.png" alt="Adding a Workday application with available actions" />
    </Frame>
  </Step>

  <Step title="Copy the Server URL">
    Go back to the **Overview** tab and copy the **Server URL**.

    ```
    https://<domain>/mcp/v1/<token>/<server_id>
    ```

    The `<token>` is tied to a specific linked account. Issue a separate URL for each end user.

    <Frame>
      <img src="/images/mcp-server-overview.png" alt="Server Overview tab showing Server URL and toggles" />
    </Frame>
  </Step>
</Steps>

## Connect a client

<Tabs>
  <Tab title="Claude Desktop">
    <Steps>
      <Step title="Open the config file">
        | OS | Path |
        |----|------|
        | macOS | `~/Library/Application Support/Claude/claude_desktop_config.json` |
        | Windows | `%APPDATA%\Claude\claude_desktop_config.json` |
      </Step>

      <Step title="Add your server">
        ```json
        {
          "mcpServers": {
            "refold": {
              "url": "https://<domain>/mcp/v1/<your-token>/<server-id>"
            }
          }
        }
        ```

        Alternatively, go to **Settings > Connectors > Add custom connector** and paste the Server URL directly.

        <Frame>
          <img src="/images/mcp-claude-desktop-setup.png" alt="Adding Refold MCP server in Claude Desktop" />
        </Frame>
      </Step>

      <Step title="Restart Claude Desktop">
        After restarting, you should see the tools icon in the chat input. This confirms your server's tools are loaded.
      </Step>
    </Steps>
  </Tab>

  <Tab title="Cursor">
    <Steps>
      <Step title="Add the server">
        Go to **Cursor Settings > MCP** and add a new server, or add it to your project's `.cursor/mcp.json`:

        ```json
        {
          "mcpServers": {
            "refold": {
              "url": "https://<domain>/mcp/v1/<your-token>/<server-id>"
            }
          }
        }
        ```
      </Step>
    </Steps>
  </Tab>

  <Tab title="Windsurf">
    <Steps>
      <Step title="Add the server">
        Go to **Windsurf Settings > MCP** and add the Server URL as a new remote MCP server.
      </Step>
    </Steps>
  </Tab>

  <Tab title="Embed Chat">
    <Steps>
      <Step title="Get the embed snippet">
        In your server's **Overview** tab, go to the **Embed Chat** section and copy the iframe snippet or Chat URL.

        ```html
        <iframe
          src="<your-embed-chat-url>"
          width="400"
          height="600"
          style="border: none;"
        />
        ```
      </Step>
    </Steps>
  </Tab>
</Tabs>

## Test the connection

In your MCP client, run:

```
What tools do you have available?
```

The agent lists the integration actions from your connected apps. Then try an actual operation:

```
List all contacts from Salesforce
```

<Info>
If an app returns an auth error, the linked account may not have completed the auth flow for that app. Check the linked account's connection status in the dashboard.
</Info>

## Troubleshooting

| Problem | Likely cause | Fix |
|---------|-------------|-----|
| No tools visible | Server URL is wrong or token is invalid | Re-copy the URL from the dashboard. Use the full URL including token and server ID. |
| "Unauthorized" error | Token expired or linked account deleted | Generate a fresh token for the linked account. |
| Action fails with auth error | App credentials missing or expired | Have the end user re-authenticate through your auth flow. |
| Tools visible but calls fail | App not properly configured | Check the app's configuration in **Connectors** and verify scopes. |
