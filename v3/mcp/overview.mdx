The [Model Context Protocol](https://modelcontextprotocol.io) is an open standard for connecting AI agents to external systems and data. Refold MCP lets SaaS platforms expose their integrations to AI agents over MCP.

Two roles share one server:

- **You (the platform team)** configure an MCP server in the Refold dashboard, connect the enterprise apps your customers use, and issue each end user their own URL.
- **The end user's AI agent** connects to that URL and can discover and call integration actions.

Without Refold, connecting agents to your integrations means building and maintaining the integration code, auth, and agent tooling yourself. With Refold, that work is done for you: no custom integration code, no agent framework. Refold handles authentication, execution, tenant isolation, and audit logging.

## What the agent can do

Once connected, an agent can:

- **Call integration actions** directly: create records, query data, update fields across connected apps
- **Run workflows** server-side as one call, with error handling
- **Discover skills** to find pre-built procedures for complex multi-step operations
- **Work across multiple apps** in a single session (for example, read from Salesforce and write to SAP)

Every tool call is authenticated and logged in [MCP Logs](/v3/mcp/operate/mcp-logs). For how the server resolves identity and executes each call, see [How it works](/v3/mcp/how-it-works).

## Modes and layers

A server runs in one of two modes. Skills and workflows then layer onto whichever mode you choose.

| Mode | What the agent sees | Best for |
|------|---------------------|----------|
| **Direct** (default) | One tool per configured action or workflow, called directly | Simple CRUD |
| **Agent** | The server plans which actions to use, then runs them | Multi-step tasks where you want the server to choose the steps |

<Note>Every client is an agent in both modes. "Agent mode" refers to where planning happens, on the server, not whether an agent is involved.</Note>

**Skills** add pre-built procedures the agent can load in either mode. **Workflows** run server-side as one call. See [Choose your pattern](/v3/mcp/choose-your-pattern) for the decision framework.
