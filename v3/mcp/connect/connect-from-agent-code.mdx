A Refold Server URL is everything an agent SDK needs. The URL carries an auth token, so there's no separate OAuth flow or token exchange to build. Paste the URL and the SDK handles the rest.

<Note>
You need a Server URL before you can run these snippets. See the [Quickstart](/v3/mcp/get-started/getting-started) to create your first MCP server and copy its URL, or [Server configuration](/v3/mcp/build/server-configuration) if your server already exists.
</Note>

## Two URL forms

You can pass the token in one of two equivalent ways. Both reach the same linked account, so pick whichever your SDK makes easier.

- **Path-token form** carries the token in the URL path. Most SDKs need nothing else: paste the URL and go.
- **Authorization-header form** carries the token in a request header. Use the shorter URL and send `Authorization: Bearer <token>`.

<CodeGroup>
```text Path-token form
https://<domain>/mcp/v1/<token>/<server_id>
```

```text Authorization header
https://<domain>/mcp/v1/<server_id>
Authorization: Bearer <token>
```
</CodeGroup>

In these examples, `<domain>`, `<token>`, and `<server_id>` are placeholders. When you copy your Server URL from the dashboard, all three are already filled in. The snippets below use the path-token form. Each tab's **Auth** row shows how to switch to the header form.

## Wire the URL into your SDK

<Tabs>
  <Tab title="Anthropic Messages API">
    Use the `mcp_servers` parameter on `messages.create` with the `mcp-client-2025-11-20` beta header. Every server in `mcp_servers` must be referenced by an `mcp_toolset` entry in `tools`.

    ```python Python
    import anthropic

    client = anthropic.Anthropic()

    response = client.beta.messages.create(
        model="claude-opus-4-7",
        max_tokens=1024,
        messages=[{"role": "user", "content": "What tools do you have?"}],
        mcp_servers=[{
            "type": "url",
            "name": "refold",
            "url": "https://<domain>/mcp/v1/<token>/<server_id>",
        }],
        tools=[{"type": "mcp_toolset", "mcp_server_name": "refold"}],
        betas=["mcp-client-2025-11-20"],
    )
    ```

    | Detail | Value |
    |--------|-------|
    | Package | `anthropic` (Python SDK) |
    | Endpoint | `client.beta.messages.create` |
    | Beta header | `anthropic-beta: mcp-client-2025-11-20` |
    | Auth | Path-token URL needs no extra fields. For the header form, set `url` to `https://<domain>/mcp/v1/<server_id>` and add an `authorization_token` field. |

    Source: [Anthropic MCP connector docs](https://platform.claude.com/docs/en/docs/agents-and-tools/mcp-connector).
  </Tab>

  <Tab title="OpenAI Agents SDK">
    Use `MCPServerStreamableHttp` from `agents.mcp`. Pass the server instance via the `mcp_servers` list on `Agent(...)`.

    ```python Python
    import asyncio
    from agents import Agent, Runner
    from agents.mcp import MCPServerStreamableHttp

    async def main():
        async with MCPServerStreamableHttp(
            name="Refold",
            params={"url": "https://<domain>/mcp/v1/<token>/<server_id>"},
        ) as refold:
            agent = Agent(
                name="Assistant",
                instructions="Use Refold tools to answer.",
                mcp_servers=[refold],
            )
            result = await Runner.run(agent, "List my available tools.")
            print(result.final_output)

    asyncio.run(main())
    ```

    | Detail | Value |
    |--------|-------|
    | Package | `openai-agents` (Python) |
    | Class | `MCPServerStreamableHttp` from `agents.mcp` |
    | Transport | Streamable HTTP, auto-selected from `params.url` |
    | Auth | Path-token URL needs no headers. For the header form, use `https://<domain>/mcp/v1/<server_id>` and add `headers={"Authorization": f"Bearer {token}"}` to `params`. |

    Source: [OpenAI Agents SDK MCP docs](https://openai.github.io/openai-agents-python/mcp/).
  </Tab>

  <Tab title="Vercel AI SDK">
    Use `createMCPClient` from `@ai-sdk/mcp` with `transport: { type: 'http' }`. Pass the resulting tools into `streamText` or `generateText`.

    ```ts TypeScript
    import { createMCPClient } from '@ai-sdk/mcp';
    import { streamText } from 'ai';
    import { anthropic } from '@ai-sdk/anthropic';

    const mcpClient = await createMCPClient({
      transport: {
        type: 'http',
        url: 'https://<domain>/mcp/v1/<token>/<server_id>',
      },
    });

    const tools = await mcpClient.tools();

    const result = await streamText({
      model: anthropic('claude-sonnet-4-5'),
      tools,
      prompt: 'List my recent records.',
      onFinish: async () => { await mcpClient.close(); },
    });
    ```

    | Detail | Value |
    |--------|-------|
    | Package | `@ai-sdk/mcp` (with `ai`) |
    | Function | `createMCPClient` (stable, not `experimental_`) |
    | Transport | `'http'` (Streamable HTTP); also supports `'sse'` and `'stdio'` |
    | Auth | Path-token URL needs no headers. For the header form, use `https://<domain>/mcp/v1/<server_id>` and pass `headers: { Authorization: 'Bearer ' + token }` inside the transport config. |

    <Tip>
    Always `await mcpClient.close()` when the stream finishes to release the connection.
    </Tip>

    Source: [Vercel AI SDK MCP tools docs](https://ai-sdk.dev/docs/ai-sdk-core/mcp-tools).
  </Tab>

  <Tab title="LangChain">
    Use `MultiServerMCPClient` from `langchain-mcp-adapters`. The transport string for Streamable HTTP is `"http"` (not `"streamable_http"`).

    ```python Python
    import asyncio
    from langchain_mcp_adapters.client import MultiServerMCPClient
    from langchain.agents import create_agent

    async def main():
        client = MultiServerMCPClient({
            "refold": {
                "transport": "http",
                "url": "https://<domain>/mcp/v1/<token>/<server_id>",
            }
        })
        tools = await client.get_tools()
        agent = create_agent("openai:gpt-4.1", tools)
        result = await agent.ainvoke(
            {"messages": [{"role": "user", "content": "List available tools"}]}
        )
        print(result)

    asyncio.run(main())
    ```

    | Detail | Value |
    |--------|-------|
    | Package | `langchain-mcp-adapters` (Python) |
    | Class | `MultiServerMCPClient` |
    | Transport key | `"http"` |
    | Auth | Path-token URL needs no headers. For the header form, use `https://<domain>/mcp/v1/<server_id>` and add a `headers` entry with `Authorization: Bearer <token>` alongside `transport` and `url`. |

    <Note>
    This snippet is async, so run it inside an event loop. You also need `langchain`, `langgraph`, and a model provider package (for example, `langchain-openai`) installed.
    </Note>

    Source: [langchain-mcp-adapters README](https://github.com/langchain-ai/langchain-mcp-adapters).
  </Tab>

  <Tab title="Mastra">
    Use `MCPClient` from `@mastra/mcp`. Mastra auto-detects Streamable HTTP from the presence of a `url` (versus a `command` for stdio).

    ```ts TypeScript
    import { Agent } from "@mastra/core/agent";
    import { MCPClient } from "@mastra/mcp";
    import { openai } from "@ai-sdk/openai";

    const mcp = new MCPClient({
      servers: {
        refold: {
          url: new URL("https://<domain>/mcp/v1/<token>/<server_id>"),
        },
      },
    });

    export const agent = new Agent({
      name: "refold-agent",
      model: openai("gpt-4o"),
      instructions: "You can use Refold integration tools.",
      tools: await mcp.getTools(),
    });
    ```

    | Detail | Value |
    |--------|-------|
    | Package | `@mastra/mcp` (with `@mastra/core`) |
    | Class | `MCPClient` |
    | Transport | Streamable HTTP first, falls back to SSE if it fails |
    | Auth | Path-token URL needs no headers. For the header form, use `https://<domain>/mcp/v1/<server_id>` and add `requestInit: { headers: { Authorization: 'Bearer ' + token } }` to the server config. |

    Source: [Mastra MCP docs](https://mastra.ai/docs/agents/using-tools-and-mcp).
  </Tab>
</Tabs>

## Common patterns

### One URL per end user

Refold issues a unique Server URL per linked account. In production, issue one URL per end user, store it alongside the end user's record, and inject it into the agent SDK at request time.

```python Python
# Pseudocode, same shape across all SDKs
url = db.fetch_refold_url(end_user_id)
agent = build_agent_with_mcp(url)
```

For the full token model, including what happens when a token is invalid and how to scope which apps an agent can reach, see [Authentication](/v3/mcp/connect/authentication).

### Rotating tokens

If you regenerate a linked account's token in the Refold dashboard, the old URL stops working immediately. Fetch the latest URL before rebuilding the agent, or store the URL with an expiry and refresh on rotation. See [Authentication](/v3/mcp/connect/authentication) for the revocation model.

### When a connection fails

If the Server URL can't authenticate, the connection fails before any tool runs and the SDK surfaces the HTTP status.

| Status | Cause | Fix |
|--------|-------|-----|
| `401` | The URL is missing its token or `<server_id>` | Copy the full Server URL from the dashboard |
| `403` | Token is invalid or expired, often after a rotation | Fetch the current URL from the dashboard and rebuild the agent |
| `404` | The MCP server is disabled | Re-enable the server in the dashboard, or check the `<server_id>` in the URL |
| `503` | Token validation is temporarily unavailable | Retry the request |
