<Update label="June 17, 2026" description="v3.7.00 · Universal Connector, MongoDB, trigger idempotency & rate limiting">

<Badge color="blue">Platform</Badge> <Badge color="orange">Agent</Badge> <Badge color="green">APIs</Badge> <Badge color="purple">Integrations</Badge>

**A production-ready Universal Connector for any API-backed service, a native MongoDB connector with 18 actions, per-tenant rate limiting, trigger idempotency, and a redesigned Custom Output field picker.**

**New Features**

- **Universal Connector**: Connect to any API-backed service and run its actions inside workflows, with authentication, execution, and lifecycle handled end-to-end. Universal Connectors support a connector-level base URL, stable operation IDs on each action (immutable across updates), OAuth and key-based auth flows via the connect portal, and cascading teardown of connections and chat memory when a connector is deleted.
- **Per-Tenant Rate Limiting**: A new token-bucket rate limiter enforces per-organization and per-environment request budgets, returning HTTP 429 when the budget is exceeded. Configurable burst capacity and refill rate, fail-open on transient infrastructure issues, and disabled by default.
- **Trigger Idempotency**: Opt-in idempotency on triggers deduplicates events by trigger and linked account. Duplicate deliveries are recorded with a new `DUPLICATE` status instead of re-executing the workflow.
- **Custom Output Field Picker**: The workflow builder's Custom Output configuration is now a checkbox list driven by the node's actual test response, with an inline Run Node action, encryption pills, and support for wildcard selectors on arrays of objects. Saved fields absent from the latest response are preserved, and duplicate fields are prevented by construction.
- **Connect Portal — Faster Field Setup**: An "Add Fields" CTA in the fields list and workflow drill-down, plus drag-and-drop of dataslot cards from the palette directly onto the fields list.
- **Legacy Connect Portal Access**: A new "Connect Portal" sidebar entry (tagged Legacy) provides continued access to the previous portal configuration UI.
- **Account-Level Sync Ledger**: The per-workflow delta cron screen has been replaced with an account-level sync ledger tab, with workflow, config, and date filters, instance links, and split workflow/instance columns.

**Improvements**

- **Connections Workflow Search**: Search workflows by name or alias across Shared and Private views, with contextual empty states and a clear button.
- **MCP Logs Unification**: Global MCP logs and server audit logs now use the shared catalog and table layout, aligning the toolbar, export, filters, and detail panel with project executions.
- **Private Workflows Grouping**: Private workflows now group by folder, show the application icon and name on each item, and provide a search-aware empty state.
- **Workflow Settings Consolidation**: Workflow metadata now lives in the settings tab; the separate workflow-details sidebar panel has been removed.
- **Group Field on Public Workflow List**: The public workflow list response now includes the workflow's group.
- **Improved Archival Efficiency**: Archived instance data is now packed into a single blob per unit, cutting object count and storage footprint. Reads auto-detect packed and legacy formats, so previously archived instances remain accessible.
- **Improved Integration Retry Visibility**: Transient integration failures (429s, 5xx responses, connection resets) are now retried at the workflow orchestration layer, so every retry attempt appears in execution history. The set of retryable status codes is configurable.
- **Terminate Semantics in Loops**: A batch carrying `terminate_with_error` inside a loop now marks the instance as ERRORED, while `terminate_with_success` completes the batch without propagating termination. Sibling batches continue to run to completion. The same semantics apply inside subflows and try-catch.
- **Try-Catch with Terminate Iteration**: `terminate_iteration` with the success toggle now ends the iteration cleanly without running the wrapping try-catch's try-success nodes.
- **Descriptive Access-Denied Messages**: Requests denied due to insufficient role permissions now return a clear, structured 403 response.
- **Start Node Payload Retention**: Start-node payloads are now retained in the UI after saving.

**API Changes**

- ⚠️ Breaking: The MCP surface has moved from `/mcp/v3` to `/mcp/v1`. Update any client, gateway, or ingress configuration that targets the MCP path.
- `POST /api/v2/public/linked-accounts` and update endpoints now accept an `env_vars` field for setting environment variable values at account creation or update time. The previous UDF field is deprecated.
- New `PATCH /api/v2/public/tables/:tableId/records/:recordId` endpoint for updating individual table records.
- Custom webhook trigger payloads now accept optional `headers` and an `IdempotencyConfig` for opt-in event deduplication.
- When rate limiting is enabled, over-budget requests return HTTP 429.

**Integration Updates**

- **MongoDB**: New native connector with key-based authentication and 18 actions covering document CRUD (`insert_one`, `insert_many`, `find_one`, `find_many`, `update_one`, `update_many`, `delete_one`, `delete_many`, `replace_one`, `find_one_and_update`, `find_one_and_delete`), aggregation, counting, distinct values, bulk writes, collection and database listing, and connectivity checks.
- **PostgreSQL**: File upload size limits on PostgreSQL actions are now configurable.
- **Dropbox**: Improved reliability of the `get_user` action for Dropbox Business accounts.

</Update>


<Update label="June 15, 2026" description="v3.6.00 · Workday custom output & Zoho webhook fixes">

<Badge color="orange">Agent</Badge> <Badge color="purple">Integrations</Badge>

**Custom output now applies for Workday actions, external node templating handles unresolved fields more cleanly, and Zoho webhook deletions are reliable.**

**Improvements**

- **External Node Templating**: Unresolved templated action fields are now dropped instead of passed through, and empty templated fields in params, headers, and body are blanked so values no longer serialize as `<nil>`.
- **External Node Output Filtering**: External node operations now support output field filtering.
- **Instance Error Persistence**: Instances are now reliably marked as errored when DAG persistence fails.

**Integration Updates**

- **Workday**: Custom output now applies for Workday SOAP API actions.
- **Zoho**: Improved reliability of `delete_webhook` and `create_webhook` actions.
- **HTTP Requests**: Resolved double-encoding of query parameters across outbound HTTP calls.

</Update>

<Update label="June 11, 2026" description="v3.5.36 · Webhook trigger UI, Knowledge Library & execution reliability">

<Badge color="blue">Platform</Badge> <Badge color="orange">Agent</Badge> <Badge color="purple">Integrations</Badge>

**A webhook trigger picker on the workflow Start node, a new Knowledge Library replacing the Integration Graph, password-expiry handling improvements, and Workday and Zoho integration enhancements.**

**New Features**

- **Webhook Trigger on Start Node**: A new webhook trigger entry is available in the Start node trigger picker, with configuration UI for inbound webhook workflows. Available behind a feature flag.
- **Knowledge Library**: The Integration Graph has been replaced with a new Knowledge Library, featuring an apps landing page, per-app library pages, and subtle layer count tags on each tab.
- **Execution Search in Project Logs**: Project logs now include a search bar, bringing project-level logs to parity with the native logs page.
- **Partial Table Record Saves**: Table records can now be saved with partial field values without being blocked.

**Improvements**

- **Password Expiry Handling**: Expired passwords at login now return a distinct error code so the login screen can guide the user to reset, instead of showing a generic locked-account message.
- **Admin Password Reset Flow**: Resetting a user's password from the members settings now unlocks the account, refreshes the password change timestamp, and surfaces the new password with a clipboard copy control and hover-to-reveal masking.
- **Two-Factor Status on Organization Users**: The organization users list now shows each user's two-factor enrollment status alongside accurate account-creation and last-updated timestamps.
- **Unsaved Changes on Role Edits**: Changing a member's role in the settings dialog now surfaces an "unsaved changes" indicator.
- **Webhook Triggers in Private Workflows**: Webhook trigger URLs for private workflows are now validated against the workflow's linked account.
- **Execution Cursor Pagination Ordering**: Cursor-based pagination for instance lists now returns results in consistent creation-time order.
- **Workflow Alias in Linked Account View**: The workflow alias is now included in the Workflows tab on linked accounts across both list responses.
- **Dependent Config Field Resolution**: Config fields with dependent options now resolve their option lists inline for the hosted portal.
- **Try-Catch Node Duplication**: Duplicating a Try & Catch node now produces independent try-success and try-failed branches.
- **Cleaner Node Settings Panel**: The Node Settings section is now hidden for Try & Catch and Delay nodes, where it had no applicable options.
- **Execution Detail Back Navigation**: The back button on execution detail now reliably returns to project executions.
- **Execution Engine Queue Tiering**: Sync and async workflow executions are now routed through separate task queues for better isolation and throughput.
- **Resilient Execution Retries**: Transient execution failures are now retried automatically with backoff, improving reliability through intermittent platform conditions.
- **Workspace-Disabled JWTs**: When workspace mode is off, session tokens no longer carry an environment claim.

**Integration Updates**

- **Workday**: Improved pagination handling for the Companies action under the Financial Management module, plus a new `Get_Supplier_Invoice_Adjustments` operation in Resource Management.
- **Zoho**: Webhook subscriptions now fan out to all linked accounts subscribed to an app, so a single event reliably triggers every connected workflow. An automatic migration backfills existing subscriptions.
- **Universal Connector**: Reworked action management with create, update, and delete support, partial-merge updates on existing actions, and optional OAuth2 client credentials at connector creation time.

</Update>

<Update label="June 9, 2026" description="v3.5.35 · Workspaces & environments, table audit events & HubSpot V3 actions">

- **Assets Page**: A new tabbed Assets page consolidates Documents, Artifacts, and Lookups into a single workspace, replacing the previous standalone pages and sidebar entries.
- **Project Settings — Integrations Tab**: Integration cards have moved into a dedicated Integrations tab within Project Settings.
- **Navigation Header**: A new top navigation header with a logo menu, product switcher, and context switcher on project and MCP server pages.
- **Persistent Logs Search State**: Execution log search type and custom-field filter values now persist in the URL across refreshes.

**Improvements**

- **Execution Logs Node Detail**: The execution logs page now surfaces top-level node status, request, response, and completion percentage for workflow runs on the new execution engine.
- **Sidebar Session Grouping**: Recent sessions in the sidebar are now grouped into Design and Maintain buckets, and the BUILD section has been renamed to AGENT with a New Session entry.
- **MCP Tools Page**: Tool counts are now split into separate actions and workflows totals.
- **Workflow AI Chat Availability**: The workflow AI chat is now available regardless of draft state.
- **Artifact Visibility Scope**: Artifacts are now visible across the entire organization (previously scoped to a single linked account).
- **Enhanced Execution Reliability for Large Workflow Trees**: Terminal status is now persisted before cleanup runs, and cleanup operations have an extended timeout with heartbeating.
- **Workflow Stop Semantics**: Stopping a workflow now extends cancellation to try-catch sub-workflow descendants.
- **Optimized Subflow Execution**: Improved reliability of subflow node execution.
- **Batches Drill-Down Coverage**: The batches drill-down now surfaces failed final iterations of do-while loops with group iterations.
- **Extended Execution Retry Backoff**: Increased retry backoff for instance persistence to ensure execution completion through transient platform conditions.
- **MCP Tool Error Detail**: MCP tool failures now surface the underlying error message instead of a generic execution error.
- **Multi-Role User Support**: Users can now hold multiple roles scoped to org, environment, or linked-account levels.

**Integration Updates**

- **HubSpot**: New `update_task_v3`, `search_tasks_v3`, and `search_tickets_v3` actions with pagination support on search actions. The `list_webhook_subscriptions` action has been removed.
- **Workday**: Improved pagination support for external app actions flagged as paginated.

</Update>

<Update label="June 3, 2026" description="v3.5.33 · Inbound webhooks, Workday OAuth2, MCP redesign & session chat">

<Badge color="blue">Platform</Badge> <Badge color="orange">Agent</Badge> <Badge color="green">APIs</Badge> <Badge color="purple">Integrations</Badge>

**Inbound webhook triggers, Workday OAuth2, a redesigned MCP server management experience, threaded session chat, linked-account scoped tables, and a new project creation wizard.**

**New Features**

- **Inbound Webhook Trigger**: Trigger workflows directly from any external system using a dedicated per-workflow webhook URL. Accepts JSON payloads up to 1 MB.
- **Linked-Account Scoped Tables**: Tables can now be scoped to a specific linked account (private) or shared workspace-wide. Toggle the scope during table creation and filter results accordingly.
- **Workday OAuth2**: Workday connections now support OAuth2 authentication alongside the existing key-based option.
- **PostgreSQL SSL Mode**: Configure SSL mode (disable / require / verify-full) on PostgreSQL connections for compatibility with cloud providers such as Neon, RDS, and Supabase.
- **MCP Server Management Redesign**: Fully rebuilt MCP server pages with a card-grid server list and infinite scroll, tabbed detail layout (Setup, Skills, Tools, Settings, Logs), VS Code / Claude Code / Generic client configuration tabs, and CSV/XLSX export from the Logs page.
- **New Project Creation Wizard**: A 3-step wizard (Details → Connect → Review) replaces the previous modal when creating a new project. Invalid project names are caught before submission.
- **Session Chat**: Threaded chat view with a sticky composer, deep-linkable sessions via `?session=` query param, and a stop button to cancel in-flight responses.
- **Edit Table Records Inline**: Table records can now be edited directly from the UI without leaving the table view.
- **Field Mappings**: New feature with a dedicated page, a table view, and direct integration with the AI agent for building and reviewing field mappings.
- **Private Workflows in Try API**: Private workflows are now listed when a linked account is selected in the Try API panel.

**Improvements**

- **ASK_USER Questionnaire**: Redesigned with numbered, keyboard-navigable options (arrow keys, number keys, Enter), single-select auto-advance, per-question Skip, and an inline "Something else" input.
- Agent session sidebar now surfaces recent sessions with direct links to the related workflow or mapping.
- Execution logs now support bidirectional cursor-based pagination with a persistent cursor in the URL — bookmarking and reloading returns to the same position.
- Password visibility toggle added to the login page.
- OpenAPI-imported actions with object-type fields now render as expandable cards with per-subfield inputs in the developer portal and Test Action dialog.
- Node action lists now load lazily for improved performance on large action sets.
- Datasource dropdowns in Map v2 at workflow level now render correctly and can be cleared.
- Row click enabled in portal tables, with context-aware back navigation.
- Fix: updating an event with its existing trigger name no longer returns an error.
- Fix: `terminate_with_success` inside a try-catch block now correctly stops execution instead of continuing to downstream nodes.
- Subflow deduplication via idempotency key prevents duplicate child instances on retries.
- Linked-account environment variable values are now scoped per environment.
- Improved AI context awareness for more accurate workflow and mapping suggestions.

**API Changes**

- New: `POST /api/v1/webhook-trigger/:orgId-:environment/:workflowId/:linkedAccountId` — inbound webhook trigger endpoint; accepts JSON payloads up to 1 MB.
- `GET /instances` — now supports bidirectional cursor pagination via `currentCursor` and `direction=next|prev`.

**Integration Updates**

- **Workday**: OAuth2 authentication added alongside key-based; SOAP requests now send OAuth2 Bearer tokens and retry automatically on token expiry; Resource & Financial Management SOAP operations expanded.
- **PostgreSQL**: SSL mode support end-to-end across connections, migrations, and connection pooling.
