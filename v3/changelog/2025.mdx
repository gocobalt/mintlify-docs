<Note>
Through 2025, Refold shipped on a monthly release cadence under an earlier versioning scheme, so these entries are grouped by month and labelled `2025.MM` for the period they cover. Unified (BOM) versioning with per-release entries began in February 2026.
</Note>

<Update label="December 2025" description="2025.12 · Public workflow APIs">

<Badge color="blue">Platform</Badge> <Badge color="orange">Agent</Badge> <Badge color="green">APIs</Badge> <Badge color="purple">Integrations</Badge>

**Public workflow APIs, QuickBooks/NetSuite/Zoho CRM/Freshdesk connectors, an AI execution service integration, a retry node, and hosted-portal (Streamlit) work — closing out the year before unified BOM versioning began in February.**

**New Features**

- **Public Workflow APIs**: Public API endpoints for managing and executing workflows.
- **Retry node**: Configurable retry handling in the execution engine.
- **AI execution service integration**: Connected the execution engine to a dedicated AI execution service.
- **Hosted portal (Streamlit)**: Incorta connection page, custom setup UI, and private node listing.

**Improvements**

- Org basics/features APIs and Incorta token handling in auth.
- Child-workflow orchestration for group/pagination nodes, batched concurrent execution, and execution-data archival.
- **ServiceNow** delta-cron fix; email-node migration.

**Integration Updates**

- Added **QuickBooks** (+ estimate object), **NetSuite**, **Zoho CRM**, **Freshdesk** (actions), and **Incorta** connectors.
- **Facebook Ads** ad-account support.

</Update>

<Update label="October 2025" description="2025.10 · Adopted automated semantic-release versioning per service">

<Badge color="blue">Platform</Badge> <Badge color="purple">Integrations</Badge>

**Adopted automated semantic-release versioning per service, added the Iconik connector, expanded RBAC, and grew the execution engine with new node types (HTTP, PDF, email, delay, tables) and a high-availability store.**

**New Features**

- **Per-service versioning**: Adopted automated semantic-release and date-based release workflows across services — the first step toward unified versioning.
- **New workflow nodes**: HTTP, PDF, email, file-handler, delay, and tables nodes added to the execution engine, plus DAG iteration support.
- **High-availability execution store**: Hybrid Raft + async key-value store for resilient execution, with Prometheus metrics.

**Improvements**

- RBAC expansion, custom-app delete, send-email, and Mongoose validation in auth.
- Serverless presigned URLs and custom content-type support.
- Dashboard: removed legacy signup and cleaned up unused routes.

**Integration Updates**

- Added **Iconik** connector.

</Update>

<Update label="September 2025" description="2025.09 · RBAC groundwork">

<Badge color="blue">Platform</Badge> <Badge color="orange">Agent</Badge> <Badge color="purple">Integrations</Badge>

**RBAC groundwork, an LLM workflow node, many connectors (Coupa, Snowflake, Workday, Miro, Gong), and major buildout of the durable execution engine (DAG execution, node registry, code/CSV nodes).**

**New Features**

- **Role-Based Access Control (groundwork)**: Began the RBAC permission model later shipped in v3.4.0.
- **LLM node**: Workflow node with PDF/DOCX support for AI processing.
- **Execution engine buildout**: DAG-based workflow execution, node config registry, code-executor and CSV/Excel nodes, a logger node, schema gRPC service, and MongoDB-backed node persistence.

**Improvements**

- **Zoom** transcript events, CSV file-format handling, and conditional group nodes.
- **Snowflake** identity-provider auth and **Miro** auth.

**Integration Updates**

- Added **Coupa**, **Workday Financial**, **Snowflake**, **Miro**, **Gong**, and **Gmail/Calendar** connectors.
- **Slack** action coverage and **Google Drive** triggers expanded.

</Update>

<Update label="August 2025" description="2025.08 · Foundation work on a new durable workflow execution engine">

<Badge color="blue">Platform</Badge> <Badge color="purple">Integrations</Badge>

**Foundation work on a new durable workflow execution engine, plus Ashby, Twilio and SharePoint connectors and OAuth/IAM auth improvements.**

**New Features**

- **Durable execution engine (foundation)**: Began building a new Temporal-based, persistent workflow execution engine with a key-value execution store for reliable long-running workflows.
- **SharePoint file upload**: Upload files to SharePoint from workflows.
- **Multi-auth**: Support for multiple authentication methods per connector.

**Improvements**

- S3 IAM-based auth, Google granted-scope handling, and Sage Intacct token expiry.
- Serverless: write-data-to-file action and CSV handling.
- **Salesforce** case triggers; **Dropbox** and zip-extract handling.

**Integration Updates**

- Added **Ashby**, **Twilio**, and **SharePoint** connectors; **Aircall** OAuth.

</Update>
