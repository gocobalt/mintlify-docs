---
title: "Token Refresh"
description: "Refold refreshes your customers' access tokens automatically, so connections keep working without any code from you."
---

**Token refresh** is how Refold keeps a customer's connection alive after their initial authorization. Refold stores the encrypted tokens from the connect flow and **refreshes the access token automatically** whenever it expires, there's nothing for you to build or call in the normal case.

This runs entirely on Refold's side once a customer has [connected an app](/v3/native/frontend/overview). You only get involved when a token *can't* be refreshed, which is handled by [re-authentication](/v3/platform/authentication/managing-connections/re-authentication).

## How it works

When a customer authorizes an app, Refold stores their credentials encrypted against their [linked account](/v3/concepts/linked-account). From then on:

- Refold signs every API call it makes on the customer's behalf with a valid access token.
- When an access token is close to expiring, Refold uses the stored **refresh token** to obtain a new one, before your workflows are affected.
- Your code never sees or handles raw tokens, and your workflows keep running uninterrupted.

<Tip>
There's no SDK method or API to refresh a token yourself, it's automatic. The application object you read for [Connection status](/v3/platform/authentication/managing-connections/connection-status) reflects a healthy connection as long as refresh keeps succeeding.
</Tip>

## When refresh isn't enough

Automatic refresh only fails when the refresh token itself is no longer valid, for example it expired, was revoked, or the customer uninstalled the app. Refold can't recover a connection on its own in those cases, so it sets `reauth_required` on the app and (if you subscribe) fires a **Connection Expired** webhook.

That's the one case where the customer has to step in. Prompt them to reconnect. See [Re-authentication](/v3/platform/authentication/managing-connections/re-authentication).
