---
title: "Re-authentication"
description: "Detect when a customer's connection stops working and prompt them to reconnect so your workflows keep running."
---

**Re-authentication** is what happens when Refold can no longer act on a customer's behalf and needs them to authorize the app again. Refold flags the connection with **`reauth_required`** and can fire a **Connection Expired** webhook; you respond by prompting the customer to reconnect.

You'll need this when a customer's credentials expire, their token is revoked, or they uninstall the app. This page assumes the customer already has a [linked account and a connected app](/v3/native/frontend/overview); it covers recovering a connection that broke, not the first connect.

## When a connection needs re-auth

Refold stores and [refreshes tokens automatically](/v3/platform/authentication/managing-connections/token-refresh), so most connections never need attention. A connection only breaks when Refold can't authenticate anymore:

- **Credentials expired**: an API key or refresh token reached the end of its life.
- **Token revoked**: the customer or the provider revoked the access or refresh token.
- **App uninstalled**: the customer removed the app on the provider's side.

When any of these happen, Refold sets `reauth_required` to `true` on that app for the customer's [linked account](/v3/platform/concepts/linked-account).

## Detect that re-auth is needed

There are two ways to know a connection has expired, use whichever fits your product:

<AccordionGroup>
  <Accordion title="Check the reauth_required flag" icon="flag">
    Read the application object for the customer (see [Connection status](/v3/platform/authentication/managing-connections/connection-status)) and check `reauth_required`. When it's `true`, show a **Reconnect** prompt.

    ```json
    {
      "name": "Slack",
      "slug": "slack",
      "auth_type": "oauth2",
      "connected": true,
      "connected_accounts": [
        { "identifier": "acme-workspace", "connectedAt": "2023-05-31T07:45:59.841Z" }
      ],
      "reauth_required": true
    }
    ```
  </Accordion>
  <Accordion title="Subscribe to the Connection Expired webhook" icon="webhook">
    Subscribe to the **Connection Expired** event to be notified the moment a connection breaks, rather than polling. Refold posts a payload identifying the linked account and the app:

    ```json
    {
      "linked_account_id": "12345",
      "org_id": "645270e37917442c4307297a",
      "environment": "test",
      "type": "General",
      "event": "Connection Expired",
      "slug": "pipedrive",
      "createdAt": "Tue, 07 Nov 2023 08:38:33 GMT"
    }
    ```

    See [Webhooks](/v3/developer/configure/webhooks) for how to subscribe and verify events.
  </Accordion>
</AccordionGroup>

## Prompt the reconnect

How the customer reconnects depends on the flow you use:

- **Hosted flow and React SDK flow**: re-auth is handled for you. When a connection has expired, the customer is automatically prompted to reconnect inside the [hosted portal](/v3/platform/authentication/connection-flows/hosted-flow) or the [React connect component](/v3/native/frontend/react-sdk-flow).
- **Build your own**: when `reauth_required` is `true`, call `cobalt.connect()` with the app slug again, exactly as you did for the first connection. See [Build your own](/v3/platform/authentication/connection-flows/build-your-own).

<CodeGroup>
```javascript OAuth app
// opens the provider's auth screen in a new tab
cobalt.connect("slack");
```

```javascript Key-based app
// saves fresh credentials for a key-based app
cobalt.connect("twilio", {
  number: "<credential>",
  sid: "<credential>",
});
```
</CodeGroup>

Once the customer finishes, re-fetch the application object, `reauth_required` returns to `false` and your UI can show the **Connected** state again.

<Frame>{/* SCREENSHOT: a Reconnect prompt on an expired app card → /images/embedded/frontend/reconnect.png */}</Frame>

<Note>
Re-authentication reuses the same connect flow as the first authorization, there's no separate "reconnect" method. The only difference is that the customer already has a linked account, so their existing config is preserved.
</Note>
