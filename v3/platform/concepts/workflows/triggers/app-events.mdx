---
title: "App Events"
description: "Run a workflow in real time from a built-in event in your users' connected apps."
---

**App events** are triggers that come built into an integration. They run a
workflow in real time when something happens in your users' connected app,
using that app's native webhooks.

For example, to keep your application in sync, you can use Salesforce's
`Contact Created` event to trigger a workflow whenever a new contact is
created in a user's Salesforce account.

## Use an app event

<Steps>
  <Step title="Open the Start node">
    On the workflow canvas, open the **Start node** and choose an **app
    trigger** for the integration you want (for example, Salesforce).
  </Step>
  <Step title="Pick a built-in event">
    Select one of the integration's available events, such as **Contact
    Created**.
  </Step>
  <Step title="Enable the workflow">
    Once enabled, the workflow runs in real time each time the event occurs
    in a user's connected account.
  </Step>
</Steps>

<Note>
  App events rely on the app's webhooks, so they're only available for events
  the integration supports. For apps or events without webhook support, use a
  [Polling](/v3/platform/concepts/workflows/triggers/polling) trigger
  instead.
</Note>
