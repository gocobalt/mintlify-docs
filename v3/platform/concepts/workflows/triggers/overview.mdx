---
title: "Triggers"
description: "Every workflow starts with a trigger that decides when it runs and what data it receives."
---

A **trigger** determines when a workflow runs and how data is passed into it.
You set the trigger in the workflow's **Start node**.

Refold supports six trigger types. Pick the one that matches how your
workflow should start.

<CardGroup cols={2}>
  <Card title="Schedule" href="/v3/platform/concepts/workflows/triggers/schedule">
    Run a workflow automatically at a set time or on a recurring interval.
  </Card>
  <Card title="Polling" href="/v3/platform/concepts/workflows/triggers/polling">
    Run a workflow when records change in a connected app that doesn't support webhooks.
  </Card>
  <Card title="Webhook" href="/v3/platform/concepts/workflows/triggers/webhook">
    Run a workflow when any external system sends an HTTP request to a unique URL.
  </Card>
  <Card title="Events" href="/v3/platform/concepts/workflows/triggers/events">
    Run a workflow from a custom event your application sends through the Refold SDK or API.
  </Card>
  <Card title="App events" href="/v3/platform/concepts/workflows/triggers/app-events">
    Run a workflow in real time from a built-in event in your users' connected apps.
  </Card>
  <Card title="API trigger" href="/v3/platform/concepts/workflows/triggers/api-trigger">
    Run a workflow programmatically with a direct API call.
  </Card>
</CardGroup>
