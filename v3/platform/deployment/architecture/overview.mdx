Refold runs as a set of stateless services in your own environment — a Kubernetes cluster or a Docker host — behind a single load balancer. The services hold no state of their own. You provision a database, a cache, and object storage, and Refold runs against them. This page covers the high-level shape of a deployment and what you need to provide before installing.

## How it works

A Refold deployment has two stateless parts: the **application** that serves the dashboard and REST API, and **workers** that execute workflows and integration calls. Both scale horizontally — add replicas to handle more load, and replace or roll back instances without downtime, because no state lives inside them.

- **All traffic enters through a load balancer or ingress** (such as nginx or an Amazon ALB), which serves the dashboard (UI access) and the REST API (API access).
- **Everything Refold persists lives in the backing services you provision** — configuration, credentials, workflow history, and files.
- **Integrations reach out from the deployment** to the third-party SaaS APIs your connectors call.

## What you provision

Provision the following before installing Refold. For production, run managed versions; for evaluation, the deployment package can run them in-cluster.

| Dependency | Purpose | Required |
|---|---|---|
| **Database (MongoDB)** | System of record — organizations, linked accounts, connectors, workflow definitions, and execution history. | Yes |
| **Redis** | Caching, distributed locks, and the job queues that drive workflow execution. | Yes |
| **PostgreSQL** | Backs durable workflow orchestration. | Yes |
| **Object storage (S3 / GCS / MinIO)** | Files, exports, and generated artifacts. | Yes |

You also need **outbound network access** from the deployment to the third-party SaaS APIs your integrations call, and to any LLM providers (OpenAI, Anthropic, OpenRouter) used by AI features.

## Access

Users reach the dashboard and your applications call the platform through the same load balancer that fronts the deployment. Both UI and API traffic terminate there before reaching the Refold application.
