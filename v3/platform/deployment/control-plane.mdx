The Refold Control Plane is a self-service management interface for Refold on-premises customers. Use it to manage your deployment and environments directly: view clusters, create namespaces, upgrade Refold versions, and configure environment variables and secrets, without opening a support request.

Your DevOps team operates the Control Plane. Refold provisions your access, and everything you see is scoped to the environments your account is authorized for.

## What you can do

- **View clusters** assigned to your project, pulled directly from your environments.
- **Manage namespaces** — see each environment's version, status, and last update.
- **Create namespaces** in any assigned cluster to run additional workloads.
- **Upgrade versions** to newer Refold releases through a guided version picker.
- **Configure environment variables and secrets** per namespace.

## Sign in

Access is credential-based. Refold provisions three things for each authorized user:

- **Username**
- **Password**
- **Control Plane URL**

<Info>
Your credentials are mapped to specific environments. After you sign in, the clusters and namespaces for your authorized environments appear automatically. No extra configuration is required.
</Info>

## Dashboard overview

After you sign in, the Deployments dashboard lists every resource assigned to your account:

- All **clusters** assigned to your account.
- The **cloud provider** for each one (AWS, GCP, or another supported platform).
- The **deployment region**.

## Cluster management

Select a cluster to open its detail page, where you manage that cluster's namespaces and review its configuration.

### Cluster metadata

Each cluster shows:

| Field | Description |
|---|---|
| Cluster name | The cluster's identifier |
| Deployment region | Where the cluster runs |
| Cloud provider | AWS, GCP, or another supported platform |
| Creation date | When the cluster was created |
| Last updated | The most recent change timestamp |

### Namespace overview

Namespaces in the selected cluster are listed in a table showing each namespace's name and current operational status.

## Create a namespace

A namespace is a distinct environment within a cluster, with its own configuration. Create one to run an additional workload in an assigned cluster.

<Frame caption="A cluster's namespaces, with the Add Namespace action">
  ![Control Plane cluster detail page listing namespaces with an Add Namespace button](/images/deployment/control-plane/namespace-creation.png)
</Frame>

<Steps>
  <Step title="Start a new namespace">
    From the cluster detail page, select **Add Namespace**.
  </Step>
  <Step title="Provide the namespace details">
    Enter a **Namespace name** (a unique identifier) and a **Release version** (the target Refold version to deploy).
  </Step>
  <Step title="Verify environment variables">
    Review the environment variables for the namespace. These are derived from the parent cluster configuration. Change any namespace-specific values as needed.
  </Step>
  <Step title="Provide secret values and submit">
    Supply a value for every required secret, then submit.
  </Step>
</Steps>

<Warning>
On the environment-variable verification screen, secrets are shown with null values by default. You must set a value for every required secret before creating the namespace, or the deployment will not operate correctly.
</Warning>

Once you submit, the Control Plane provisions the namespace and makes it available for use.

## Upgrade a namespace version

Select a namespace row to see its details:

- Namespace name
- Current operational status
- Current Refold version
- Creation date
- Last release trigger timestamp

Available versions are retrieved automatically from Refold's official version streams.

<Frame caption="The Refold Version control on a namespace">
  ![Namespace Information panel with the Refold Version edit button highlighted](/images/deployment/control-plane/version-upgrade.png)
</Frame>

<Steps>
  <Step title="Open the version picker">
    Under **Refold Version**, select the edit button next to the current version.
  </Step>
  <Step title="Choose the target version">
    Select the version you want to deploy.
  </Step>
  <Step title="Trigger the upgrade">
    Select **Trigger Update**. A confirmation appears once the update request is submitted.
  </Step>
</Steps>

<Note>
This release does not track upgrade status in real time. Real-time health monitoring and upgrade status are planned for a future release. Until then, confirm an upgrade's outcome with your Refold support representative if you need to verify it.
</Note>

## Manage environment variables and secrets

Environment variables set namespace-level configuration. For a namespace in scope, you can:

- **Update** an existing variable's value.
- **Clear** a value by setting it to `NA` when it is no longer required.

All changes apply only to the namespace you are editing.

<Frame caption="A namespace's environment variables, with the Bulk Edit action">
  ![Namespace Environment Variables section with the Bulk Edit action highlighted](/images/deployment/control-plane/secrets-bulk-edit.png)
</Frame>

### Secrets

Secrets are supported in this release, and they behave differently from plain variables:

- Provide secret values **during namespace creation**.
- After a successful deployment, secret values are **automatically redacted**.
- To change a secret later, use **Bulk Edit** on the namespace's environment variables and replace the `[Redacted]` placeholder with the new value.
