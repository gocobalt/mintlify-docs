---
title: "Credentials"
description: "Manage your workspace API key and the credentials for external services."
---

The **Credentials** page (shown as the **Credentials vault** in the
dashboard) holds two kinds of secrets: your **Refold workspace API key** and
the **credentials for external services** such as LLM providers.

Open it from **Settings > Organization > Credentials**.

## Workspace API key

The **Refold** section manages the API key for your workspace. Use this key
to authenticate requests to the Refold API. To replace it, click
**Regenerate** to issue a new key.

<Warning>
  Regenerating the key immediately invalidates the old one. Any integration
  or script still using the previous key will stop working until you update
  it with the new value.
</Warning>

## External services

The **External Services** section stores the credentials Refold uses to call
external providers on your behalf — currently the **LLM provider** used across
the application.

Select your provider from the **LLM Provider** dropdown, then enter that
provider's API key and click **Save**. Two providers are supported:

- **OpenAI** — enter your **OpenAI API Key**.
- **OpenRouter** — enter your **OpenRouter API Key**. OpenRouter is a unified
  API that gives you access to multiple LLM providers through a single key.

The key field shown matches the provider you select.
