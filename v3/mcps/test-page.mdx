## Getting Started with MCP Testing

<Steps>
<Step title="Set up your test environment">
  First, configure your development environment for MCP testing:
  
  ```bash
  npm install @cobalt/mcp-sdk
  export COBALT_API_KEY="your_api_key_here"
  ```
  
  <Check>
  Verify your API key is working by calling the health endpoint.
  </Check>
</Step>

<Step title="Initialize MCP client">
  Create a new MCP client instance:
  
  ```javascript
  import { MCPClient } from '@cobalt/mcp-sdk';
  
  const mcpClient = new MCPClient({
    apiKey: process.env.COBALT_API_KEY,
    environment: 'development'
  });
  ```
  
  <Warning>
  Never commit your API keys to version control. Always use environment variables.
  </Warning>
</Step>

<Step title="Test multi-tenant configuration">
  Configure your first customer organization:
  
  ```javascript
  const customerOrg = await mcpClient.organizations.create({
    name: "Test Customer Inc",
    subdomain: "testcustomer",
    settings: {
      whiteLabel: true,
      customBranding: {
        primaryColor: "#007bff",
        logo: "https://example.com/logo.png"
      }
    }
  });
  ```
</Step>
</Steps>

## Code Examples

### Single Language Example

Here's how to authenticate a customer's integration:

```javascript
// Authenticate customer integration
const authenticateCustomer = async (customerId, integrationSlug) => {
  try {
    const authUrl = await mcpClient.auth.getAuthUrl({
      customerId,
      integrationSlug,
      redirectUrl: `https://yourapp.com/auth/callback`,
      scopes: ['read', 'write']
    });
    
    return { success: true, authUrl };
  } catch (error) {
    console.error('Authentication failed:', error);
    return { success: false, error: error.message };
  }
};
```

### Parameter Documentation

<ParamField path="organization_id" type="string" required>
Unique identifier for the customer organization. Must be a valid UUID v4 format.
</ParamField>

<ParamField body="name" type="string" required>
Display name for the customer organization. Used in the admin dashboard and customer-facing interfaces.
</ParamField>

<ParamField query="include_integrations" type="boolean" default="false">
Whether to include active integrations in the response. Set to `true` to get full organization details.
</ParamField>

<ParamField header="X-Customer-Context" type="string">
Optional header to specify customer context for the request. Format: `customer:CUSTOMER_ID`
</ParamField>

### Response Fields

<ResponseField name="organization_id" type="string" required>
Unique identifier assigned to the newly created organization.
</ResponseField>

<ResponseField name="integrations" type="array">
List of active integrations for this organization.

### Accordion for FAQ/Troubleshooting

<AccordionGroup>
<Accordion title="How do I handle authentication failures in MCP?">
  When authentication fails in an MCP environment, follow these steps:
  
  1. **Check customer context**: Ensure you're authenticating within the correct customer organization
  2. **Verify scopes**: Confirm the integration has the required permissions
  3. **Refresh tokens**: Implement automatic token refresh for long-running integrations
  
  ```javascript
  // Handle auth failure with retry logic
  const handleAuthFailure = async (error, customerId, integrationSlug) => {
    if (error.code === 'AUTH_EXPIRED') {
      return await mcpClient.auth.refreshToken(customerId, integrationSlug);
    }
    
    // Log for customer-specific debugging
    await mcpClient.audit.log({
      customerId,
      event: 'auth_failure',
      details: error.message
    });
  };
  ```
</Accordion>

<Accordion title="What are the rate limits for MCP operations?">
  MCP rate limits vary by plan and operation type:
  
  - **Professional Plan**: 1,000 requests/minute per organization
  - **Enterprise Plan**: 5,000 requests/minute per organization
  - **Agency Plan**: 10,000 requests/minute across all customers
  
  <Warning>
  Rate limits are enforced per customer organization in MCP mode, not per API key.
  </Warning>
</Accordion>

<Accordion title="How do I set up white-label branding?">
  White-label branding allows your customers to see your brand instead of Cobalt's:
  
  ```javascript
  const brandingConfig = {
    primaryColor: '#007bff',
    secondaryColor: '#6c757d',
    logo: 'https://yourapp.com/logo.png',
    favicon: 'https://yourapp.com/favicon.ico',
    customDomain: 'integrations.yourapp.com'
  };
  
  await mcpClient.branding.update(organizationId, brandingConfig);
  ```
</Accordion>
</AccordionGroup>
