---
title: "Linked accounts"
description: "How Refold represents each of your customers and the third-party apps they've connected."
---

A **linked account** represents one of your customers (your app's end user),
together with the third-party apps they've connected through your platform.
It's the unit Refold uses to track and manage integrations per customer: each
linked account holds that customer's connections, configuration, executions,
and API calls, so you can monitor and troubleshoot them one customer at a time.

A few things to know before you start:

- **`linked_account_id` must be unique per user.** Use any stable unique value,
  such as an email or your own primary key for that user.
- **Create and modify linked accounts server-side.** These operations involve
  your API key and customer credentials, so they should never run in the
  browser.

## Create a linked account

You can create linked accounts from the dashboard for one-off setup, or via the
API to provision them programmatically.

<AccordionGroup>
  <Accordion title="via Dashboard" icon="laptop">
    <Steps>
      <Step title="Open Linked Accounts">
        Go to **Dashboard > Linked Accounts**.
      </Step>
      <Step title="Add the account">
        Click **+ Add Account**, then enter the **Linked Account ID**,
        **Name**, and any **UDF parameters** you want to attach to this
        customer.
      </Step>
      <Step title="(Optional) Bulk import">
        To create many accounts at once, import a `.csv` file instead.
      </Step>
    </Steps>

  <Accordion title="via API" icon="code">
    <Warning>
      Run linked account operations from your server, never from client-side
      code. The request uses your secret API key.
    </Warning>

    <CodeGroup>

    ```bash cURL
    curl --request POST \
      --url https://app.refold.ai/api/v2/public/linked-account \
      --header 'Content-Type: application/json' \
      --header 'x-api-key: YOUR_API_KEY' \
      --data '{
        "linked_account_id": "user@example.com",
        "name": "User",
        "your_app": {}
      }'
    ```

    ```javascript Node.js
    try {
      await Client.createLinkedAccount({
        linked_account_id: "user@example.com",
        name: "User",
        udf: {
          phone: "1234567890",
          address: "123, Queen Street",
        },
        your_app: {
          app_id: "YOUR_APP_ID",
          auth_credentials: {
            Authorization: "YOUR_APP_AUTH_TOKEN",
          },
        },
      });
    } catch (error) {
      // Handle the error
    }
    ```

    </CodeGroup>

    Replace `YOUR_API_KEY` with your server-side API key, `YOUR_APP_ID` with
    your application's ID, and `YOUR_APP_AUTH_TOKEN` with the auth credential
    for the customer's connection to your app. See the [create linked account
    API reference](/v3/api-reference/linked-accounts/create-linked-account) for the
    full schema.
  </Accordion>
</AccordionGroup>

## Configure a linked account

After a customer authenticates, you can adjust the configuration for any of
their connected applications. For example, set field values or enable and
disable specific [workflows](/v3/developer/configure/workflows/overview) for that customer.

<Steps>
  <Step title="Open the integration's configuration">
    In the linked account's **Overview** tab, find the application under
    **Integrations**. Click the three-dot menu in that row and select
    **Configure**.
  </Step>
  <Step title="Adjust the settings">
    Set the **Fields** values, or enable and disable **Workflows**, to match
    what the customer needs.
  </Step>
  <Step title="Save">
    Click **Save** to apply the configuration.
  </Step>
</Steps>

## Monitor a linked account

Each linked account gives you per-customer visibility into how the integration
is behaving. Open the account from **Dashboard > Linked Accounts** and use the
tabs across the top to inspect each kind of activity.

<AccordionGroup>
  <Accordion title="Execution logs" icon="bolt">
    Every workflow run for this customer, with inputs, outputs, status, and
    duration. Open the **Executions** tab.
  </Accordion>
  <Accordion title="Auth" icon="key">
    Authentication and connectivity events for each connected integration. In
    **Overview > Integrations**, open the three-dot menu on the integration
    row and select **Logs** to see auth events in chronological order.
  </Accordion>
  <Accordion title="User configs" icon="sliders">
    The field values and workflows the customer has enabled for each connected
    integration. Open the **Workflows** tab to review what the customer has
    turned on.
  </Accordion>
  <Accordion title="Schedules" icon="calendar">
    Scheduled triggers running for this customer, with the next-fire time and
    history of past runs. Open the **Schedules** tab.
  </Accordion>
  <Accordion title="Events" icon="bell">
    Inbound events for this customer that triggered workflows. Open the
    **Events** tab to see event payloads and the workflows they fired.
  </Accordion>
  <Accordion title="Webhook logs" icon="webhook">
    Outbound webhook deliveries from this customer's workflows, with status
    and retry history. Open the **Webhooks** tab.
  </Accordion>
  <Accordion title="Polling" icon="rotate">
    Polling activity for data refs and connectors that pull on a schedule.
    Open the **Polling** tab.
  </Accordion>
</AccordionGroup>

For the full reference on what each log type records, see
[Logs](/v3/developer/monitor/logs).

## Personal access tokens

A **personal access token** (PAT) authenticates API calls scoped to a single
linked account. Use a PAT when an end-user-facing tool or script needs to act
as one specific customer without exposing your platform API key.

<Steps>
  <Step title="Open the linked account">
    Go to **Dashboard > Linked Accounts** and open the account you want a
    token for.
  </Step>
  <Step title="Generate the token">
    In the detail panel on the right, click **Generate** next to **Personal
    Access Token**. Copy the token immediately. Refold won't show it again.
  </Step>
</Steps>

<Warning>
  Treat a PAT like a password. Store it in a secrets manager and rotate it if
  it leaks.
</Warning>

## Environment variables

Attach **per-customer environment variables** to a linked account when a
workflow needs values that differ between customers (a region, an account
number, a feature flag). Workflows reference these by name at runtime, and
Refold substitutes the linked account's value at execution time.

<Steps>
  <Step title="Open the linked account">
    Go to **Dashboard > Linked Accounts** and open the account.
  </Step>
  <Step title="Add a variable">
    In the detail panel on the right, click **+ Add** next to **Environment
    Variables**, then enter a name and value.
  </Step>
  <Step title="Reference it from a workflow">
    Use the variable name in any workflow node that needs the value. Refold
    resolves it per linked account at execution time.
  </Step>
</Steps>
