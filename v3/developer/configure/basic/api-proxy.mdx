---
title: API Proxies
description: Create reusable API endpoints that integrate your services with Refold workflows.
---

API Proxies let you define custom API endpoints that your Refold workflows can execute. Each API Proxy wraps an external API call with configurable fields, request parameters, and response handling, so you can reuse the same action across multiple workflows.

Use API Proxies when you need to:

- Fetch data from an external service during a workflow execution
- Send data to your own backend or a third-party API
- Create reusable actions that multiple workflows can share

<Info>
API Proxies are also referred to as **Custom Actions** in some parts of the platform. They are the same feature.
</Info>

## Prerequisites

- A Refold account with access to the Dashboard
- An API endpoint you want to integrate

## Create an API Proxy

<Steps>

  <Step title="Open API Proxies">
    Navigate to **Developer** > **API Proxies** in the Refold Dashboard.
  </Step>

  <Step title="Create a new action">
    Click `New Action`. Provide a **Name** and **Description** for the API Proxy, then click `Create Action`.
  </Step>

  <Step title="Configure the API Proxy">
    After creation, you can configure the proxy using three tabs:

    - **Basic Information**: Update the name or description.
    - **Fields**: Define input fields that pass data between workflow steps.
    - **API Call**: Configure the HTTP request that executes when the action runs.
  </Step>

</Steps>

## Add Fields

Fields allow you to pass dynamic data into the API Proxy from a workflow. For example, you might add a field for a record ID or a customer email that changes per execution.

<Steps>

  <Step title="Open the Fields tab">
    Click the **Fields** tab in your API Proxy configuration.
  </Step>

  <Step title="Add a new field">
    Click the `+` icon. Configure the following properties:

    <AccordionGroup>
      <Accordion title="Field properties">
        <ParamField body="Type" type="string" required>
          The data type of the field (e.g., text, number, select).
        </ParamField>
        <ParamField body="Label" type="string" required>
          Display name shown in the workflow editor.
        </ParamField>
        <ParamField body="Placeholder" type="string">
          Hint text displayed inside the field input.
        </ParamField>
        <ParamField body="Description" type="string">
          Help text explaining what this field is for.
        </ParamField>
        <ParamField body="Mandatory" type="boolean">
          Whether this field must be filled before execution.
        </ParamField>
      </Accordion>
    </AccordionGroup>
  </Step>

  <Step title="Save the field">
    Click `Add Field` to save.
  </Step>

</Steps>

## Configure the API Call

The API Call tab defines the HTTP request that Refold executes when the action is triggered.

<Steps>

  <Step title="Open the API Call tab">
    Click the **API Call** tab in your API Proxy configuration.
  </Step>

  <Step title="Set the request endpoint">
    Enter the API endpoint URL and select the HTTP method (GET, POST, PUT, DELETE, etc.).

    <Tip>
    You can include variables in the URL using curly brackets. For example: `https://api.example.com/users/{user_id}`.
    </Tip>
  </Step>

  <Step title="Add request details">
    Configure the following as needed:

    - **Path Params**: Dynamic segments in the URL.
    - **Query Params**: Key-value pairs appended to the URL.
    - **Headers**: Custom headers for authentication or content type.
    - **Body**: Request payload. Click the `(x)` icon to insert variables or fields you created in the Fields tab.
    - **Pre-request Scripts**: Logic to run before the API call executes.
  </Step>

  <Step title="Save the API call">
    Click `Save API Call`.
  </Step>

</Steps>

## Response Settings

Refold provides additional options in the API Call tab to control how responses are handled.

<AccordionGroup>

  <Accordion title="Response Format">
    Choose between two formats:

    - **Default**: Standard API response. Enabled by default on all API Proxies.
    - **Stream**: Streams the response in chunks. Use this when the API returns data incrementally.
  </Accordion>

  <Accordion title="Save Response to Bucket">
    Enable this toggle to store the API response in an S3 bucket provided by Refold instead of returning it directly.

    This is useful when the response contains binary data such as PDFs or non-JSON content.
  </Accordion>

  <Accordion title="Use Public URL">
    <Warning>
    **Save Response to Bucket** must be enabled for this setting to take effect.
    </Warning>

    By default, the S3 URL generated by the bucket is private and accessible only within the same AWS organization. You can also use this private URL in Refold nodes such as the PDF Node.

    Enable **Use Public URL** to generate a publicly accessible link that you can share externally.
  </Accordion>

  <Accordion title="Pagination">
    If the target API supports pagination, enable the `Pagination` toggle in the API Call section.

    Select the pagination **Type**, fill in the required fields for that type, and click `Save API Call`.
  </Accordion>

</AccordionGroup>

## Use API Proxies in Workflows

Once created, an API Proxy becomes available as an action in your **Custom App** under **Native Apps** in the workflow editor.

A single API Proxy can be used across multiple workflows. You can attach it to trigger conditions or place it as a step within any workflow to automate API calls as part of your integration logic.

<Tip>
You can reference field values and variables from previous workflow nodes in your API Proxy configuration using [templating](/build/workflow/templating).
</Tip>
