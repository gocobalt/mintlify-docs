The **Certificates** page stores the TLS certificates and matching private
keys your workspace needs to authenticate against third-party systems that
require mutual TLS, SAML signing, or client-certificate authentication.
Upload each certificate once and reference it by name from any connector
that needs it.

Open it from **Settings > Organization > Certificates**.

## Upload a certificate

<Steps>
  <Step title="Open Certificates">
    Go to **Settings > Organization > Certificates**.
  </Step>
  <Step title="Click Add Certificate">
    Click **+ Add Certificate** in the top-right corner.
  </Step>
  <Step title="Pick a type">
    Select the certificate type from the list.
    {/* NEEDS INPUT: list the certificate types Refold supports (mTLS
        client, SAML signing, CA root, etc.). */}
  </Step>
  <Step title="Paste the PEM-encoded certificate">
    Paste the certificate body in PEM format. Include the
    `-----BEGIN CERTIFICATE-----` and `-----END CERTIFICATE-----` lines.
  </Step>
  <Step title="Paste the private key (if required)">
    For client certificates, paste the matching private key in PEM format.
    Refold encrypts it at rest and never returns it to the dashboard.
  </Step>
  <Step title="Name and save">
    Give the certificate a **Name** you'll reference from connectors, then
    click **Save**.
  </Step>
</Steps>

## Rotation and expiry

Refold shows each certificate's expiry date in the list. Rotate before
expiry to avoid outbound calls failing in production.

<Warning>
  Refold does not auto-rotate certificates. Set a calendar reminder, or
  use the [Audit](/v3/developer/configure/audit) log to monitor approaching
  expiry.
</Warning>
