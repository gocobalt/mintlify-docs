**Credentials** are the secrets your workspace uses to talk to third-party
systems and to other parts of Refold: OAuth client IDs and secrets, API
keys, signing keys, and webhook shared secrets. Store them once on this
page and reference them by name from any workflow or connector.

Open it from **Settings > Organization > Credentials**.

## Add a credential

<Steps>
  <Step title="Open Credentials">
    Go to **Settings > Organization > Credentials**.
  </Step>
  <Step title="Click Add Credential">
    Click **+ Add Credential** in the top-right corner.
  </Step>
  <Step title="Pick a type">
    Select the credential type from the list (for example, OAuth Client,
    API Key, or Signing Key).
    {/* NEEDS INPUT: list every credential type the workspace supports. */}
  </Step>
  <Step title="Fill in the details">
    Give the credential a **Name** you'll reference from workflows, then
    paste the secret value. Refold stores it encrypted at rest.
  </Step>
  <Step title="Save">
    Click **Save**. Refold shows the credential in the list and hides the
    secret value from view.
  </Step>
</Steps>

## Use a credential in a workflow

{/* NEEDS INPUT: provide the exact reference syntax workflows use (likely
    something like {{credentials.NAME}} or a node-level picker). */}

Reference a credential by its **Name** anywhere a workflow node accepts a
secret value. Refold injects the decrypted value at execution time. The
secret value never appears in workflow definitions or execution logs.

## Rotate or revoke

<Warning>
  Treat credentials as live secrets. Rotate immediately if a value leaks,
  and revoke credentials for any integration you stop using.
</Warning>
