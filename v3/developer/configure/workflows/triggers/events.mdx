---
title: "Events"
description: "Run a workflow from a custom event your application sends through the Refold SDK or API."
---

**Events** are custom events your application sends to Refold — through the
Refold SDK or the Event API — to start a workflow. They're typically used to
map data from your app into your users' connected apps.

For example, your app can send a `Contact Created` event to trigger a
workflow that creates a matching contact in each user's Salesforce CRM.

<Info>
  One event can trigger multiple workflows. This is useful when the same
  event should drive similar workflows across different integrations.
</Info>

## Create an event

An event has a **name** and a **sample payload**. The fields in the sample
payload become **data variables** you can reference inside a workflow as
placeholders; the real values sent when the event fires replace them at run
time.

<Steps>
  <Step title="Open Events">
    Go to your organization's **Events** settings and click **New Event**.
  </Step>
  <Step title="Name it and define the payload">
    Set the event name and a sample payload that mirrors the data you'll send.
  </Step>
  <Step title="Use the payload in a workflow">
    In a workflow's Start node, select the event. Its payload fields are
    available as data variables throughout the workflow.
  </Step>
</Steps>

## Trigger an event

Send an event with a `POST` request that includes the target
`linked_account_id` and your `x-api-key`:

```bash
curl --request POST \
  --url https://api.gocobalt.io/api/v1/event/trigger \
  --header 'Content-Type: application/json' \
  --header 'linked_account_id: <linked_account_id>' \
  --header 'x-api-key: <x-api-key>' \
  --data '{
    "event": "Contact Created",
    "payload": {
      "first_name": "Jane",
      "last_name": "Doe",
      "email": "jane@acme.com"
    }
  }'
```

The workflow runs for the given linked account with the payload values mapped
to its data variables.
