---
title: "Webhook"
description: "Run a workflow when any external system sends an HTTP request to a unique URL."
---

The **Webhook** trigger lets any external system start a workflow by sending
an HTTP `POST` request to a unique URL. Use it when the source application
has no Refold connector, or for events that an existing connector doesn't
model.

<Note>
  One webhook trigger maps to one workflow. If a source app supports only a
  single outbound webhook URL, it can't route different events to separate
  workflows.
</Note>

## Configure the trigger

Add a **Webhook** trigger in the workflow's **Start node**, then set it up:

| Field | Description |
|---|---|
| **Method** | `POST` (the only supported method). |
| **URL** | Auto-generated and unique to this trigger. Copy it from the Start node and give it to the source system. |
| **Payload** | Paste a sample JSON request body. Refold uses it to build the schema. |
| **Schema** | Derived from the sample payload and editable — set field types and mark fields as required. |
| **Validate schema** | When on, requests that don't match the schema are rejected. When off, they're accepted and the workflow runs with the raw body. |
| **Idempotency** | When on, Refold ignores duplicate requests using a key from a header or payload field. |
| **Conditions** | Optional trigger-level conditions, same as other triggers. |

## How requests are handled

When a request hits the webhook URL, Refold validates it and returns a status
code. The first failed check stops processing — the event isn't queued.

| Status | Meaning |
|---|---|
| `200 OK` | Event received and queued for processing. |
| `400 Bad Request` | Payload didn't match the schema (when **Validate schema** is on). |
| `401 Unauthorized` | Authentication failed (when configured). |
| `405 Method Not Allowed` | The request wasn't a `POST`. |
| `409 Conflict` | Duplicate request, ignored (when **Idempotency** is on). |
| `413 Payload Too Large` | Body exceeded the 1 MB limit. |
| `415 Unsupported Media Type` | Body wasn't `application/json`. |
| `429 Too Many Requests` | The per-URL rate limit was exceeded. |

Accepted events are queued and processed **asynchronously** by the workflow
engine. If you delete the trigger, the URL immediately stops accepting new
requests, but events already in the queue finish processing.

<Warning>
  Webhook triggers currently accept **JSON** payloads over `POST` only, up to
  **1 MB** per request.
</Warning>
