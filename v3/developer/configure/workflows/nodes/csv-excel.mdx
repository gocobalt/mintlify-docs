The Data Converter Node provides comprehensive file processing capabilities, allowing you to convert between different data formats (CSV, JSON, Excel, X12 EDI, cXML), merge multiple files, and generate new files from workflow data. All processed files are hosted on publicly accessible URLs for easy integration with external systems.

## When to Use

- Convert SAP or NetSuite exports from CSV to JSON for API processing
- Generate Excel reports from JSON data with formula support
- Merge multiple data files into comprehensive reports
- Parse CSV files into arrays for workflow processing
- Convert between EDI and JSON formats for B2B integrations
- Transform cXML documents for procurement system integrations

## Actions

<AccordionGroup>
  <Accordion title="CSV (file) to JSON file">
    Converts a publicly hosted CSV file to JSON format and returns a hosted JSON file URL.

    **Parameters:**

    | Field | Required | Description |
    |-------|----------|-------------|
    | File URL | Yes | Publicly accessible URL to the source CSV file |
    | Delimiter | No | Character to separate values. Default: `,` |
    | Line Ending | No | Row separator. Default: `\n` |

    **Output:** Hosted JSON file URL
  </Accordion>

  <Accordion title="JSON to CSV file">
    Converts JSON data to CSV format and returns a hosted CSV file URL.

    **Parameters:**

    | Field | Required | Description |
    |-------|----------|-------------|
    | JSON Data | Yes | JSON array of objects to convert |
    | Delimiter | No | Character to separate values. Default: `,` |
    | Line Ending | No | Row separator. Default: `\n` |
    | Wrap | No | Character to enclose values. Default: `"` |

    **Output:** Hosted CSV file URL
  </Accordion>

  <Accordion title="CSV Data to JSON">
    Parses CSV content string into an array of objects for workflow processing.

    **Parameters:**

    | Field | Required | Description |
    |-------|----------|-------------|
    | CSV Input | Yes | CSV content string to parse |
    | Delimiter | No | Character to separate values. Default: `,` |
    | Line Ending | No | Row separator. Default: `\n` |

    **Output:** Parsed JSON array (not a file URL)
  </Accordion>

  <Accordion title="Array of rows to CSV">
    Converts an array of objects to CSV content string.

    **Parameters:**

    | Field | Required | Description |
    |-------|----------|-------------|
    | Rows | Yes | Array of objects to convert to CSV |
    | Delimiter | No | Character to separate values. Default: `,` |
    | Line Ending | No | Row separator. Default: `\n` |

    **Output:** CSV content string (not a file URL)
  </Accordion>

  <Accordion title="JSON file to Excel file">
    Converts a publicly hosted JSON file to Excel format (.xlsx).

    **Parameters:**

    | Field | Required | Description |
    |-------|----------|-------------|
    | JSON File URL | Yes | Publicly accessible URL to source JSON file |
    | Has Formulas? | No | Enable formula detection for values starting with `=` |

    **Output:** Hosted Excel file URL
  </Accordion>

  <Accordion title="JSON data to Excel file">
    Converts direct JSON input to Excel format (.xlsx).

    **Parameters:**

    | Field | Required | Description |
    |-------|----------|-------------|
    | JSON Data | Yes | JSON array of objects to convert |
    | Has Formulas? | No | Enable formula detection for values starting with `=` |

    **Output:** Hosted Excel file URL
  </Accordion>

  <Accordion title="CSV to Excel file">
    Converts CSV data (URL or direct content) to Excel format (.xlsx).

    **Parameters:**

    | Field | Required | Description |
    |-------|----------|-------------|
    | CSV File URL or Data | Yes | URL to CSV file or direct CSV content |
    | Has Formulas? | No | Enable formula detection for values starting with `=` |
    | Delimiter | No | Character to separate values. Default: `,` |
    | Line Ending | No | Row separator. Default: `\n` |

    **Output:** Hosted Excel file URL
  </Accordion>

  <Accordion title="Merge Excel files">
    Merges multiple Excel files into a single Excel file.

    **Parameters:**

    | Field | Required | Description |
    |-------|----------|-------------|
    | Excel Files Array | Yes | Array of Excel file URLs to merge |

    **Output:** Hosted merged Excel file URL
  </Accordion>

  <Accordion title="Write data to file">
    Creates a file from any content with customizable format and name.

    **Parameters:**

    | Field | Required | Description |
    |-------|----------|-------------|
    | Data | Yes | Content to write (JSON, text, or any string) |
    | File Format | Yes | Output format: `json`, `txt`, `csv`, or `xlsx` |
    | File Name | Yes | Custom file name (without extension) |
    | Content Type | No | MIME type. Example: `application/json` |

    **Output:** Hosted file URL with custom name
  </Accordion>

  <Accordion title="JSON to X12 EDI format">
    Converts JSON data to X12 EDI format for B2B integrations.

    **Parameters:**

    | Field | Required | Description |
    |-------|----------|-------------|
    | JSON Data | Yes | JSON object containing EDI transaction data |
    | Element Delimiter | No | Separator for elements within segments. Default: `*` |
    | Segment Terminator | No | Character to terminate segments. Default: `~` |

    **Output:** Hosted EDI file URL
  </Accordion>

  <Accordion title="X12 EDI format to JSON">
    Converts X12 EDI format data to JSON for processing.

    **Parameters:**

    | Field | Required | Description |
    |-------|----------|-------------|
    | X12 EDI Data or File URL | Yes | EDI content string or URL to EDI file |

    **Output:** Parsed JSON object
  </Accordion>

  <Accordion title="cXML to JSON">
    Converts cXML data to JSON format for procurement integrations.

    **Parameters:**

    | Field | Required | Description |
    |-------|----------|-------------|
    | cXML Data or File URL | Yes | cXML content string or URL to cXML file |

    **Output:** Parsed JSON object
  </Accordion>

  <Accordion title="JSON to cXML">
    Converts JSON data to cXML format for procurement system integrations.

    **Parameters:**

    | Field | Required | Description |
    |-------|----------|-------------|
    | JSON Data | Yes | JSON object to convert to cXML |
    | Root Element Name | No | Custom root element name for the cXML document |

    **Output:** Hosted cXML file URL
  </Accordion>
</AccordionGroup>

## Common Parameters

The following parameters appear across multiple actions and control file storage settings.

<AccordionGroup>
  <Accordion title="Bucket Object Settings">
    These parameters control file storage and URL generation for output files.

    | Field | Description |
    |-------|-------------|
    | Infinite Expiry | When enabled, generated file URLs do not expire. Default: `false` |
    | TTL | Time-to-live in seconds for the generated file URL. Only applies when Infinite Expiry is disabled. |
    | Preserve Cloud Provider Presigned URL | Preserves the original cloud provider presigned URL format. Default: `false` |
  </Accordion>

  <Accordion title="Custom Output">
    Define custom fields to include in the node output. Click **Add Field** to add key-value pairs that will be included alongside the default response.

    Available in: CSV Data to JSON, Array of rows to CSV, JSON data to Excel file, Merge Excel files, X12 EDI format to JSON, cXML to JSON
  </Accordion>
</AccordionGroup>

## Output

<AccordionGroup>
  <Accordion title="File URL Output">
    Actions that generate hosted files return:
```json
    {
      "file_url": "https://storage.gocobalt.io/files/abc123/vendors.json",
      "file_name": "vendors.json",
      "file_size": 2048,
      "format": "JSON",
      "created_at": "2024-01-15T14:30:00Z"
    }
```

    **Actions:** CSV (file) to JSON file, JSON to CSV file, JSON file to Excel file, JSON data to Excel file, CSV to Excel file, Merge Excel files, Write data to file, JSON to X12 EDI format, JSON to cXML
  </Accordion>

  <Accordion title="Parsed Data Output">
    Actions that parse data return the content directly:
```json
    {
      "parsed_data": [
        {
          "vendor_id": "V001234",
          "name": "Acme Corporation",
          "country": "US"
        }
      ],
      "total_rows": 1,
      "columns": ["vendor_id", "name", "country"]
    }
```

    **Actions:** CSV Data to JSON, X12 EDI format to JSON, cXML to JSON
  </Accordion>

  <Accordion title="Content String Output">
    Actions that generate content strings return:
```json
    {
      "csv_content": "vendor_id,name,country\nV001234,Acme Corporation,US",
      "row_count": 1,
      "column_count": 3
    }
```

    **Actions:** Array of rows to CSV
  </Accordion>
</AccordionGroup>

## Adding to Your Workflow

<Steps>
  <Step title="Add the node">
    Click **Add Node** in the workflow canvas, select **Data Converter** from the list of available nodes, then position it in your workflow.
  </Step>
  <Step title="Select processing action">
    Choose the action that matches your conversion needs from the dropdown list.
  </Step>
  <Step title="Configure parameters">
    Provide the required input data and adjust optional parameters as needed.
  </Step>
  <Step title="Test the node">
    Run the node to verify the output format and file accessibility.
  </Step>
</Steps>

<Info>Generated file URLs are publicly accessible and can be used immediately in subsequent workflow steps or external integrations.</Info>

## Troubleshooting

<AccordionGroup>
  <Accordion title="File URL Issues">
    | Problem | Solution |
    |---------|----------|
    | File URL not accessible | Verify URL is publicly accessible without authentication. Check file exists and uses HTTPS. |
    | File URL expired | Enable Infinite Expiry or increase TTL in Bucket Object Settings. |
  </Accordion>

  <Accordion title="CSV Processing Issues">
    | Problem | Solution |
    |---------|----------|
    | CSV parsing errors | Verify Delimiter matches the actual separator in your data. Check Line Ending format. |
    | Missing columns | Ensure first row contains headers. Check for inconsistent column counts. |
  </Accordion>

  <Accordion title="Excel Processing Issues">
    | Problem | Solution |
    |---------|----------|
    | Excel formula errors | Verify formula syntax matches Excel standards. Ensure Has Formulas? is enabled. |
    | Merge operation failures | Verify all source Excel files are accessible and valid .xlsx format. |
  </Accordion>

  <Accordion title="EDI/cXML Issues">
    | Problem | Solution |
    |---------|----------|
    | EDI conversion errors | Verify JSON structure matches expected EDI transaction format. Check Element Delimiter and Segment Terminator. |
    | cXML parsing failures | Verify cXML is well-formed XML. Check document type matches expected schema. |
  </Accordion>
</AccordionGroup>
