The Send Email Node sends emails directly from your workflow using SMTP. It supports dynamic content from previous nodes, multiple recipients, and file attachments. Use it for internal notifications, alerts, and automated communications triggered by workflow events.

## When to Use

- Send notifications when workflow processes complete or fail
- Alert teams about sync errors or data validation issues
- Deliver automated reports with dynamic data
- Notify stakeholders when records are created or updated in enterprise systems
- Send confirmation emails after payment processing

<Info>This node uses SMTP for email delivery. Rate limiting and deliverability depend on your SMTP provider settings.</Info>

## Input Parameters

### SMTP Configuration

<ParamField path="Use Organization SMTP Settings" type="boolean">
  Enable to use SMTP credentials configured at the organization level under **Organization > Settings > Email**. When disabled, configure SMTP settings directly in this node.
</ParamField>

<ParamField path="SMTP Host" type="string">
  SMTP server hostname. Example: `smtp.gmail.com`, `smtp.office365.com`
</ParamField>

<ParamField path="SMTP Port" type="number">
  SMTP server port. Common values: `25`, `465` (SSL), `587` (TLS). Example: `587`
</ParamField>

<ParamField path="SMTP Username" type="string">
  Authentication username for the SMTP server. Example: `notifications@company.com`
</ParamField>

<ParamField path="SMTP Password" type="string">
  Authentication password for the SMTP server. This field is masked for security.
</ParamField>

<ParamField path="TLS Encryption" type="boolean">
  Enable TLS encryption for secure email transmission. Recommended for port 587.
</ParamField>

### Email Configuration

<ParamField path="From Address" type="string" required>
  Sender email address. Supports dynamic variables. Example: `notifications@company.com` or `{{config.from_email}}`
</ParamField>

<ParamField path="To Address" type="string" required>
  Recipient email addresses. Separate multiple addresses with commas. Supports dynamic variables. Example: `team@company.com, team2@company.com` or `{{contact.email}}`
</ParamField>

<ParamField path="CC Address" type="string">
  Carbon copy recipients. Separate multiple addresses with commas. Example: `manager@company.com, admin@company.com`
</ParamField>

<ParamField path="BCC Address" type="string">
  Blind carbon copy recipients. Separate multiple addresses with commas. Example: `archive@company.com`
</ParamField>

<ParamField path="Subject" type="string" required>
  Email subject line. Supports dynamic variables. Example: `SAP Purchase Order Created - {{po_number}}`
</ParamField>

<ParamField path="Body" type="text area" required>
  Email body content. Supports plain text, HTML, and dynamic variables. Example: `Purchase order {{po_number}} has been created for vendor {{vendor_name}}.`
</ParamField>

<ParamField path="Attachments" type="code editor">
  File attachments to include with the email.
</ParamField>

## Output

The node returns email delivery status and metadata.
```json
{
  "email_sent": true,
  "message_id": "msg_12345abc",
  "recipients": {
    "to": ["team@company.com"],
    "cc": ["manager@company.com"],
    "bcc": ["archive@company.com"]
  },
  "subject": "SAP Purchase Order Created - PO-2024-001",
  "sent_at": "2024-01-15T10:30:00Z",
  "attachments_processed": 1,
  "smtp_host": "smtp.gmail.com"
}
```

## Adding to Your Workflow

<Steps>
  <Step title="Add the Send Email Node">
    In the workflow editor, click **Add Node** and select **Send Email** from the Utility Nodes section.
  </Step>
  
  <Step title="Configure SMTP settings">
    Either enable **Use Organization SMTP Settings** to use preconfigured credentials, or enter SMTP details directly:
    - SMTP Host and Port
    - Username and Password
    - TLS Encryption (recommended)
  </Step>
  
  <Step title="Set email addresses">
    Configure the sender and recipients:
    - **From Address**: Your sending email address
    - **To Address**: Primary recipients (comma-separated for multiple)
    - **CC/BCC**: Optional additional recipients
  </Step>
  
  <Step title="Compose email content">
    Enter the subject and body. Use `{{variable}}` syntax to include dynamic data from previous nodes.
  </Step>
  
  <Step title="Test the node">
    Click the **Run** tab to send a test email and verify delivery.
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="SMTP Connection Errors">
    | Problem | Solution |
    |---------|----------|
    | Connection refused | Verify SMTP host and port are correct. Common ports: 25, 465, 587. |
    | Connection timeout | Check network connectivity. Verify SMTP server is accessible from your environment. |
    | TLS handshake failed | Enable or disable TLS Encryption based on your SMTP provider requirements. |
  </Accordion>

  <Accordion title="Authentication Errors">
    | Problem | Solution |
    |---------|----------|
    | Authentication failed | Verify SMTP username and password are correct. |
    | App password required | Some providers (Gmail, Microsoft 365) require app-specific passwords instead of account passwords. |
    | Account locked | Check if too many failed attempts triggered a security lock on your SMTP account. |
  </Accordion>

  <Accordion title="Delivery Issues">
    | Problem | Solution |
    |---------|----------|
    | Email not received | Check recipient spam/junk folders. Verify email addresses are valid. |
    | Bounced emails | Confirm recipient addresses exist and accept external email. |
    | Rate limited | Check SMTP provider sending limits. Spread bulk sends across multiple workflow executions. |
  </Accordion>

  <Accordion title="Content Issues">
    | Problem | Solution |
    |---------|----------|
    | Variables not resolved | Verify variable syntax uses double curly braces: `{{node_name.field}}` |
    | HTML not rendering | Ensure email client supports HTML. Some recipients may have HTML disabled. |
    | Missing recipients | Verify comma separation for multiple addresses with no extra spaces. |
  </Accordion>
</AccordionGroup>

<Tip>Configure organization-level SMTP settings under **Organization > Settings > Email** to avoid repeating SMTP configuration in every workflow.</Tip>
