The Transform by JSONata Node enables advanced JSON data transformation and querying using JSONata expressions. It allows you to restructure, extract, filter, and format data from complex JSON objects or arrays, making it ideal for preparing data for external APIs or internal processing.

## When to Use

- Simplify complex objects from SAP or NetSuite into flat structures
- Extract specific fields from nested API responses
- Format dates from ISO format to custom formats
- Combine multiple fields into computed values
- Filter and reshape arrays of records
- Map fields between different data structures

<Tip>To learn about JSONata expressions, refer to [jsonata.org](https://jsonata.org/).</Tip>

## Input Parameters

<ParamField path="Input" type="object" required>
  JSON object containing the data to transform. Can reference variables from previous nodes or DataSlots. The input must be wrapped in an object with a key name of your choice.
</ParamField>

<ParamField path="Query" type="object" required>
  JSONata expression wrapped in an object with the key `output`. The expression transforms or queries the input data.
</ParamField>

<Warning>Both Input and Query fields are required. The Query must use `output` as the key name, otherwise the node returns an empty response that cannot be used in subsequent nodes.</Warning>

**Input Format:**
```javascript
{
  "input": "{Array_name}"           // When providing an array directly
  "input": "{{Node_name.data}}"     // When referencing a previous node
}
```

**Query Format:**
```javascript
{
  "output": "JSONata expression"    // Key must be "output"
}
```

## Output

The Transform Node outputs the result of the JSONata expression applied to the input data.
```json
{
  "output": {
    "vendor_name": "Acme Corporation",
    "vendor_id": "V001234",
    "formatted_date": "2024-01-15"
  }
}
```

## Adding to Your Workflow

<Steps>
  <Step title="Add the node">
    Click **Add Node** in the workflow canvas, select **Transform** from the list of available nodes, then position it in your workflow.
  </Step>
  <Step title="Configure input data">
    Provide the JSON object or array to transform. Reference data from previous nodes using variable syntax or provide static JSON.
  </Step>
  <Step title="Write JSONata query">
    Create your transformation expression using JSONata syntax. Wrap the expression in an object with `output` as the key.
  </Step>
  <Step title="Test the node">
    Run the node to verify the JSONata expression produces the expected output structure.
  </Step>
</Steps>

## JSONata Reference

<AccordionGroup>
  <Accordion title="Data Access">
    | Syntax | Description | Example |
    |--------|-------------|---------|
    | Field Access | Access object properties | `vendor.name`, `address.city` |
    | Array Access | Access array elements | `items[0]`, `records[-1]` |
    | Conditional Access | Default values for missing fields | `vendor.phone ? vendor.phone : "N/A"` |
    | Wildcard | Access all elements | `*.name`, `items.*.id` |
  </Accordion>

  <Accordion title="String Functions">
    | Function | Description | Example |
    |----------|-------------|---------|
    | `$uppercase()` | Convert to uppercase | `$uppercase(vendor.name)` |
    | `$lowercase()` | Convert to lowercase | `$lowercase(status)` |
    | `$substring()` | Extract substring | `$substring(id, 0, 5)` |
    | `$trim()` | Remove whitespace | `$trim(description)` |
    | `$replace()` | Replace text | `$replace(phone, "-", "")` |
    | `&` | Concatenate strings | `FirstName & " " & LastName` |
  </Accordion>

  <Accordion title="Date Functions">
    | Function | Description | Example |
    |----------|-------------|---------|
    | `$now()` | Current timestamp | `$now()` |
    | `$toMillis()` | Convert to milliseconds | `$toMillis(created_at)` |
    | `$fromMillis()` | Format from milliseconds | `$fromMillis($toMillis(date), "[Y0001]-[M01]-[D01]")` |
  </Accordion>

  <Accordion title="Array Functions">
    | Function | Description | Example |
    |----------|-------------|---------|
    | `$map()` | Transform each element | `$map(items, function($v){ $v.name })` |
    | `$filter()` | Filter elements | `$filter(items, function($v){ $v.status = "active" })` |
    | `$reduce()` | Reduce to single value | `$reduce(items, function($acc, $v){ $acc + $v.amount }, 0)` |
    | `$count()` | Count elements | `$count(records)` |
    | `$sum()` | Sum numeric values | `$sum(items.price)` |
    | `$average()` | Calculate average | `$average(scores)` |
  </Accordion>

  <Accordion title="Operators">
    | Operator | Description | Example |
    |----------|-------------|---------|
    | `+`, `-`, `*`, `/` | Arithmetic | `price * quantity` |
    | `=`, `!=` | Equality | `status = "active"` |
    | `>`, `<`, `>=`, `<=` | Comparison | `amount > 1000` |
    | `and`, `or` | Logical | `status = "active" and amount > 100` |
    | `?` | Conditional | `score > 80 ? "high" : "low"` |
  </Accordion>
</AccordionGroup>

## Examples

<AccordionGroup>
  <Accordion title="Flatten SAP Vendor Response">
    Transform nested SAP vendor data into a flat structure for internal processing.

    **Input:**
```json
    {
      "input": "{{sap_vendor_response}}"
    }
```

    **Sample Input Data:**
```json
    {
      "vendor": {
        "LIFNR": "V001234",
        "NAME1": "Acme Corporation",
        "LAND1": "US",
        "ORT01": "Chicago",
        "PSTLZ": "60601"
      },
      "metadata": {
        "ERDAT": "2024-01-15T10:30:00Z",
        "AEDAT": "2024-01-20T14:45:00Z"
      }
    }
```

    **Query:**
```json
    {
      "output": {
        "vendor_id": input.vendor.LIFNR,
        "name": input.vendor.NAME1,
        "location": input.vendor.ORT01 & ", " & input.vendor.LAND1,
        "created_date": $fromMillis($toMillis(input.metadata.ERDAT), "[Y0001]-[M01]-[D01]")
      }
    }
```

    **Output:**
```json
    {
      "output": {
        "vendor_id": "V001234",
        "name": "Acme Corporation",
        "location": "Chicago, US",
        "created_date": "2024-01-15"
      }
    }
```
  </Accordion>

  <Accordion title="Map NetSuite Records to Name-Value Pairs">
    Convert NetSuite records into name-value pairs for Config DataSource dropdowns.

    **Input:**
```json
    {
      "records": "{{netsuite_vendors.records}}"
    }
```

    **Query:**
```json
    {
      "output": $map(records, function($v){
        {
          "name": $v.companyName,
          "value": $v.internalId
        }
      })
    }
```

    **Output:**
```json
    {
      "output": [
        { "name": "Acme Corporation", "value": "12345" },
        { "name": "TechSupply Inc", "value": "12346" },
        { "name": "Global Parts Ltd", "value": "12347" }
      ]
    }
```
  </Accordion>

  <Accordion title="Filter and Transform Order Array">
    Filter active orders and extract specific fields with formatted dates.

    **Input:**
```json
    {
      "orders": "{{netsuite_orders.records}}"
    }
```

    **Sample Input Data:**
```json
    [
      { "id": "SO-001", "customer": "Acme Corp", "status": "active", "total": 5000, "date": "2024-01-15T10:00:00Z" },
      { "id": "SO-002", "customer": "TechSupply", "status": "closed", "total": 3000, "date": "2024-01-10T09:00:00Z" },
      { "id": "SO-003", "customer": "Global Ltd", "status": "active", "total": 7500, "date": "2024-01-20T11:00:00Z" }
    ]
```

    **Query:**
```json
    {
      "output": $map($filter(orders, function($o){ $o.status = "active" }), function($order){
        {
          "order_id": $order.id,
          "customer": $order.customer,
          "amount": $order.total,
          "order_date": $fromMillis($toMillis($order.date), "[Y0001]-[M01]-[D01]")
        }
      })
    }
```

    **Output:**
```json
    {
      "output": [
        { "order_id": "SO-001", "customer": "Acme Corp", "amount": 5000, "order_date": "2024-01-15" },
        { "order_id": "SO-003", "customer": "Global Ltd", "amount": 7500, "order_date": "2024-01-20" }
      ]
    }
```
  </Accordion>

  <Accordion title="Create Nested Structure from Flat Data">
    Restructure flat Tipalti payment data into a nested format for reporting.

    **Input:**
```json
    {
      "payment": "{{tipalti_payment}}"
    }
```

    **Sample Input Data:**
```json
    {
      "payee_id": "P001",
      "payee_name": "Vendor ABC",
      "payee_email": "ap@vendorabc.com",
      "amount": 15000,
      "currency": "USD",
      "status": "completed",
      "submitted_date": "2024-01-15T10:00:00Z",
      "completed_date": "2024-01-17T14:30:00Z"
    }
```

    **Query:**
```json
    {
      "output": {
        "payee": {
          "id": payment.payee_id,
          "name": payment.payee_name,
          "email": payment.payee_email
        },
        "transaction": {
          "amount": payment.amount,
          "currency": payment.currency,
          "status": payment.status
        },
        "dates": {
          "submitted": $fromMillis($toMillis(payment.submitted_date), "[Y0001]-[M01]-[D01]"),
          "completed": $fromMillis($toMillis(payment.completed_date), "[Y0001]-[M01]-[D01]")
        }
      }
    }
```

    **Output:**
```json
    {
      "output": {
        "payee": {
          "id": "P001",
          "name": "Vendor ABC",
          "email": "ap@vendorabc.com"
        },
        "transaction": {
          "amount": 15000,
          "currency": "USD",
          "status": "completed"
        },
        "dates": {
          "submitted": "2024-01-15",
          "completed": "2024-01-17"
        }
      }
    }
```
  </Accordion>

  <Accordion title="Calculate Aggregates from Array">
    Calculate summary statistics from an array of transaction records.

    **Input:**
```json
    {
      "transactions": "{{netsuite_transactions.records}}"
    }
```

    **Query:**
```json
    {
      "output": {
        "total_count": $count(transactions),
        "total_amount": $sum(transactions.amount),
        "average_amount": $round($average(transactions.amount), 2),
        "max_amount": $max(transactions.amount),
        "min_amount": $min(transactions.amount)
      }
    }
```

    **Output:**
```json
    {
      "output": {
        "total_count": 25,
        "total_amount": 125000,
        "average_amount": 5000,
        "max_amount": 15000,
        "min_amount": 500
      }
    }
```
  </Accordion>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Empty or Null Output">
    **Problem:** Transform Node returns empty response

    | Cause | Solution |
    |-------|----------|
    | Query not wrapped in object | Ensure Query uses `{"output": "expression"}` format |
    | Wrong key name in Query | Use `output` as the key name, not other names |
    | Input field missing | Verify Input field contains valid JSON data |
  </Accordion>

  <Accordion title="JSONata Syntax Errors">
    **Problem:** Node returns syntax error instead of transformed data

    | Cause | Solution |
    |-------|----------|
    | Missing quotes or brackets | Check for balanced quotes and brackets |
    | Invalid function name | Verify function names are correct (case-sensitive) |
    | Wrong operator | Use `=` for equality, `&` for concatenation |
    | Test in playground | Use [jsonata.org](https://jsonata.org/) to validate syntax |
  </Accordion>

  <Accordion title="Missing Field References">
    **Problem:** Query returns null for expected values

    | Cause | Solution |
    |-------|----------|
    | Field name mismatch | Field names are case-sensitive, verify exact names |
    | Missing parent object | Use full path: `input.vendor.name` not just `vendor.name` |
    | Field does not exist | Add conditional: `field ? field : "default"` |
    | Use Debug Node | Inspect actual input structure before Transform |
  </Accordion>

  <Accordion title="Date Formatting Issues">
    **Problem:** Date functions produce unexpected results

    | Cause | Solution |
    |-------|----------|
    | Invalid date format | Ensure input dates are valid ISO format |
    | Missing toMillis | Always convert to milliseconds first: `$toMillis(date)` |
    | Wrong format string | Use `[Y0001]-[M01]-[D01]` for YYYY-MM-DD |
  </Accordion>

  <Accordion title="Array Processing Errors">
    **Problem:** Array functions fail or return unexpected results

    | Cause | Solution |
    |-------|----------|
    | Input not an array | Verify input is actually an array with `$type()` |
    | Function syntax error | Check function signature matches JSONata spec |
    | Empty array | Add check: `$count(array) > 0 ? $map(...) : []` |
  </Accordion>
</AccordionGroup>

<Tip>Use the [JSONata online playground](https://try.jsonata.org/) to test and debug your expressions before adding them to the Transform Node.</Tip>
