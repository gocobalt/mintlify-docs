The Human Intervention Node pauses workflow execution and sends an email containing a link to a customizable form. The workflow remains paused until a recipient submits the form or the timeout period expires. This enables approval workflows, manual data entry, and human decision points within automated processes.

## When to Use

- Require manager approval before processing large transactions
- Collect missing data from users during workflow execution
- Enable manual review checkpoints for compliance workflows
- Request human decisions at critical workflow branch points
- Gather feedback or confirmation before completing integrations

<Info>Forms must be created in **More > User Forms** before they can be used in this node. Each form can include Text, Number, Date, Dropdown, and Radio input fields.</Info>

## Input Parameters

### SMTP Configuration

You can either use organization-level SMTP settings or configure custom SMTP credentials for sending the form email.

<ParamField path="Use Org SMTP" type="toggle">
  Enable to use SMTP settings configured at the organization level. When enabled, the SMTP Settings section is hidden.
</ParamField>

<AccordionGroup>
  <Accordion title="SMTP Settings (Custom Configuration)">
    Configure custom SMTP credentials when **Use Org SMTP** is disabled.

    <ParamField path="SMTP Host" type="string">
      Hostname or IP address of your SMTP server.
    </ParamField>

    <ParamField path="SMTP Port" type="number">
      Port number for the SMTP server. Common ports: `587` (TLS), `465` (SSL), `25` (unencrypted).
    </ParamField>

    <ParamField path="SMTP Username" type="string">
      Username for SMTP server authentication.
    </ParamField>

    <ParamField path="SMTP Password" type="string">
      Password for SMTP server authentication.
    </ParamField>

    <ParamField path="Use TLS Encryption" type="toggle">
      Enable TLS encryption for secure email transmission.
    </ParamField>
  </Accordion>
</AccordionGroup>

### Form and Email Configuration

<ParamField path="Forms" type="dropdown" required>
  Select the form to send to recipients. Forms are created and managed in **More > User Forms**.
</ParamField>

<ParamField path="Sender Email" type="string" required>
  Email address that appears as the sender. Must be a valid address authorized by your SMTP server.
</ParamField>

<ParamField path="Receiver Emails" type="string" required>
  Comma-separated list of recipient email addresses. If multiple recipients are specified, only the first person to submit the form will complete the task.
</ParamField>

<ParamField path="Subject" type="string" required>
  Email subject line. Supports dynamic variables using `{{variable}}` syntax.
</ParamField>

<ParamField path="Body" type="textarea" required>
  Email body content. Use `[[form_url]]` to insert the form link. Supports dynamic variables using `{{variable}}` syntax.
</ParamField>

<ParamField path="Timeout" type="number" required>
  Time in minutes within which the form must be submitted. Minimum value: `1`. If the timeout expires before submission, the form is marked as timed out and the workflow continues.
</ParamField>

<Warning>The `[[form_url]]` placeholder must be included in the Body field. Without it, recipients will not receive the link to submit the form.</Warning>

## Output

The node returns the form submission data when completed successfully.

**Success Response:**
```json
{
  "status": "Success",
  "node_id": "12",
  "node_name": "Human Intervention",
  "body": {
    "approval_decision": "approved",
    "reviewer_comments": "All details verified, approved for processing",
    "reviewed_amount": 75000
  }
}
```

**Timeout Response:**
```json
{
  "status": "Success",
  "node_id": "12",
  "node_name": "Human Intervention",
  "body": {
    "form_status": "timed_out"
  }
}
```

**Error Response:**
```json
{
  "status": "Errored",
  "node_id": "12",
  "node_name": "Human Intervention",
  "body": "SMTP authentication failed"
}
```

## Adding to Your Workflow

<Steps>
  <Step title="Create a form">
    Navigate to **More > User Forms** and create a new form. Add input fields (Text, Number, Date, Dropdown, Radio) that capture the information you need from reviewers.
  </Step>
  
  <Step title="Add the Human Intervention Node">
    In the workflow editor, click **Add Node** and select **Human Intervention** from the Utility Nodes section.
  </Step>
  
  <Step title="Configure SMTP settings">
    Enable **Use Org SMTP** to use organization-level settings, or expand **SMTP Settings** to configure custom credentials.
  </Step>
  
  <Step title="Select the form">
    Choose the form you created from the **Forms** dropdown.
  </Step>
  
  <Step title="Configure email details">
    Enter the **Sender Email**, **Receiver Emails**, **Subject**, and **Body**. Include `[[form_url]]` in the body to insert the form link.
  </Step>
  
  <Step title="Set timeout">
    Enter the **Timeout** value in minutes. The workflow will continue after this period if no submission is received.
  </Step>
  
  <Step title="Handle the response">
    Add subsequent nodes to process the form submission data. Use a [Rule Node](/developer-guide/workflows/nodes/rule) to branch based on approval decisions or timeout status.
  </Step>
</Steps>

## Examples

<AccordionGroup>
  <Accordion title="SAP Purchase Order Approval">
    Require manager approval for SAP purchase orders above a threshold before syncing to the ERP system.

    **Form Configuration (User Forms):**

    | Field Name | Type | Options |
    |------------|------|---------|
    | Approval Decision | Dropdown | Approve, Reject, Request Changes |
    | Reviewer Comments | Text | - |
    | Approved Amount | Number | - |

    **Node Configuration:**

    | Field | Value |
    |-------|-------|
    | Forms | `PO Approval Form` |
    | Sender Email | `workflows@yourcompany.com` |
    | Receiver Emails | `{{sap_po.approver_email}}` |
    | Subject | `Approval Required: SAP PO #{{sap_po.PONumber}} - {{sap_po.VendorName}}` |
    | Timeout | `1440` (24 hours) |

    **Body:**
```
    A new purchase order requires your approval.

    PO Number: {{sap_po.PONumber}}
    Vendor: {{sap_po.VendorName}}
    Total Amount: ${{sap_po.TotalAmount}}
    
    Please review and submit your decision: [[form_url]]
```

    **Post-Processing:**
    Use a Rule Node to check `{{human_intervention.body.approval_decision}}` and route to either the SAP sync node or a rejection notification.
  </Accordion>

  <Accordion title="NetSuite Journal Entry Review">

    | Field | Value |

  <Accordion title="Handling Timeout with Escalation">
    Configure a workflow that escalates to a backup approver when the primary approver times out.

    **Workflow Pattern:**

    1. **First Human Intervention Node** - Send to primary approver with 24-hour timeout
    2. **Rule Node** - Check if `{{node.body.form_status}}` equals `timed_out`
    3. **Second Human Intervention Node** (on timeout branch) - Send to backup approver or manager

    **First Node Timeout Check (Rule Node):**

    | Condition | Value |
    |-----------|-------|
    | Field | `{{human_intervention_1.body.form_status}}` |
    | Operator | `equals` |
    | Value | `timed_out` |

    This pattern ensures critical approvals are never blocked by unresponsive reviewers.
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Email Delivery Issues">
    | Problem | Solution |
    |---------|----------|
    | Email not received | Verify SMTP credentials are correct. Check spam/junk folders. Confirm sender email is authorized by the SMTP server. |
    | SMTP authentication failed | Verify username and password. Some providers require app-specific passwords. Check if 2FA is blocking the connection. |
    | TLS connection errors | Ensure **Use TLS Encryption** matches your SMTP server requirements. Verify the port number is correct for TLS (typically 587). |
    | Emails going to spam | Configure SPF, DKIM, and DMARC records for your sending domain. Use a verified sender address. |
  </Accordion>

  <Accordion title="Form Submission Issues">
    | Problem | Solution |
    |---------|----------|
    | Form link not in email | Ensure `[[form_url]]` is included in the Body field. The placeholder is case-sensitive. |
    | Form shows "expired" message | The timeout period has elapsed. Increase the **Timeout** value or ensure recipients respond promptly. |
    | Multiple people trying to submit | Only the first submission is accepted. Subsequent recipients will see a "form already submitted" message. |
    | Form fields not appearing | Verify the form was saved correctly in **More > User Forms**. Check that fields were added to the form canvas. |
  </Accordion>

  <Accordion title="Workflow Continuation Issues">
    | Problem | Solution |
    |---------|----------|
    | Workflow stuck waiting | Check if the timeout has been reached. Verify the form was submitted by checking **Logs > Executions**. |
    | Form data not accessible | Reference form fields using `{{node_name.body.field_name}}` syntax. Verify field names match exactly as defined in the form. |
    | Timeout not triggering | Confirm the **Timeout** value is in minutes. Minimum value is 1 minute. |
  </Accordion>

  <Accordion title="SMTP Configuration Issues">
    | Problem | Solution |
    |---------|----------|
    | Org SMTP not working | Verify SMTP settings are configured in **Organization > Settings**. Check that the Org SMTP is properly authenticated. |
    | Connection refused | Verify the SMTP host and port are correct. Check firewall rules. Confirm the SMTP server is accessible from Refold's infrastructure. |
    | Sender address rejected | Use an email address that belongs to a domain authorized in your SMTP server configuration. |
  </Accordion>
</AccordionGroup>

<Tip>Create dedicated forms for different approval types rather than reusing generic forms. This keeps form fields relevant and improves completion rates.</Tip>
