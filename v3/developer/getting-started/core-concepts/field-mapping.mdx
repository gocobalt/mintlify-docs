Field mapping allows your customers to define how fields in your application map to fields in their connected third-party apps. This is essential because different systems use different field names and structures for the same data. Your "Company Name" field might correspond to "Account Name" in Salesforce or "CompanyName" in NetSuite.

## How Field Mapping Works

You define the **source fields** (your application's fields) and the **destination fields** (the third-party app's fields). Your customers then create the mapping between the two through a configuration interface. This mapping is stored per Linked Account, so each customer can have their own field mapping tailored to their specific setup.

<Frame>
  <img src="/images/concepts/field-mapping-config.png" alt="Configuration panel showing a Map field type with source column containing your app fields and destination column containing third-party app object fields, with a customer mapping Name to Full Name and Email to Primary Email" />
</Frame>

## Static vs. Dynamic Mapping

Refold supports two approaches to field mapping.

**Static mapping** uses a predefined set of fields that you configure in advance. This works when you know exactly which fields need to be mapped.

**Dynamic mapping** fetches fields at runtime from the third-party application. This is useful when the available fields vary per customer account, such as custom fields in a CRM or ERP system.

| Approach | Best for |
|----------|----------|
| **Static** | Known, fixed fields that are the same for every customer |
| **Dynamic** | Custom fields or fields that vary across customer accounts |

<Tip>
Dynamic mapping is particularly valuable for enterprise integrations where customers often have heavily customized schemas in systems like SAP, NetSuite, or Tipalti.
</Tip>

## Where Mapping Fits in a Workflow

Field mappings are typically consumed by a **Data Mapper** node within a workflow. When a workflow runs for a specific Linked Account, the Data Mapper reads that customer's field mapping configuration and transforms the data accordingly before sending it to the destination system.
