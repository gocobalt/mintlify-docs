This page defines the core terms you will encounter throughout the Refold documentation. Each term links to its detailed guide where applicable.

## Platform

| Term | Definition |
|------|-----------|
| **Application** | A third-party service you integrate with through Refold, such as Salesforce, SAP, or NetSuite. Refold provides 300+ pre-built applications. [Learn more](/v3/developer/getting-started/core-concepts/applications-connectors) |
| **Custom Application** | An application you build yourself in Refold when a pre-built connector is not available. You can create one from scratch, from an OpenAPI spec, or using AI-assisted generation. |
| **Linked Account** | A representation of one of your end users in Refold. Stores their credentials, configurations, and execution history. All workflow runs are scoped to a Linked Account. [Learn more](/v3/developer/getting-started/core-concepts/linked-accounts) |
| **Workflow** | A series of automated steps (a directed acyclic graph) that define how data moves between your application and third-party services. [Learn more](/v3/developer/getting-started/core-concepts/workflows) |
| **Orchestration** | A group of related workflows organized under a single application. Each orchestration handles the integration logic for one third-party app. |

## Workflows

| Term | Definition |
|------|-----------|
| **Trigger** | The starting condition that launches a workflow. Types include Event, Webhook, Schedule, and Manual/API. [Learn more](/v3/developer/getting-started/core-concepts/triggers-actions) |
| **Node** | A single step in a workflow that performs an action, such as making an HTTP request, transforming data, or evaluating a condition. |
| **Event** | A signal from your application (e.g., "Contact Created") that triggers a workflow through the Refold Events API. |
| **Execution** | A single run of a workflow for a specific Linked Account, including all node outputs and logs. |
| **Variable** | A reference to dynamic data within a workflow, such as trigger payloads, node outputs, or configuration values. |

## Data

| Term | Definition |
|------|-----------|
| **Field Mapping** | The process of defining how fields in your application correspond to fields in a third-party system. [Learn more](/v3/developer/getting-started/core-concepts/field-mapping) |
| **Static Mapping** | Field mappings where you predefine the source fields. Your customers select which destination fields to map them to. |
| **Dynamic Mapping** | Field mappings where both source and destination fields are loaded at runtime from the connected application. |
| **Data Mapper Node** | A workflow node that applies field mappings during execution, transforming data from source format to destination format. |
| **Config Fields** | Custom settings you define for an application or workflow that your customers can personalize, such as selecting which object types to sync. |
| **UDF (User Defined Fields)** | Custom metadata you attach to a Linked Account for use in workflows. |

## Authentication

| Term | Definition |
|------|-----------|
| **API Key** | Your Refold API key used to authenticate server-side requests. Found in Settings → Credentials. |
| **Session Token** | A short-lived token generated by your backend, passed to your frontend to initialize the Connect SDK securely. |
| **OAuth** | An authorization framework used by many third-party apps. Refold manages the full OAuth flow on behalf of your customers. |
| **Connection** | A stored set of credentials (OAuth tokens or API keys) linking a Linked Account to a specific third-party application. |

## Infrastructure

| Term | Definition |
|------|-----------|
| **Webhook** | An HTTP callback that delivers real-time notifications when events occur, either from third-party apps into Refold or from Refold to your application. |
| **API Proxy** | A feature that lets you make authenticated API calls to third-party services through Refold, using the stored credentials of a Linked Account. |
| **Schedule** | A time-based trigger that runs a workflow at a defined interval (e.g., every hour, daily at 9 AM). |
