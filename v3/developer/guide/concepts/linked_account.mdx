A linked account represents one of your end users or customers — the person who connects and uses third-party apps through your platform. Use linked accounts to track every customer's connections, executions, and API calls, and to create, configure, and monitor their integrations from the dashboard or the API.

## How linked accounts work

Each linked account holds the integration details for a single customer, so you can monitor and manage their connections independently.

At the customer level, you can track:

- **App connections** — which third-party apps the customer has connected
- **Total executions** — how many times their workflows have run
- **API calls** — their usage against your integration

<Info>
A linked account ID must be **unique** for every user. Use any stable unique value — an email address or the primary key that represents the user in your system.
</Info>

## Managing linked accounts

Create, update, and delete linked accounts from the Refold dashboard or programmatically through the API.

### Using the dashboard

<Steps>
  <Step title="Open Linked Accounts">
    In the Refold dashboard, go to **Dashboard > Linked Accounts**.
  </Step>
  <Step title="Add an account">
    Click **+ Add Account**, then enter the **Linked Account ID**, **Name**, and **UDF Parameter**.
    <Frame>
      <img height="200" src="/images/Guides/Linked-account/linked-create.png" alt="Create a single linked account" />
    </Frame>
  </Step>
  <Step title="Bulk import (optional)">
    To create many accounts at once, import a `.csv` file instead of adding them one by one.
  </Step>
</Steps>

<Tip>
{/* TODO: link to the v3 UDF parameters page once it exists (legacy: docs.gocobalt.io/guides/platform/linked-account/udf) */}
UDF (user-defined fields) let you attach your own metadata to a linked account, such as a phone number or address.
</Tip>

### Using the API

Create a linked account with a `POST` request to the [Create linked account](/v3/api-reference/linked-accounts/create-linked-account) endpoint.

<Note>
Linked account operations should always run on the server side, never from client-side code.
</Note>

<CodeGroup>
```bash cURL
curl --request POST \
  --url https://api.gocobalt.io/api/v2/public/linked-account \
  --header 'Content-Type: application/json' \
  --header 'x-api-key: <api-key>' \
  --data '{
    "linked_account_id": "<string>",
    "name": "<string>",
    "your_app": {}
  }'
```

```javascript SDK
try {
  await Client.createLinkedAccount({
    linked_account_id: "user@example.com",
    name: "User",
    udf: {
      phone: "1234567890",
      address: "123, Queen Street",
    },
    your_app: {
      app_id: "64da0b57c9ae95561bb0a2ea",
      auth_credentials: {
        Authorization: "<token>",
      },
    },
  });
} catch (error) {
  // Handle any error
}
```
</CodeGroup>

## Monitoring linked accounts

See how each customer uses their integration through logs for executions, schedules, events, and webhooks. These give you the observability you need to debug a specific customer's setup.

### End-user app configuration

Your customer sets their own configuration after they authenticate. You can review or change it from the dashboard.

<Steps>
  <Step title="Open the integration's config">
    In the linked account's **Overview** tab, find the app under **Integrations**, click the **⋯** menu in its row, and select **Configure**.
  </Step>
  <Step title="Choose what to configure">
    Set the value for a **Field**, or enable or disable a **Workflow** for this linked account.
  </Step>
  <Step title="Save your changes">
    Make the changes the customer needs, then click **Save**.
    <Frame>
      <img height="200" src="/images/Guides/Linked-account/linked-config.png" alt="Configuration management in a linked account" />
    </Frame>
  </Step>
</Steps>

### Integration logs

Integration logs let you monitor and troubleshoot the authentication and connectivity status of a customer's integration.

In **Integrations** under the **Overview** tab, click the **⋯** menu of any integration and select **Logs**. Refold shows every auth event for that integration in chronological order.
