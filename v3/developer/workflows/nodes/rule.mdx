The Rule Node enables conditional logic in workflows by evaluating conditions and creating different execution paths. It acts as an if/else statement, routing workflow execution down different branches depending on whether defined conditions are met.

## When to Use

- Filter records based on criteria (status, score, region)
- Route data to different systems based on properties
- Skip processing for records that do not meet requirements
- Create approval workflows with conditional branching
- Validate data before proceeding with sync operations

## Actions

### Match All Conditions

Evaluates to true only when ALL defined conditions are met (AND logic). Use when you need strict criteria matching where every condition must pass.

**Parameters:**

| Field | Required | Description |
|-------|----------|-------------|
| Variable | Yes | The field or data point to evaluate |
| Operator | Yes | Comparison operator for evaluation |
| Value | Conditional | Value to compare against. Not required for Is Empty or Is Not Empty operators |

Click **Add Condition** to add multiple conditions. All must be true for the rule to pass.

### Match Any Condition

Evaluates to true when ANY of the defined conditions are met (OR logic). Use when you have alternative criteria where only one condition needs to pass.

**Parameters:**

| Field | Required | Description |
|-------|----------|-------------|
| Variable | Yes | The field or data point to evaluate |
| Operator | Yes | Comparison operator for evaluation |
| Value | Conditional | Value to compare against. Not required for Is Empty or Is Not Empty operators |

Click **Add Condition** to add multiple conditions. Any one being true passes the rule.

### Available Operators

| Operator | Description |
|----------|-------------|
| Equals | Exact text or number match |
| Not Equals | Value does not match |
| Greater Than | Numeric comparison |
| Less Than | Numeric comparison |
| Greater Than or Equal | Inclusive numeric comparison |
| Less Than or Equal | Inclusive numeric comparison |
| Contains | Text substring match |
| Does Not Contain | Text substring not present |
| Is Empty | Value is null or empty |
| Is Not Empty | Value exists and is not empty |

## Output

The Rule Node creates two execution paths based on condition evaluation:

- **Left Path (True)**: Executes when conditions are met
- **Right Path (False)**: Executes when conditions are not met
```json
{
  "condition_result": true,
  "evaluated_conditions": [
    {
      "variable": "sap_vendor.STATUS",
      "operator": "equals",
      "value": "Active",
      "result": true
    }
  ],
  "execution_path": "true"
}
```

<Warning>Both left (true) and right (false) paths must be connected to prevent workflow execution errors. If you do not need actions on the false path, connect it directly to the next shared node or end the branch.</Warning>

## Adding to Your Workflow

<Steps>
  <Step title="Add the node">
    Click **Add Node** in the workflow canvas, select **Rule** from the list of available nodes, then position it in your workflow.
  </Step>
  <Step title="Select condition logic">
    Choose your evaluation strategy: **Match All Conditions** (AND logic) or **Match Any Condition** (OR logic). AND logic is selected by default.
  </Step>
  <Step title="Configure conditions">
    Set Variable, Operator, and Value for each condition.
  </Step>
  <Step title="Connect execution paths">
    Connect nodes to both output paths: **Left (True)** and **Right (False)**.
  </Step>
  <Step title="Test the node">
    Run the workflow with sample data to verify conditions evaluate correctly.
  </Step>
</Steps>

## Examples

### Filter Active High-Value SAP Vendors

Route only vendors that are both active and have purchase volume above threshold.

**Action:** Match All Conditions

**Conditions:**

| Variable | Operator | Value |
|----------|----------|-------|
| sap_vendor.STATUS | Equals | Active |
| sap_vendor.PURCHASE_VOLUME | Greater Than or Equal | 100000 |

**Left Path (True):** Proceed to sync vendor to internal system

**Right Path (False):** Skip vendor, log to monitoring

**Result (True Path):**
```json
{
  "condition_result": true,
  "evaluated_conditions": [
    {"variable": "Active", "operator": "equals", "value": "Active", "result": true},
    {"variable": "250000", "operator": "greater than or equal", "value": "100000", "result": true}
  ],
  "execution_path": "true"
}
```

### Process Multiple Order Sources

Process orders from any of the approved channels.

**Action:** Match Any Condition

**Conditions:**

| Variable | Operator | Value |
|----------|----------|-------|
| netsuite_order.source | Equals | Web |
| netsuite_order.source | Equals | EDI |
| netsuite_order.source | Equals | API |

**Left Path (True):** Process order through fulfillment workflow

**Right Path (False):** Route to manual review queue

**Result (True Path when source is EDI):**
```json
{
  "condition_result": true,
  "evaluated_conditions": [
    {"variable": "EDI", "operator": "equals", "value": "Web", "result": false},
    {"variable": "EDI", "operator": "equals", "value": "EDI", "result": true}
  ],
  "execution_path": "true"
}
```

### Validate Required Fields Before Sync

Ensure required fields are populated before syncing to Tipalti.

**Action:** Match All Conditions

**Conditions:**

| Variable | Operator | Value |
|----------|----------|-------|
| event.vendor.tax_id | Is Not Empty | - |
| event.vendor.bank_account | Is Not Empty | - |
| event.vendor.email | Contains | @ |

**Left Path (True):** Create vendor in Tipalti

**Right Path (False):** Return validation error via Response Node

## Runtime Behavior

### Condition Evaluation

The Rule Node evaluates conditions in the following order:

1. **Variable Resolution**: Resolves variables from previous nodes or event data
2. **Type Conversion**: Automatically converts data types for comparison
3. **Condition Checking**: Applies the specified operator logic
4. **Path Selection**: Routes to left (true) or right (false) branch based on result

### Evaluation Logic

- **Match All Conditions**: All conditions must evaluate to true (AND logic)
- **Match Any Condition**: At least one condition must evaluate to true (OR logic)

## Troubleshooting

| Problem | Solution |
|---------|----------|
| Rule always goes to false path | Verify variable references are correct. Check data types match (numbers vs strings). |
| Variable not found error | Confirm referenced node executed successfully. Check variable path syntax. |
| Unexpected data type comparison | Use text operators (Contains, Equals) for string fields. Add Transform Node to convert types. |

<Tip>Add a Custom Code Node after your Rule Node during testing to inspect condition evaluation results and understand path selection.</Tip>
