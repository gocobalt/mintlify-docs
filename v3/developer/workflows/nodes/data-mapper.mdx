The Data Mapper Node transforms field names between different systems by applying configurable field mappings. It resolves mapping configurations without custom code, translating data formats between your internal system and external applications.

## When to Use

- Map internal fields to external application field names (NetSuite, SAP)
- Transform incoming data to match your internal system format
- Apply user-defined field mapping configurations from Config settings
- Standardize data formats between multiple third-party applications

<Info>Data Mapper handles flat field mapping only. For nested objects, arrays, or data type conversion, use the Transform Node or Custom Code Node.</Info>

## Input Parameters

<ParamField path="Input" type="object" required>
  Source data object containing fields to map. Accepts static data or variables from previous nodes.
</ParamField>

<ParamField path="Mapping" type="object" required>
  Field mapping configuration as key-value pairs. Maps source field names to destination field names.
</ParamField>

<ParamField path="Direction" type="dropdown" required>
  Mapping direction. Options: `Source to Destination`, `Destination to Source`.
</ParamField>

<ParamField path="Output Schema" type="string">
  Optional output schema definition for advanced mapping scenarios.
</ParamField>

## Output

Returns an object with transformed field names. Only mapped fields appear in output.

**Source to Destination:**
```json
{
  "FirstName": "John",
  "LastName": "Doe",
  "Email": "john@example.com"
}
```

**Destination to Source:**
```json
{
  "fname": "John",
  "lname": "Doe", 
  "email_id": "john@example.com"
}
```

## Adding to Your Workflow

<Steps>
  <Step title="Add Data Mapper Node">
    Drag the Data Mapper Node from **Utility Nodes** onto your workflow canvas.
  </Step>
  <Step title="Configure input data">
    Provide the source object from a previous node or static data. Input must be a flat JSON object.
  </Step>
  <Step title="Define field mapping">
    Create mapping rules as key-value pairs: `{"source_field": "destination_field"}`.
  </Step>
  <Step title="Select direction">
    Choose **Source to Destination** when sending data out, or **Destination to Source** when receiving data.
  </Step>
  <Step title="Test the node">
    Run the node to verify field transformations.
  </Step>
</Steps>

## Examples

### Map Internal Fields to NetSuite Contact

Transform internal contact data to NetSuite format:

**Input:**
```json
{
  "fname": "Alice",
  "lname": "Johnson",
  "email_id": "alice@techcorp.com",
  "phone_num": "+1-555-123-4567"
}
```

**Mapping:**
```json
{
  "fname": "firstName",
  "lname": "lastName",
  "email_id": "email",
  "phone_num": "phone"
}
```

**Direction:** `Source to Destination`

**Output:**
```json
{
  "firstName": "Alice",
  "lastName": "Johnson",
  "email": "alice@techcorp.com",
  "phone": "+1-555-123-4567"
}
```

### Map SAP Data to Internal Format

Transform incoming SAP vendor data to your internal structure:

**Input:** `{{sap_vendor_response}}`
```json
{
  "NAME1": "Acme Corporation",
  "LIFNR": "V001234",
  "LAND1": "US",
  "ORT01": "Chicago"
}
```

**Mapping:**
```json
{
  "vendor_name": "NAME1",
  "vendor_id": "LIFNR",
  "country": "LAND1",
  "city": "ORT01"
}
```

**Direction:** `Destination to Source`

**Output:**
```json
{
  "vendor_name": "Acme Corporation",
  "vendor_id": "V001234",
  "country": "US",
  "city": "Chicago"
}
```

## Troubleshooting

| Problem | Solution |
|---------|----------|
| Missing fields in output | Verify field names in mapping exactly match input (case-sensitive) |
| Wrong mapping direction | Check direction setting matches your data flow |
| Empty output | Confirm input is not null and contains at least one mapped field |
| Nested field errors | Use Transform Node for nested objects |
