The Delay Node pauses workflow execution for a specified duration before proceeding to subsequent steps. Use it to introduce waiting periods between actions, schedule follow-ups relative to dates, or add time buffers for external processes to complete.

## When to Use

- Add a waiting period before sending follow-up notifications
- Schedule actions relative to a base date (e.g., 2 days before a due date)
- Introduce time buffers between API calls to respect rate limits
- Delay processing until a specific time window
- Coordinate timing between dependent workflow steps

<Info>The maximum delay duration is **30 days**. All duration fields accept only integer values.</Info>

## Actions

<AccordionGroup>
  <Accordion title="Wait For">
    Pauses the workflow for a fixed duration from the current execution time.

    <ParamField path="Days" type="integer" required>
      Number of days to wait. Range: 0-30. Supports dynamic variables. Example: `0` or `{{workflow.delay_days}}`
    </ParamField>

    <ParamField path="Hours" type="integer" required>
      Number of hours to wait. Supports dynamic variables. Example: `2` or `{{config.wait_hours}}`
    </ParamField>

    <ParamField path="Minutes" type="integer" required>
      Number of minutes to wait. Supports dynamic variables. Example: `30` or `{{trigger.minutes}}`
    </ParamField>

    <ParamField path="Seconds" type="integer" required>
      Number of seconds to wait. Supports dynamic variables. Example: `0` or `{{delay_seconds}}`
    </ParamField>
  </Accordion>

  <Accordion title="Wait Till">
    Pauses the workflow until a calculated target time relative to a base date. Use this to schedule actions before or after a specific date.

    <ParamField path="Before or After Base Date" type="dropdown" required>
      Whether to wait until before or after the base date. Options: `Before`, `After`
    </ParamField>

    <ParamField path="Base Date" type="string" required>
      The reference date for calculating the wait time. Accepts valid JavaScript date formats: ISO format, `YYYY/MM/DD`, or `MM/DD/YYYY`. Supports dynamic variables. Example: `2026/01/19` or `{{order.due_date}}`
    </ParamField>

    <ParamField path="Days" type="integer" required>
      Number of days before or after the base date. Range: 0-30. Supports dynamic variables.
    </ParamField>

    <ParamField path="Hours" type="integer" required>
      Number of hours offset from the calculated date. Supports dynamic variables.
    </ParamField>

    <ParamField path="Minutes" type="integer" required>
      Number of minutes offset from the calculated date. Supports dynamic variables.
    </ParamField>

    <ParamField path="Seconds" type="integer" required>
      Number of seconds offset from the calculated date. Supports dynamic variables.
    </ParamField>

    <Warning>`DD/MM/YYYY` is **not** a valid JavaScript date format. Use `YYYY/MM/DD` or `MM/DD/YYYY` instead.</Warning>
  </Accordion>
</AccordionGroup>

## Output

<AccordionGroup>
  <Accordion title="Wait For">
```json
    {
      "status": "Success",
      "node_id": "6",
      "node_name": "Delay for 5d",
      "body": "Waited for 432000 seconds."
    }
```
  </Accordion>

  <Accordion title="Wait Till">
```json
    {
      "status": "Success",
      "node_id": "6",
      "node_name": "Delay for 1d",
      "body": {
        "days": 1,
        "hours": 0,
        "minutes": 0,
        "seconds": 0,
        "base_date": "2026/01/19",
        "operand": "-"
      }
    }
```

    | Field | Description |
    |-------|-------------|
    | `days`, `hours`, `minutes`, `seconds` | The configured offset values |
    | `base_date` | The reference date used for calculation |
    | `operand` | `-` for Before, `+` for After |
  </Accordion>

  <Accordion title="Sync Execution">
    When the workflow runs in sync execution mode, delays are skipped:
```json
    {
      "status": "Success",
      "node_id": "6",
      "node_name": "Delay for 5d",
      "body": "Waited for 0 seconds since sync execution is true."
    }
```
  </Accordion>
</AccordionGroup>

## Adding to Your Workflow

<Steps>
  <Step title="Add the Delay Node">
    In the workflow editor, click **Add Node** and select **Delay** from the Utility Nodes section.
  </Step>
  
  <Step title="Select an action">
    Choose the action based on your use case:
    - **Wait For** to pause for a fixed duration from now
    - **Wait Till** to pause until a time relative to a specific date
  </Step>
  
  <Step title="Configure the duration">
    Enter the delay duration using Days, Hours, Minutes, and Seconds fields. For **Wait Till**, also specify the base date and whether to wait until before or after that date.
  </Step>
  
  <Step title="Test the node">
    Click the **Run** tab to execute. Note that delays may be skipped in sync execution mode for testing purposes.
  </Step>
</Steps>

## Examples

<AccordionGroup>
  <Accordion title="Rate Limit Buffer Between API Calls">
    Add a 5-second delay between SAP API calls to avoid rate limiting.

    **Action:** Wait For

    | Field | Value |
    |-------|-------|
    | Days | `0` |
    | Hours | `0` |
    | Minutes | `0` |
    | Seconds | `5` |

    **Use Case:** Place this node between consecutive SAP API calls in a Loop Node to prevent 429 errors.
  </Accordion>

  <Accordion title="Send Payment Reminder Before Due Date">
    Send a payment reminder to vendors 2 days before their invoice due date.

    **Action:** Wait Till

    | Field | Value |
    |-------|-------|
    | Before or After Base Date | `Before` |
    | Base Date | `{{invoice.due_date}}` |
    | Days | `2` |
    | Hours | `9` |
    | Minutes | `0` |
    | Seconds | `0` |

    **Use Case:** Workflow triggers when an invoice is created in NetSuite. The Delay node waits until 2 days and 9 hours before the due date, then sends a reminder email.
  </Accordion>

  <Accordion title="Follow-Up After Order Delivery">
    Send a feedback request 3 days after an order's expected delivery date.

    **Action:** Wait Till

    | Field | Value |
    |-------|-------|
    | Before or After Base Date | `After` |
    | Base Date | `{{order.estimated_delivery}}` |
    | Days | `3` |
    | Hours | `0` |
    | Minutes | `0` |
    | Seconds | `0` |

    **Use Case:** After syncing an order to SAP, wait until 3 days after delivery to trigger a customer satisfaction survey.
  </Accordion>

  <Accordion title="Dynamic Delay from Configuration">
    Use customer-configurable delay values from workflow settings.

    **Action:** Wait For

    | Field | Value |
    |-------|-------|
    | Days | `{{config.retry_delay_days}}` |
    | Hours | `{{config.retry_delay_hours}}` |
    | Minutes | `0` |
    | Seconds | `0` |

    **Use Case:** Allow linked accounts to customize retry timing for failed Tipalti payment processing through workflow configuration.
  </Accordion>

  <Accordion title="Batch Processing Window">
    Wait for 1 hour to accumulate records before batch processing.

    **Action:** Wait For

    | Field | Value |
    |-------|-------|
    | Days | `0` |
    | Hours | `1` |
    | Minutes | `0` |

    **Use Case:** Collect multiple incoming webhook events, then process them as a batch to optimize NetSuite API usage.
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Delay Not Working">
    | Problem | Solution |
    |---------|----------|
    | Delay is skipped (0 seconds) | Check if the workflow is running in sync execution mode. Delays are skipped during sync execution for testing. |
    | Delay duration incorrect | Verify all fields contain integer values. Decimal values are not supported. |
    | Dynamic variable not resolving | Ensure the variable path is correct and the source node has executed successfully. |
  </Accordion>

  <Accordion title="Date Format Errors">
    | Problem | Solution |
    |---------|----------|
    | Invalid date error | Use valid JavaScript date formats: ISO format, `YYYY/MM/DD`, or `MM/DD/YYYY`. The format `DD/MM/YYYY` is not valid. |
    | Base date not parsing | Check that dynamic date variables return strings in a valid format. Use a Custom Code node to format dates if needed. |
  </Accordion>

  <Accordion title="Duration Limits">
    | Problem | Solution |
    |---------|----------|
    | Delay exceeds maximum | The maximum delay is 30 days. Split longer delays into multiple Delay nodes or use a Loop with scheduled triggers. |
    | Negative duration | Ensure all duration fields contain non-negative integers. |
  </Accordion>
</AccordionGroup>

<Tip>For delays longer than 30 days, consider using a [Schedule Trigger](/v3/developer/workflows/triggers/schedule-triggers) to re-trigger the workflow at specific intervals.</Tip>
