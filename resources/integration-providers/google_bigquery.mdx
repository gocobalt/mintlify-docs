To setup your Google BigQuery app in Refold for OAuth, you will need the following credentials from your Google Cloud Console:

- Client ID
- Client Secret
- Scopes

### Pre-requisites

1. Google Cloud Platform account with a project that has BigQuery API enabled.

<Info>
  If you haven't already created an OAuth 2.0 app in Google Cloud Console, you'd need to create one.
</Info>

### Creating an app in Google BigQuery

To create a Google BigQuery OAuth app and acquire the above mentioned credentials, please follow the steps mentioned below:

1. Log in to your Google Cloud Console at [https://console.cloud.google.com](https://console.cloud.google.com).
2. Select or create a project, then navigate to `APIs & Services` → `Credentials` in the left sidebar.

<img
  height="200"
  src="/images/googlebigquery/googlebigquery_navigation.png"
  alt="Navigation for OAuth App setup"
/>

3. Click on `Create Credentials` button and select `OAuth client ID` from the dropdown menu.
4. If prompted, configure the OAuth consent screen first:
   - Select `External` or `Internal` based on your requirements
   - Fill in the required app information (App name, User support email, Developer contact)
   - Add scopes including `https://www.googleapis.com/auth/bigquery`
   - Save and continue
5. On the Create OAuth client ID page, select `Web application` as the Application type.
6. Go to your [`Apps catalog`](https://app.refold.ai/apps) in **Refold** \> Search for `Google BigQuery` \> `Settings` \> `Use your credentials` \> `Callback Url` \> Copy it.
7. Under `Authorized redirect URIs`, click `Add URI` and paste the Callback URL copied from Refold. Click `Create`.

<img
  height="200"
  src="/images/googlebigquery/googlebigquery_config.png"
  alt="Setting up Google BigQuery OAuth App"
/>

8. A popup will display your **Client ID** and **Client Secret**. Copy both values and store them securely. You can also download the credentials as JSON.

<img
  height="200"
  src="/images/googlebigquery/googlebigquery_credentials.png"
  alt="Getting Credentials"
/>

#### Configuring credentials in Refold

App settings page lets you configure the authentication settings for an `OAuth2` based application. For your customers to provide you authorization to access their data, they would first need to install your application. This page lets you set up your application credentials.

<img
  height="200"
  src="/images/googlebigquery/googlebigquery_refold.png"
  alt="Setting up application client credentials"
/>

Provide the acquired Client ID and Client Secret under Settings of the app and save it.

#### Configuring Scopes

Refold lets you configure what permissions to ask from your users while they install your application. The scopes can be added or removed from the App settings page, under `Permissions & Scopes` section.

For some applications Refold sets **mandatory scopes** which cannot be removed. Additional scopes can be selected from the drop down. Refold also has the provision to add any custom scopes supported by the respective platform.

<img
  height="200"
  src="/images/googlebigquery/googlebigquery_scopes.png"
  alt="OAuth Scopes"
/>

Once the scopes has been added to the application in Refold, go to your Google Cloud Console and update the scopes as added on Refold. Navigate to `APIs & Services` → `OAuth consent screen` → `Edit App` → `Scopes` to add or modify the scopes for your OAuth app.

<Warning>
  If you are facing scopes missing or invalid scope error. Make sure you are not passing any custom scope not supported by the platform. And, the scopes selected here are identical to the ones selected in the platform.
</Warning>

#### Actions and triggers

Once the above setup is completed, you can create orchestrations of your use-cases using Google BigQuery actions and triggers. Following are the set of Google BigQuery actions and triggers supported by Refold.

<Tabs>
  <Tab title="Actions">
    <AccordionGroup>
      <Accordion title="Dataset">
        1. **List Datasets** - List all datasets in Google BigQuery.
        2. **Create Dataset** - Create a dataset in Google BigQuery.
        3. **Update Dataset** - Update a dataset in Google BigQuery.
        4. **Delete Dataset** - Delete a dataset in Google BigQuery.
      </Accordion>
      <Accordion title="Project">
        5. **List Projects** - List all projects in Google BigQuery.
      </Accordion>
      <Accordion title="Table">
        6. **List Tables** - List all tables in Google BigQuery.
        7. **Create Table** - Create a table in Google BigQuery.
        8. **Patch Table** - Patch a table in Google BigQuery.
        9. **List Table Data** - List table data in Google BigQuery.
        10. **Insert All Table Data** - Insert all table data in Google BigQuery.
      </Accordion>
      <Accordion title="Service Account">
        11. **Get Service Account** - Get service account details in Google BigQuery.
      </Accordion>
      <Accordion title="Others">
        12. **HTTP Request** - Make HTTP API calls to any Google BigQuery documented REST APIs.
        13. **Incremental Sync** - Check for new data in the endpoint.
      </Accordion>
    </AccordionGroup>
  </Tab>
