To connect your Sentry app in Refold using a Personal Access Token, you will need the following credentials from your Sentry account:

- Personal Access Token

### Pre-requisites

1. Sentry account with Member role or higher.

<Info>
  Personal Access Tokens act on behalf of your individual Sentry user account. If you need a token scoped to an organization rather than a user, use an Organization Token instead, available under the same Developer Settings section.
</Info>

### Creating a Personal Access Token in Sentry

To create a Sentry Personal Access Token and acquire the above mentioned credentials, please follow the steps mentioned below:

1. Log in to your Sentry account at [https://sentry.io](https://sentry.io).
2. Click on your **organization name** in the top-left corner to open the organization menu.

<img
  height="200"
  src="/images/Sentry/sentry_landing_page.png"
  alt="Sentry organization menu"
/>

3. Click on `Organization Settings` to navigate to your settings.
4. In the left sidebar, scroll down to the **Developer Settings** section and click on `Personal Tokens`.

<img
  height="200"
  src="/images/sentry/sentry_token_setting.png"
  alt="Developer Settings section in Sentry sidebar"
/>

5. You will see a list of all existing Personal Access Tokens associated with your account. Click on `Create New Token` in the top-right corner.

<img
  height="200"
  src="/images/Sentry/sentry_PAT_listpage.png"
  alt="Personal Tokens list page"
/>

6. On the **Create New Personal Token** page, enter a descriptive **Name** for your token in the Name field to help identify it later (e.g., `refold-integration`).

7. Under **Permissions**, set the access level for each module using the dropdown. Each module supports the following access levels:
   - **No Access** — the token cannot interact with this module
   - **Read** — view-only access
   - **Read & Write** — view and modify access
   - **Admin** — full administrative control

<Note>
  Create tokens with minimum read access for atleast one module. Otherwise user may not be allowed to perform any action with the application.
</Note>

<img
  height="200"
  src="/images/Sentry/PAT_sentry.png"
  alt="Create New Personal Token form with permission dropdowns"
/>

8. Once you have configured the required permissions, scroll down and click `Save Token`.
9. **Copy the token value immediately** — it will only be displayed once.

<Warning>
  Once you leave the token creation page, the full token value cannot be retrieved again. If you lose it, you will need to revoke the token and create a new one. Store your token securely in a password manager or secrets vault.
</Warning>

#### Configuring credentials in Refold

Navigate to the Refold hosted portal and enter the Personal Access Token when prompted during the Sentry connection setup.

Provide the acquired Personal Access Token in the hosted portal and complete the connection.

#### Actions and triggers

Once the above setup is completed, you can create orchestrations of your use-cases using Sentry actions and triggers. Following are the set of Sentry actions and triggers supported by Refold.

<Tabs>
  <Tab title="Actions">
    <AccordionGroup>
      <Accordion title="Organization Members">
        1. **List Members** - List all organization members in Sentry.
        2. **Add Member** - Adds a new member to the organization in Sentry.
        3. **Delete Member** - Deletes a member from the organization in Sentry.
      </Accordion>
      <Accordion title="Projects">
        4. **Create Project** - Creates a new project in Sentry.
        5. **List Projects** - Lists all projects in Sentry.
        6. **Delete Project** - Deletes a project in Sentry.
        7. **Update Project** - Updates a project in Sentry.
      </Accordion>
      <Accordion title="Others">
        8. **HTTP Request** - Make HTTP API calls to any Sentry documented REST APIs.
        9. **Incremental Sync** - Check for new data in the endpoint.
      </Accordion>
    </AccordionGroup>
  </Tab>
  <Tab title="Triggers">
    1. **Issue Created** - Triggers when an issue is created on Sentry.
    2. **Issue Resolved** - Triggers when an issue is resolved on Sentry.
    3. **Comment Created** - Triggers when a comment is created on Sentry.
  </Tab>
</Tabs>
