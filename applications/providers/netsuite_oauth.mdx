This is the **OAuth 2.0 (Authorization Code)** setup for NetSuite. Refold connects as a **confidential client** (Client ID + Client Secret). For the unattended machine-to-machine grant, see [Client Credentials (M2M)](/applications/providers/netsuite_m2m); for the Key-Based alternative and a side-by-side comparison of every method, see the [NetSuite overview](/applications/providers/netsuite#authentication-at-a-glance).

<Note>
Refold's OAuth uses the Authorization Code grant as a **confidential client** — your NetSuite Client ID/Secret, a callback, and the mandatory scopes below.
</Note>

To setup your NetSuite app in Refold for OAuth, you will need the following credentials from your NetSuite account:
- Client ID
- Client Secret
- Account ID
- Scopes

### Pre-requisites

1. NetSuite account.
2. OAuth Enabled in Netsuite.

### Enable OAuth in NetSuite

Before creating an app in NetSuite, you are required to enable OAuth. Follow the steps given below:

1. Navigate to `Setup` > `Company` > `Enable Features` in your Netsuite account.
<img height="200" src="/images/Netsuite/netsuite_enable.png" alt="Enabling OAuth in Netsuite"/>
2. Select the `SuiteCloud` tab present, scroll down and enable the following in your settings:
- `SOAP Web Services` (under **SuiteTalk (Web Services)**)
- `REST Web Services` (under **SuiteTalk (Web Services)**)
- `SuiteSignon` (under **Manage Authentication**)
- `OAuth 2.0` (under **Manage Authentication**)


### Creating an app in NetSuite

Follow the steps below. The required and optional fields are called out at the step where you set them.

1. Log in to your Netsuite account with Admin permissions.
2. To get your **Account ID**, navigate to `Setup` > `Company` > `Company Information` and copy the `Account ID` present there.
<img height="200" src="/images/Netsuite/netsuite_navigation.png" alt="Navigating for Account ID"/>
3. Click on `Setup` > `Integration` > `Manage Integrations` and choose `New`.
<img height="200" src="/images/Netsuite/netsuite_create.png" alt="Navigating for App creation"/>
4. Enter the App **Name**, then in the **OAuth 2.0** section:
- ✅ **Required — Authorization Code Grant:** check this box.
- ⬜ **Leave Public Client unchecked.** This is what makes NetSuite issue the **Client Secret** that Refold's OAuth requires. (A public client is issued no secret, so the connection would fail.)
5. ✅ **Required — Redirect URI:** open your [`Apps catalog`](https://app.refold.ai/applications) in **Refold** > search `Netsuite` > `Settings` > select **OAuth 2.0** > copy the `Callback Url` (`https://app.refold.ai/api/v1/netsuite/callback`) and paste it into the `Redirect URI` field under **OAuth 2.0** in NetSuite.
<img height="200" src="/images/Netsuite/netsuite_refold_callback.png" alt="Refold NetSuite app settings — OAuth 2.0 auth method and Callback URL"/>
6. ✅ **Required — Scope:** check **REST Web Services** (`rest_webservices`) and **SuiteAnalytics Connect** (`suite_analytics`). *Optional:* **RESTlets** if you use RESTlet-based actions. Click `Save`.
<img height="200" src="/images/Netsuite/netsuite_config.png" alt="Setting up Developer app"/>

7. Scroll down and copy the **Client ID** and **Client Secret** from the `Client Credentials` section.
<img height="200" src="/images/Netsuite/netsuite_credentials.png" alt="Getting Credentials"/>

<Warning>
Leaving **Public Client** unchecked is what makes NetSuite issue the **Client Secret** Refold's OAuth needs today. If you check it, no secret is generated and the confidential setup will fail.
</Warning>

<Tip>
**Set Refresh Token Validity high to reduce re-authentication.** On a confidential client, the connection must re-authorize whenever the integration record's **Refresh Token Validity** lapses — and on the Authorization Code Grant it **defaults to 168 h (7 days)**, so users get prompted to reconnect weekly. This field is editable, so set it toward the **720-hour (30-day) maximum** to minimize re-auth. For a connection that must never require re-auth, use [Client Credentials (M2M)](/applications/providers/netsuite_m2m). See the [method comparison](/applications/providers/netsuite#authentication-at-a-glance) for details.
</Tip>

### Configuring credentials in Refold
App settings page lets you configure the authentication settings for an `OAuth 2.0` based application. For your customers to provide you authorization to access their data,
they would first need to install your application. This page lets you set up your application credentials.

<img height="200" src="/images/Netsuite/netsuite_refold_credentials.png" alt="Refold NetSuite app settings — Client ID/Secret, Callback URL, Account ID, Single Webhook URL, and mandatory scopes"/>

Provide the acquired Client ID and Client Secret under Settings of the app and save it.

### Configuring Scopes
Refold lets you configure what permissions to ask from your users while they install your application. The scopes can be added or removed from the App settings page, under `Permissions & Scopes`
section. 

For some applications Refold sets **mandatory scopes** which cannot be removed. Additional scopes can be selected from the drop down. Refold also has the provision to add any 
custom scopes supported by the respective platform. The **Permissions & Scopes** section is shown in the settings screenshot above.

Once the scopes have been added to the application in Refold, go to your Netsuite account and update the scopes as added on Refold.

Select the OAuth App created for Refold > Click on `Edit` > Add the scopes as added in Refold > Click on `Save`. 

<Warning>
If you are facing scopes missing or invalid scope error. Make sure you are not passing any custom scope not supported by the platform. And, the scopes selected here are
identical to the ones selected in the platform.
</Warning>
