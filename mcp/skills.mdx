title: "Skills"

Skills are reusable procedures attached to an MCP server. They give agents structured guidance for operations that span multiple API calls, like "sync contacts from Salesforce to HubSpot" or "create a purchase order with vendor validation in SAP."

Without skills, agents figure out multi-step operations on their own. With skills, they follow tested procedures with defined steps, pre-checks, and error handling.

## Creating a skill

Go to your MCP server's **Skills** tab and click **+ Add Skill**.

<img src="/images/mcp-skill-create.png" alt="Creating a skill with name, description, app, and content" />

### Skill fields

| Field | Description |
|-------|-------------|
| **Name** | What the agent sees during discovery. Be descriptive (e.g., "Salesforce Lead Qualification Process"). |
| **Summary** | Short description of what the skill does and when to use it. |
| **App** | Which connected application the skill relates to. |
| **Content** | The procedure body: steps, API patterns, conditional logic, input/output specs. |
| **Status** | Enable or disable the skill without deleting it. |

## Writing skill content

Write the content as structured instructions an agent can follow. Skill content typically mixes prose with the concrete API or query patterns the agent should use — SOQL queries, REST endpoints, parameter names — so the agent doesn't have to guess them at runtime.

**Example: Lead qualification**

```markdown
## Goal
Qualify an inbound lead and assign to the right sales rep.

## Pre-checks
- Email address is provided
- Salesforce connection is active

## Steps

1. Check if contact exists in Salesforce.
   Use the SOQL query:
   `SELECT Id, AccountId, OwnerId FROM Contact WHERE Email = :email LIMIT 1`

2. If a contact exists, check for open opportunities on its account:
   `SELECT Id, StageName FROM Opportunity WHERE AccountId = :accountId AND IsClosed = false`

3. If no contact exists, create one via the `create_contact` action with:
   `{ Email, FirstName, LastName, LeadSource: "Webinar" }`

4. Score the lead: company size > 500 employees → "high",
   200–500 → "medium", < 200 → "low".

5. Assign to a sales rep using the `assign_owner` action,
   selecting the rep whose territory matches the contact's country.

## On success
Return the contact ID, qualification score, and assigned rep.

## On failure
If Salesforce is unreachable, return an error asking the user to retry.
```

<Tip>
Write skills the way you'd write a runbook for a junior team member. Clear steps, explicit conditions, no ambiguity.
</Tip>

## How agents use skills

When **Retrieve Skill** is enabled, two tools are added to the server: `SEARCH_INTEGRATION_GRAPH` and `LOAD_SKILL`. The full parameter reference for both is in [Tools Reference](/mcp/tools-reference#skill-tools).

### Discovery: how skills reach the agent

When an agent connects to a server with skills enabled, Refold injects skill summaries (name, ID, and a short description) into the agent's context at session start. To keep the context window lean, only the **top 10 most relevant skills** are surfaced, ranked by:

1. **App match** — skills associated with apps configured on this server rank higher
2. **Usage count** — skills used more often by other agents on this server rank higher
3. **Confidence** — Refold's match score for the skill against the server's configured apps and actions

The agent doesn't get full skill content upfront. It loads content on demand when a skill matches the task.

### Typical flow

```
User: "Qualify this lead from the webinar"

1. Agent already sees "Lead Qualification" in the injected skill list.
   It calls LOAD_SKILL(skill_id="...") to get the full procedure.

2. If the skill isn't in the injected top 10, the agent calls
   SEARCH_INTEGRATION_GRAPH("Lead Qualification") to find it,
   then LOAD_SKILL on the result.

3. With the procedure loaded, the agent follows the steps,
   calling RESOLVE_ACTIONS and EXECUTE_ACTION for each
   integration action the skill requires.
```

## When to use skills vs raw actions

| | Skills | Raw actions |
|--|--------|-------------|
| **Scope** | Multi-step procedures with logic | Single API calls |
| **Agent guidance** | Agent follows your defined steps | Agent decides the approach |
| **Error handling** | Defined in the procedure | Agent handles on its own |
| **Best for** | Complex business processes | Simple CRUD operations |

Start with raw actions. Add skills when you notice agents struggling with multi-step operations or making inconsistent decisions about step ordering.
